# wunder 宣讲叙事主线

本主线用于向“只用过对话式大模型”的同事讲清 wunder 的价值与结构，强调“从提问到落地”的连贯故事线，适配 `docs/ppt` 的页面结构与制作要求。

## 1. 核心叙事句
- 主线：从“提问”走向“落地”，把聊天变成可执行结果并沉淀为资产。
- 关键词：核心理念 → 工具体系 → 工作区 → 前沿特性（记忆/压缩、A2UI、A2A） → 智能体线程管理 → 快速开始。

## 2. 章节叙事结构（与 PPT 对应）
### 01 核心理念：从“会聊”到“会做事”
- 讲什么：对开发者一切是接口、对模型一切皆工具；/wunder 统一入口、流式过程、任务编排。
- 想让听众记住：wunder 不是聊天工具，而是“可执行链路”。
- 承接句：先把理念说清楚：从提问到落地。

### 02 工具体系：机制就位后，能力被拆成六类工具
- 讲什么：内置 / MCP / Skills / 知识库 / 自建 / 共享，统一“是什么 / 有什么用 / 如何管控”。
- 想让听众记住：能力被工具化后才能被治理和复用。
- 承接句：理念落地到工具，能力开始可控。

### 03 工作区：工具执行需要长期落脚点
- 讲什么：每个用户持久化工作区、文件与产物沉淀、可再次复用。
- 想让听众记住：输出不是对话文本，而是可交付资产。
- 承接句：工具要落地，就必须有长期的存放与复用。
- 演示页：紧跟此节展示“Python 画爱心 → 保存到临时文件区 → 下载”，形成工具 + 工作区闭环。

### 04 前沿特性：记忆与压缩 / A2UI / A2A
- 讲什么：记忆与上下文压缩保证长对话续航；A2UI 让输出可直接渲染为界面；A2A 让能力标准化对外。
- 想让听众记住：前沿特性让系统可持续、可展示、可互通。
- 承接句：基础能力稳定后，补齐记忆、界面与对外标准化。

### 05 智能体线程管理：当任务并发起来，监视保障稳定
- 讲什么：生命周期、监控事件、可中止。
- 想让听众记住：过程可观测、可追责、可控。
- 承接句：当任务并发起来，线程管理保障稳定。

### 06 快速开始：把能力回到真实业务场景
- 讲什么：选场景、选工具、固化流程。
- 想让听众记住：从一个高频需求开始，就能快速见效。
- 承接句：把能力回到真实业务场景，形成持续闭环。

## 3. 一句话结尾
- wunder 让大模型从“会聊”变成“会做事”，并能长期稳定沉淀与复用。

## 4. PPT 制作要求
- 全局规范：浅色主题、16:9 安全区、内容不溢出、纯 HTML + JS、无备注模式；每一点只用且必须用一页呈现；欢迎页和目录页之外每页顶部必须标注“第几节”定位与章节进度，承接句用于串联，不出现“下一节”字样。
- 欢迎页：只保留大标题 + 一句副标题或口号，避免过多元素。
- 目录页：章节顺序固定为 核心理念 / 工具体系 / 工作区 / 前沿特性 / 智能体线程管理 / 快速开始。
- 核心理念：仅 1 页，突出“/wunder 统一入口、SSE 流式过程、编排 + 工具执行链路”。
- 工具体系：先 1 页总览，再按 6 类各 1 页；每一类工具只用 1 页，结构统一为“是什么 / 有什么用 / 治理要点”，便于横向对比。
- 内置工具：覆盖文件读写/搜索/替换、命令执行、ptc 临时脚本；强调 allow_paths / allow_commands 管控。
- MCP 工具：说明 server@tool 调用、外部系统接入、allow_tools 白名单与超时控制。
- Skills 工具：强调 SKILL.md + run.py、SOP 固化、一次触发完整流程。
- 知识库工具：说明 Markdown 构建、标题切分、query/limit 输入、重排序与可追溯回答。
- 自建工具：路径 `data/user_tools/<user_id>`，隔离可控、个人可用、可选共享。
- 共享工具：共享的是工具清单与配置，不共享工作区；命名 `owner_id@tool`，使用者需显式启用。
- 工作区：说明路径、持久化资产、与工具联动；强调“产出可复用”。
- 智能体功能演示：用 Python 绘制爱心图并保存到临时文件区，随后展示用户下载，证明“工具 + 工作区”闭环。
- 前沿特性：拆成三页，依次为“记忆与上下文压缩 / A2UI / A2A”，统一标注“第4节 前沿特性”。
- 记忆与压缩：讲清触发阈值、结构化摘要注入与自动续航；强调统计的是“上下文占用 token”，不是总消耗。
- 长期记忆：说明记忆生成时机、存储条数、注入方式与可控开关。
- A2UI：说明模型输出 A2UI JSON、SSE 事件与前端渲染方式，强调“回答即界面”价值。
- A2A：说明 JSON-RPC + SSE 接口、AgentCard 发现与统一鉴权，强调跨系统互通。
- 智能体线程管理：展示生命周期与并发规则，强调可监控、可取消、冲突返回。
- 快速开始：三步走（选场景/选工具组合/固化流程），给出高频试点示例。
- 结束页面：谢谢 + 一句收束口号 + Q&A 引导。
