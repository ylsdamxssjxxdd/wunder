# wunder 宣讲叙事主线（重构版）

本主线用于向“只接触过聊天式大模型”的听众讲清 wunder 的最新价值：它已经从单点问答工具，演进为**可执行、可治理、可长期运行**的智能体调度底座。

## 1. 核心叙事句

- 主线：把“提问”升级为“可执行任务”，再把一次次执行沉淀成组织资产。
- 关键词：一核三形态 → 主链路可追踪 → 能力底座可复用 → 多入口可接入 → 治理体系可落地 → 业务上线可复制。

## 2. 受众与目标

- 受众：业务负责人、架构负责人、平台开发、交付团队。
- 目标：在 20 页内回答三件事：
  - wunder 现在到底是什么（重构后定位）
  - 为什么它能稳定支撑复杂任务（架构与治理）
  - 如何低风险落地（运行形态与试点路径）

## 3. 章节叙事结构（与 PPT 对应）

### 第1节 重构定位（第2-3页）

- 讲什么：wunder 不是“又一个聊天壳”，而是统一调度内核；server/cli/desktop 三形态共用同一 Rust 核心。
- 想让听众记住：能力统一为接口，模型统一调用工具，组织统一沉淀资产。
- 承接句：定位明确后，下一步是看它如何把一次请求稳定跑通。

### 第2节 主链路与并发模型（第4-7页）

- 讲什么：`/wunder` 请求进入 AgentRuntime 与 Orchestrator，支持 WebSocket 优先、SSE 兜底、事件回放；主线程与队列协同处理并发。
- 想让听众记住：系统不是“黑箱回复”，而是“可观测、可恢复、可排队”的执行链路。
- 承接句：链路稳定后，核心问题变成“能力从哪里来、怎么沉淀”。

### 第3节 能力底座（第8-12页）

- 讲什么：工具体系（内置/MCP/Skills/知识库/自建/共享/A2A）+ 工作区 + 提示词模板治理 + 记忆压缩 + 蜂群协作。
- 想让听众记住：wunder 的扩展方式统一、资产沉淀持续、长会话可续航。
- 承接句：底座有了，才有资格谈“多入口接入与跨系统协同”。

### 第4节 多入口与外部互通（第13-15页）

- 讲什么：用户前端、管理端、CLI、Desktop、渠道 webhook、Gateway、A2A、MCP 的统一接入。
- 想让听众记住：一次能力建设，可在多渠道复用，不需要重复造轮子。
- 承接句：入口越多，越需要治理体系保证长期稳定。

### 第5节 治理与稳定性（第16-18页）

- 讲什么：组织权限与配额、安全边界（allow_paths/allow_commands/sandbox）、监控压测评估。
- 想让听众记住：wunder 不只“能跑”，还“可管、可控、可审计、可回归”。
- 承接句：治理闭环建立后，落地就不再依赖个人经验。

### 第6节 运行与落地（第19-20页）

- 讲什么：server/cli/desktop 选型与持久化策略，试点三步法（选场景→配能力→固化流程）。
- 想让听众记住：可以从一个高频场景小步上线，再逐步扩展到组织级能力平台。
- 收束句：wunder 让大模型从“会回答”走向“可执行、可治理、可持续交付”。

## 4. 一句话结尾

- wunder 的本质，不是把模型接进来，而是把“执行能力 + 治理能力 + 资产沉淀能力”长期留在组织里。

## 5. PPT 编写与演讲要求（重构版）

- 页面总数：20 页（含封面目录、结尾）。
- 章节定位：除第 1 页外，每页顶部必须有“第 X 节”标识与章节内导航。
- 叙事节奏：每页只传达 1 个核心结论，承接句只做过渡，不写“下一节”。
- 信息口径：
  - `/wunder` 入口传入的 `user_id` 可为虚拟标识，不要求是已注册账号。
  - 会话轮次区分“用户轮次/模型轮次”。
  - token 统计口径强调“上下文占用量”，不是总消耗量。
  - 通讯策略强调“WebSocket 优先，SSE 兜底”。
- 图示要求：所有核心链路页必须配图，图中术语与代码/文档口径一致（AgentRuntime、Orchestrator、Team Runs、Gateway 等）。
- 风格要求：浅色主题，16:9 安全区，避免信息堆叠；中文术语统一，不混用旧版隐喻名称。

## 6. 各页目标速查（20页）

1. 欢迎与目录：给出“执行型智能体平台”的总命题。
2. 重构背景：为什么必须从“聊天”升级到“执行”。
3. 一核三形态：server/cli/desktop 统一内核。
4. 架构总览：入口层-调度层-能力层-治理层。
5. 请求主链路：从用户请求到最终回复。
6. 流式与恢复：WS 首选、SSE 兜底、断点续传。
7. 并发模型：主线程、队列、会话状态。
8. 能力地图：工具体系全景。
9. 工作区：资产沉淀与复用。
10. 提示词治理：模板包与双语策略。
11. 记忆压缩：长会话续航机制。
12. 蜂群协作：TeamRun/TeamTask 流程。
13. 多入口协同：前端、管理端、CLI、Desktop。
14. 渠道与多模态：飞书/WhatsApp/QQ 等接入。
15. 对外互通：A2A、MCP、Gateway 与外部嵌入。
16. 组织治理：用户、单位、权限与配额。
17. 安全边界：白名单、拒绝规则、沙盒执行。
18. 可观测性：监控、吞吐压测、能力评估、回归。
19. 运行策略：三形态选型与持久化落点。
20. 落地路线：试点方案与收束口号。
