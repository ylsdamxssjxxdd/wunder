# wunder 宣讲叙事主线

本主线用于向“只用过对话式大模型”的同事讲清 wunder 的价值与结构，强调“从提问到落地”的连贯故事线，适配 `docs/ppt` 的页面结构。

## 1. 核心叙事句
- 主线：从“提问”走向“落地”，把聊天变成可执行结果。
- 关键词：机制 → 工具 → 资产 → 稳定性 → 治理 → 落地。

## 2. 章节叙事结构（与 PPT 对应）
### 01 工作机制：故事从一次请求开始
- 讲什么：/wunder 如何把用户问题编排成可执行动作。
- 想让听众记住：模型不只是回答，而是进入“执行链路”。
- 承接句：故事从一次请求开始。

### 02 六种工具体系：机制就位后，能力被拆成工具
- 讲什么：内置 / MCP / Skills / 知识库 / 自建 / 共享。
- 想让听众记住：一切能力都被工具化，工具可控、可复用。
- 承接句：机制就位后，能力被拆成六类工具。

### 03 工作区：工具执行需要长期落脚点
- 讲什么：每个用户的持久化工作空间与产物沉淀。
- 想让听众记住：产出不是聊天记录，而是可复用的资产。
- 承接句：工具执行需要长期落脚点，这就是工作区。

### 04 上下文压缩：对话越长越重，需要稳定机制
- 讲什么：触发条件、结构化摘要、自动续航。
- 想让听众记住：长对话不会“跑崩”，系统会自我减负。
- 承接句：对话越长越重，需要机制保持可持续。

### 05 长期记忆：压缩解决单次会话，记忆贯穿长期协作
- 讲什么：会话结束后的记忆生成与注入。
- 想让听众记住：系统会“记住你”，跨会话持续一致。
- 承接句：压缩解决单次会话，记忆贯穿长期协作。

### 06 线程监视：当任务并发起来，监视保障稳定
- 讲什么：生命周期、监控事件、可中止。
- 想让听众记住：过程可观测、可追责、可控。
- 承接句：当任务并发起来，监视保障稳定。

### 07 落地与 Q&A：把能力回到真实业务场景
- 讲什么：选场景、选工具、固化流程。
- 想让听众记住：从一个高频需求开始，就能快速见效。
- 承接句：把能力回到真实业务场景，形成持续闭环。

## 3. 一句话结尾
- wunder 让大模型从“会聊”变成“会做事”，并且能长期稳定复用。
