# 管理端面板索引

管理员侧前端的关键价值不是“把所有东西堆在一个页面”，而是把治理入口分组。

## 系统分组

这里更偏全局治理：

- 内部状态
- 系统设置
- 单位管理
- 用户管理
- 外链管理
- 渠道监控

如果你要看运行态、组织和全局配置，优先来这一组。

## 智能体分组

这里更偏会话和能力治理：

- 模型配置
- 线程管理
- 预设智能体
- 工具管理
- 系统提示词

如果你要看“模型到底怎么被配置和暴露”，优先来这一组。

## 调试分组

这里更偏工程验证：

- 吞吐量测试
- 性能测试
- 蜂群测试
- 能力评估
- 调试面板
- LSP

如果你在做联调、压测或能力验证，这组最重要。

## 文档分组

当前管理员侧已经把文档入口单独分组，主要包括：

- 使用说明书
- 接口

它们的职责并不一样。

### 使用说明书

这是公开文档站的嵌入入口。

适合查：

- 入门
- 概念
- 接入
- 运维
- 参考

### 接口

适合：

- 快速查 `/wunder` 和相关接口
- 看字段、示例和接入方式

## 不同人最常用什么

### 管理员

最常看：

- 系统设置
- 用户管理
- 渠道监控
- 使用说明书

### 开发者

最常看：

- 调试面板
- 接口
- 工具管理
- 性能测试

### 交付或培训人员

最常看：

- 使用说明书

## 一个实用建议

如果你不知道某个问题应该从哪里开始查，按这个顺序：

1. 先看使用说明书有没有现成说明
2. 再看接口页有没有明确字段
3. 再回到对应治理面板做验证
