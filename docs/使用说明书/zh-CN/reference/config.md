# 系统设置参考

这页解释各项系统设置的含义，帮助你在需要调整配置时做出正确选择。

## 设置分类

Wunder 的设置按职责分为几大类：

- **基础设置**：服务地址、端口、运行模式
- **安全设置**：访问控制、权限边界
- **存储设置**：数据存储位置和方式
- **工具设置**：可用工具范围
- **模型设置**：AI 模型选择和参数
- **渠道设置**：外部平台接入

## 基础设置

### 服务地址与端口

**作用**：决定 Wunder 服务监听的网络地址和端口。

**推荐做法**：
- 本地使用：默认值即可
- 团队部署：设置团队可访问的地址

### 运行模式

**作用**：决定 Wunder 以何种形态运行。

**可选模式**：
- **Desktop 本地模式**：适合个人本地使用，所有数据存本地
- **Server 模式**：适合团队/组织使用，支持多用户和集中管理

**选择建议**：
- 个人使用 → Desktop 本地模式
- 需要多用户协作、集中管理 → Server 模式

### 并发限制

**作用**：限制同时活跃的会话数量，防止系统过载。

**推荐做法**：
- 个人使用：默认值足够
- 团队使用：根据用户数和硬件资源调整

## 安全设置

### API 密钥

**作用**：保护管理端和核心接口的访问安全。

**重要提醒**：
- 密钥应足够复杂（建议超过 16 位，包含字母、数字、特殊字符）
- 不要使用容易被猜到的密钥
- 定期更换密钥

### 访问路径控制

**作用**：限制智能体能访问的文件路径和执行的命令。

**配置要点**：
- **允许的路径**：指定智能体可以操作的目录范围
- **禁止的模式**：指定禁止访问的文件类型或路径

**安全建议**：
- 不要无限制开放整个文件系统
- 明确指定允许访问的工作目录
- 禁止访问敏感系统目录

### 外部认证密钥

**作用**：当需要把 Wunder 嵌入其他系统时，用于验证外部请求。

**适用场景**：
- 从其他系统调用 Wunder 的能力
- 嵌入式集成场景

## 存储设置

### 数据库类型

**作用**：决定使用哪种数据库存储业务数据。

**可选类型**：
- **SQLite**：轻量级本地数据库，适合 Desktop 本地模式
- **PostgreSQL**：专业级数据库，适合 Server 多用户模式

**选择建议**：
- Desktop 本地使用 → SQLite（默认）
- Server 团队部署 → PostgreSQL（推荐）

### 工作区位置

**作用**：决定用户文件和智能体产物的存储位置。

**重要提醒**：
- Server 模式下需要确保工作区目录持久化（使用 NAS 或网络存储）
- 不要把工作区放在临时目录

### 向量存储

**作用**：知识库检索功能使用的存储后端。

**默认方案**：Weaviate

**适用场景**：需要使用知识库检索能力时配置。

## 工具设置

### 内置工具开关

**作用**：控制哪些内置工具可供智能体使用。

**安全考虑**：
- 敏感环境建议关闭命令执行类工具
- 按实际需求开放必要的工具

### 浏览器工具

**作用**：让智能体能够浏览网页、截图。

**配置要点**：
- 本地使用：直接启用
- Server 部署：可能需要 Docker 容器支持

### MCP 扩展工具

**作用**：通过 MCP 协议接入外部工具服务。

**适用场景**：
- 需要更多专业能力（如专业数据库操作、特定 API 调用）
- 需要接入第三方服务

## 模型设置

### 可用模型列表

**作用**：决定智能体可以选择哪些 AI 模型。

**配置要点**：
- 添加模型时需要填写 API 密钥和接口地址
- 不同模型有不同的计费方式和能力特点
- 建议提供多个选项让用户按需选择

### 上下文限制

**作用**：限制对话的上下文长度，防止超出模型能力。

**推荐做法**：
- 设置为模型支持的最大值
- 超出时系统会自动压缩历史内容

## 渠道设置

### 渠道账号绑定

**作用**：绑定飞书、企业微信等外部平台账号。

**配置要点**：
- 需要在对应平台创建应用并获取凭证
- 按渠道类型填写相关配置

### Webhook 配置

**作用**：让外部平台能把消息推送到 Wunder。

**配置要点**：
- 设置可公开访问的接收地址
- 在外部平台配置 Webhook URL

## 配置生效规则

### 修改后何时生效

- **新设置**：立即对新会话生效
- **已有会话**：不会自动改变，保持创建时的设置

### 配置层级

- **系统默认配置**：最底层，所有未指定值回退到这里
- **用户个人设置**：覆盖系统默认，只影响该用户
- **智能体设置**：最上层，只影响该智能体

**生效顺序**：智能体设置 > 用户设置 > 系统默认

## 常见配置误区

| 误区 | 正确理解 |
|------|----------|
| 改了设置旧会话自动变 | 旧会话保持创建时的配置，只有新会话使用新设置 |
| SQLite 和 PostgreSQL 只是性能不同 | SQLite 适合本地单机，PostgreSQL 适合多用户并发 |
| 工作区可以放临时目录 | 工作区需要持久化，临时目录会被清理 |
| 密钥简单点方便记忆 | 密钥复杂是安全的基本要求 |

## 按场景的配置建议

### 个人本地使用（Desktop）

- 数据库：SQLite（默认）
- 工作区：本地目录
- 安全：适度限制敏感路径
- 工具：按需开放

### 团队协作（Server）

- 数据库：PostgreSQL
- 工作区：网络存储（持久化）
- 安全：严格配置密钥和路径限制
- 工具：按角色分配权限
