# 提示词与个性化

提示词模板是决定智能体"性格"和回答风格的关键配置。通过选择或自定义模板，你可以让智能体更适合你的使用场景。

## 什么是提示词模板

**简单理解**：提示词模板就是给智能体的"工作手册"，告诉它：
- 自己的角色定位
- 如何思考和回答问题
- 有哪些工作原则和限制
- 如何使用工具和技能

不同的模板会让智能体表现出不同的风格和能力侧重。

## 内置模板包

Wunder 提供两套内置模板包：

### 中文模板包（default-zh）

- **特点**：使用中文进行思考和回答
- **适合**：中文对话场景
- **风格**：默认的中文助手风格

### 英文模板包（default-en）

- **特点**：使用英文进行思考和回答
- **适合**：英文对话场景
- **风格**：默认的英文助手风格

**选择建议**：根据你主要使用的语言选择对应模板包。

## 模板包的组成

每个模板包包含多个分段，共同构成智能体的完整"指导手册"：

| 分段 | 作用 |
|------|------|
| 角色 | 定义智能体是谁、扮演什么角色 |
| 工程原则 | 定义工作方式和思考方法 |
| 工具协议 | 定义如何使用工具 |
| 技能协议 | 定义如何调用技能 |
| 记忆 | 定义如何使用长期记忆 |
| 附加说明 | 其他补充指令 |

## 如何切换模板包

### 在界面中切换

1. 进入智能体设置页面
2. 找到"提示词模板"选项
3. 选择想要的模板包
4. 保存设置

### 生效规则

- **新对话**：切换后创建的新对话会使用新模板
- **已有对话**：不会改变，保持创建时的模板

**重要提醒**：模板切换只影响新会话，不会修改已有的对话。

## 创建自定义模板

如果你想进一步定制智能体风格，可以创建自己的模板包：

### 创建方法

1. 在模板设置页面点击"新建模板包"
2. 选择一个基础模板复制（推荐从内置模板开始）
3. 编辑各个分段的内容
4. 保存后即可使用

### 编辑建议

- **角色分段**：明确智能体的定位（如"编程助手"、"数据分析专家"）
- **工程原则**：设定回答的风格（如简洁/详细、正式/轻松）
- **工具协议**：不需要修改（除非有特殊需求）
- **附加说明**：可以添加你的特殊要求

### 示例：创建编程助手模板

角色分段可以这样写：

```
你是一位专业的编程助手，擅长多种编程语言和软件工程实践。
你的回答要：
- 代码示例清晰完整
- 解释简洁易懂
- 遵循最佳实践
- 注意安全性
```

## 模板生效的层级

智能体的最终提示词由多个层级组合：

1. **系统模板**：管理员设置的全局基础模板
2. **用户模板**：你个人选择的模板包
3. **智能体设置**：特定智能体的额外配置

**组合规则**：上层设置覆盖下层，缺失部分回退到下层。

## 常见问题解答

### 切换模板后为什么旧对话没变化？

因为每个对话在创建时就确定了提示词，之后保持冻结。这是为了保证对话的一致性，防止中途改变导致行为混乱。

### 想修改某个对话的提示词怎么办？

没法直接修改已有对话的提示词。你可以：
- 创建新对话使用新模板
- 或者把原对话的内容复制到新对话继续

### 不同智能体可以用不同模板吗？

可以。每个智能体可以独立设置模板。

### 模板内容越长越好吗？

不一定。模板应该：
- 明确关键要求
- 避免冗余和冲突
- 简洁有效

过长或矛盾的模板反而可能影响效果。

### 中文模板可以用英文回答吗？

可以，模板只是默认倾向。你可以在对话中明确要求用英文回答，智能体会遵从。

## 模板管理技巧

### 推荐做法

1. 先从内置模板开始，熟悉后再自定义
2. 创建多个模板包应对不同场景
3. 保持模板简洁明确
4. 定期回顾和优化模板内容

### 避免的做法

1. 随意拼凑模板内容
2. 设置相互矛盾的要求
3. 模板过长导致核心要求不突出
4. 不考虑实际效果只追求"完美"
