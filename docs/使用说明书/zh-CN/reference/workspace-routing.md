# 工作区管理参考

Wunder 的工作区是你和智能体的文件存放空间。理解工作区的组织方式，能帮助你更好地管理文件、找到智能体生成的产物。

## 工作区的概念

**简单来说**：工作区就是一个专门的文件夹，用于存放：
- 你上传给智能体分析的文件
- 智能体在工作中生成的文件
- 任务执行过程中的临时文件

## 工作区类型

### 个人工作区

**用途**：存放你个人的文件。

**位置**：每个用户都有独立的个人工作区，其他人无法访问。

**你可以在里面放**：
- 自己的文档、资料
- 想让智能体分析的文件
- 智能体为你生成的产物

### 智能体工作区

**用途**：存放特定智能体的工作文件。

**特点**：
- 每个智能体可以有独立的工作区
- 智能体在这个空间里执行文件操作
- 不同智能体的文件相互隔离

**为什么需要**：
- 防止不同智能体的文件混在一起
- 智能体可以有独立的"工作台"

## 如何访问工作区

### 通过界面访问

- 位置：桌面应用或网页端 → 左栏"文件"或"工作区"
- 操作：可以浏览目录、上传文件、下载文件、删除文件

### 通过对话使用

在对话中，你可以：
- 直接上传文件让智能体使用
- 让智能体读取你工作区里的文件
- 让智能体把结果保存到工作区

## 文件上传与处理

### 上传方式

- **对话框上传**：点击上传按钮或直接拖拽文件到对话框
- **工作区页面上传**：在工作区界面点击上传

### 支持的文件类型

| 类型 | 说明 |
|------|------|
| 文档 | PDF、Word、Markdown 等，会自动转换为文本供智能体分析 |
| 图片 | PNG、JPG 等，智能体可以直接查看和分析 |
| 音频 | MP3、WAV 等，会自动转写为文字 |
| 视频 | MP4 等，会按设定帧率提取画面分析 |
| 代码文件 | 各种编程语言文件，智能体可直接读取 |
| 其他 | 其他类型文件按原样存储，智能体可尝试读取 |

### 上传后的处理

某些文件上传后需要处理（如文档转文字、音频转写），处理完成后才能使用。处理过程中发送按钮会被禁用，这是正常保护机制。

## 文件下载

### 智能体生成的文件

当智能体创建了文件（如报告、代码、图片等），你可以：
- 在工作区中找到并下载
- 有些可以直接在对话中预览

### 下载位置

- 界面上点击文件 → 选择下载
- 文件会下载到你本机的下载目录

## 工作区组织建议

### 推荐做法

1. **分类存放**：不同类型的项目文件放在不同子目录
2. **及时清理**：不需要的文件及时删除，保持整洁
3. **命名规范**：给文件取有意义的名字，方便查找

### 避免的做法

1. **堆在一起**：所有文件都放根目录，难以管理
2. **长期不清理**：积累了大量无用文件
3. **随意命名**：文件名没有意义，找起来困难

## 工作区与临时目录的区别

### 工作区（长期存储）

- 存放长期使用的文件
- 内容会持久保存
- 你主动管理的空间

### 临时目录（中转区域）

- 用于文件上传下载的中转
- 内容可能被自动清理
- 不适合存放长期文件

**重要提醒**：不要把需要长期保存的文件放在临时目录。

## 常见问题解答

### 文件上传后在哪里？

上传的文件会存入当前智能体的工作区。你可以在"文件"或"工作区"页面查看。

### 为什么智能体说找不到文件？

可能原因：
- 文件名或路径不对
- 文件还没上传
- 文件在其他工作区（当前智能体无权访问）

解决办法：
- 确认文件已上传
- 使用完整正确的文件名
- 检查当前智能体的工作区

### 智能体生成的文件怎么保存？

智能体通常会自动把文件保存到工作区。你可以在工作区页面找到并下载。

### 可以访问其他用户的文件吗？

不可以。每个用户的工作区是隔离的，你只能访问自己的文件。

### 文件会自动清理吗？

工作区的文件不会自动清理（长期存储）。临时目录的文件可能会被定期清理。

### Desktop 本地模式的特殊之处

Desktop 本地模式下，智能体可以访问你本机的其他目录（取决于安全设置）。这比纯工作区更灵活，但也要注意安全边界。
