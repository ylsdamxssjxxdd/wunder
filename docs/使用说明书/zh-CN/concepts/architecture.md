# 系统架构

理解 Wunder 的架构，不需要记很多术语。核心思路很简单：**一套统一内核，承载多种使用形态**。

## 三层架构

Wunder 从上到下可以理解为三层：

```
┌────────────────────────────────────────────┐
│            你看到的界面                      │
│    桌面应用 / 网页端 / 管理端 / 飞书微信     │
├────────────────────────────────────────────┤
│            系统的大脑                        │
│    调度 · 对话 · 工具 · 蜂群 · 记忆         │
├────────────────────────────────────────────┤
│            数据的地基                        │
│    数据库 · 文件存储 · 向量知识库             │
└────────────────────────────────────────────┘
```

### 第一层：界面层

这是你直接接触的部分。Wunder 有多种使用形态：

- **桌面应用**：个人本地使用的主要方式，功能最完整
- **网页端**：通过浏览器访问，适合在 Server 模式下使用
- **管理端**：管理员配置系统、管理用户的后台
- **外部渠道**：通过飞书、企业微信等第三方平台使用

不同入口的体验不同，但背后共享同一套核心能力。

### 第二层：调度与能力层

这是 Wunder 的"大脑"，负责：

- **调度执行**：接收请求，安排智能体执行任务
- **对话管理**：维护会话历史，管理线程状态
- **工具调用**：让智能体能读写文件、搜索网页、执行命令等
- **蜂群协作**：调度多个智能体协同工作
- **记忆管理**：长期资料的存储和检索

### 第三层：数据与存储层

这是数据持久化的基础：

- **数据库**：存储用户、会话、配置等核心业务数据
- **文件存储**：存放用户文件和智能体产物（工作区）
- **向量知识库**：用于知识库检索功能

## 三种运行形态

Wunder 有三种运行形态，选择取决于你的使用场景：

### Desktop（桌面应用）

**适合谁**：个人用户

**特点**：
- 本地安装，开箱即用
- 数据存在本地，隐私性强
- 可以访问本机文件和桌面
- 不需要搭建服务器

### Server（服务端）

**适合谁**：团队、组织

**特点**：
- 支持多用户同时使用
- 集中管理用户、权限和资源
- 可以接入外部渠道
- 适合生产环境部署

### CLI（命令行）

**适合谁**：开发者、自动化场景

**特点**：
- 在终端中使用
- 适合脚本化和自动化
- 开发调试方便

## 关键设计原则

### 统一内核

不管你用哪种形态，背后的调度逻辑、工具体系、事件系统都是同一套。这意味着：
- 在桌面应用里能做的事，在网页端同样能做
- 从飞书发消息给智能体，和直接在网页端对话，体验一致
- 管理员修改的设置，对所有入口统一生效

### 线程为重

Wunder 不是"一问一答"的模式，而是以持续运行的线程为中心。每条线程维护着自己的对话历史、执行状态和上下文。这让智能体可以：
- 处理需要多步骤完成的复杂任务
- 在对话中途使用工具、等待审批后继续
- 保持长时间的上下文连续性

### 安全边界

从架构层面就内置了安全考虑：
- 不同用户的数据严格隔离
- 工具执行受权限控制
- 敏感操作需要审批
- 资源使用有配额限制

## 层级关系图

Wunder 的组织层级从上到下是：

```
心舰（系统）
  └── 用户
        └── 智能体
              └── 线程（对话）
                    └── 轮次（每一步交互）
```

每一层都有明确的职责边界：
- **系统**管理全局配置和资源
- **用户**拥有独立的文件空间和设置
- **智能体**有独立的角色配置和工具集
- **线程**承载一次完整的对话或任务
- **轮次**记录每一次交互

## 常见架构问题

### 为什么桌面应用和网页端看起来一样？

因为它们复用了同一套前端界面代码。区别在于桌面应用额外提供了本地文件访问、本地运行时等桌面能力。

### 管理端为什么是独立的？

管理端面向的是系统管理员，负责的是全局配置、用户管理、渠道管理等治理工作。它和普通用户的使用场景完全不同，所以独立设计。

### Server 模式和 Desktop 模式可以共存吗？

可以。你可以在本地用桌面应用做个人工作，同时组织里的 Server 为团队提供服务。两者互不干扰。
