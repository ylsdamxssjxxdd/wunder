# 运行时与在线状态

## 原理

你和智能体的对话不只是一条聊天记录，它还带着**运行时状态**——告诉你线程现在在干什么。

### 两层状态

**线程事件状态**：表达线程现在在做什么

| 状态 | 含义 |
|------|------|
| `running` | 正在执行 |
| `waiting_approval` | 等待审批 |
| `waiting_user_input` | 等待你输入 |
| `interrupting` | 正在中断 |
| `idle` | 空闲，等待下一条消息 |
| `not_loaded` | 未加载到内存 |

**会话运行时投影**：给界面直接消费的汇总状态

| 字段 | 含义 |
|------|------|
| `streaming` | 是否正在持续输出 |
| `waiting` | 是否处于等待态 |
| `loaded` | 会话是否已加载 |
| `active` | 会话是否活跃 |
| `terminal_status` | 是否已有终态 |

## 作用

### 会话列表展示

你在会话列表看到的状态徽标来自运行时状态：
- 绿点 = 活跃会话
- 旋转图标 = 正在执行
- 等待图标 = 等待审批或输入

### 详情页状态

详情页的状态指示器告诉你：
- 当前在哪个阶段
- 是否可以发送新消息
- 是否有待处理的审批

### 排障依据

状态异常时，运行时状态是排障的起点：
- 卡住了？看 streaming 和 waiting
- 消失了？看 loaded 和 active
- 状态不对？看 terminal_status

## 实践

### 状态判断

| 想知道... | 看什么 |
|-----------|--------|
| 是否还在执行 | `runtime.streaming` |
| 是否在等待 | `runtime.waiting` |
| 是否已结束 | `runtime.terminal_status` |
| 是否能继续观察 | `runtime.loaded` 和 `runtime.active` |

### 不要只看 running

`running=true/false` 太粗了，不能区分：
- 是持续输出还是等待审批
- 会话是否已加载
- 是否已经有终态

要看完整的 `runtime` 对象。

### 状态不一致时

如果界面显示和实际状态不一致：
1. 等待几秒看是否自动恢复
2. 刷新页面或重启应用
3. 使用"重置工作状态"功能

## 常见误区

- **等待审批 = 已经结束？** 不是。等待审批是暂停，不是终态。
- **只在详情页看状态？** 不是。列表、事件页都应该显示状态。
- **事件流和摘要状态是同一层？** 不是。事件负责时间线，runtime 负责当前快照。
