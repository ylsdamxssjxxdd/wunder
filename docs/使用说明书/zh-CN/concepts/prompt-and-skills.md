# 提示词与技能

## 原理

智能体的行为由两个核心配置决定：

- **系统提示词**：定义智能体是谁、怎么思考、遵循什么原则
- **技能**：定义智能体在某类任务中的具体操作方法

### 系统提示词

系统提示词是智能体的"工作手册"，告诉它：
- 角色定位（你是谁）
- 工程原则（怎么思考）
- 工具协议（怎么使用工具）
- 技能协议（怎么调用技能）
- 记忆上下文（记住什么）

**关键规则**：系统提示词在线程创建时确定，之后保持冻结。这保证了同一线程中智能体行为一致。

### 技能（Skills）

技能是"面向智能体的操作手册"，包含：
- 完整的方法说明（SKILL.md）
- 执行脚本
- 示例和资源
- 约束和边界

智能体需要时可以通过 `skill_call` 主动读取完整的技能文档。

## 作用

### 提示词的作用

- **角色定位**：让智能体知道自己是"编程助手"还是"分析师"
- **行为约束**：定义回答风格、质量标准
- **工具使用规范**：定义如何安全高效地使用工具

### 技能的作用

- **流程化经验**：把某类任务的最佳实践封装起来
- **减少错误**：智能体不用每次从零摸索
- **一致性**：相同类型的任务用相同方法处理

### 技能分层

| 层级 | 说明 | 可修改 |
|------|------|--------|
| **内置** | 系统自带的技能 | 只读 |
| **自定义** | 用户或管理员创建 | 可编辑 |
| **外部** | 从外部引入的技能 | 取决于来源 |

## 提示词模板包

Wunder 支持多套提示词模板包：

- **中文模板包**：默认中文对话风格
- **英文模板包**：默认英文对话风格
- **自定义模板包**：根据你的需求定制

### 切换模板包

1. 在智能体设置中找到"提示词模板"
2. 选择想要的模板包
3. 保存

**生效规则**：新对话使用新模板，已有对话保持不变。

### 创建自定义模板

1. 基于一个内置模板复制
2. 编辑各分段内容（角色、原则、附加说明等）
3. 保存后即可使用

## 关键规则

### 冻结原则

线程的提示词一旦确定就冻结。修改提示词模板只影响**新创建的对话**，不会回写已有对话。

如果你改了模板但当前对话没变化——这是正常的。开新对话就会用新模板。

### 技能资源路径

技能文档中的资源路径使用 `{{SKILL_ROOT}}` 占位符。这是因为技能在不同运行形态下实际路径不同，占位符会被系统替换为正确路径。

### 提示词、技能、记忆的配合

| | 职责 |
|--|------|
| **提示词** | 定义长期稳定的行为边界 |
| **技能** | 定义某类任务的附加执行协议 |
| **记忆** | 定义线程初始化时的长期事实 |

三者配合，不混用职责。

## 实践

### 优化提示词

好的提示词模板应该：
- **角色明确**：清晰定义智能体是谁
- **原则简洁**：不要堆砌太多约束，抓住关键
- **避免矛盾**：前后一致的指导
- **注重实用**：从实际效果出发调整

### 管理技能

- **查看技能**：在"工具"页面查看已安装的技能
- **启用/禁用**：按需开启或关闭技能
- **创建技能**：为你的特定工作场景创建专属技能
- **分享技能**：好的技能可以分享给团队

### 自定义智能体风格

通过组合不同的提示词模板 + 技能 + 模型，你可以创建出各种风格的智能体：
- 编程助手
- 文案写手
- 数据分析师
- 翻译专家

## 常见误区

- **改模板会立即影响当前对话？** 不会。只影响新对话。
- **技能只是说明文档？** 不是。技能是智能体可以主动读取的运行资源。
- **记忆会每轮重写提示词？** 不会。记忆只在初始化注入一次。
