# 蜂群协作

在 wunder 中，蜂群用于承载多智能体协作。

## 蜂群解决什么问题

单个智能体适合独立完成一个任务，但很多任务天然需要拆分：

- 一个负责搜集信息
- 一个负责生成结果
- 一个负责复核
- 一个负责交付产物

蜂群能力就是把这种协作变成系统内的正式结构。

## 相关概念

在当前仓库语义里，常见概念包括：

- 蜂群
- 工蜂
- WorkerCard
- HivePack

这些概念分别用于表达：

- 协作单元
- 单智能体角色
- 角色配置载体
- 可导入导出的协作资产包

## 为什么它重要

如果多智能体只是“多开几个聊天窗口”，就很难治理：

- 难以复用
- 难以迁移
- 难以导出
- 难以批量配置

蜂群能力让多智能体从临时玩法变成正式资产。

## 当前你需要先知道的事

- 蜂群是正式能力，不只是 UI 概念
- 它和工具、技能、工作区、会话状态都有关系
- 它适合复杂任务，不适合每个简单问题都强行拆分
