# 目标态系统

目标态是一种任务驱动模式。当用户设定目标后，智能体会持续工作直到目标完成，期间自动处理等待、重试、预算检查等边界情况。

## 适用场景

目标态适合以下场景：

- **复杂任务**：需要多轮执行才能完成的任务，如"开发一个完整的用户认证模块"
- **长时间任务**：智能体需要持续工作，不能一次性完成
- **有明确完成条件**：任务有清晰的"完成"定义
- **需要预算控制**：限制智能体使用的 Token 数量

**不适合目标态的场景：**

- 简单问答（一次回复就能解决）
- 开放式探索（没有明确完成条件）
- 需要频繁人工干预的任务

## 启动目标态

### 方法一：使用命令

在聊天输入框中输入：

```
/goal 完成用户认证模块的开发
```

### 方法二：使用目标编辑器

1. 点击聊天面板上方的目标图标（靶心图标）
2. 在弹出的对话框中输入目标描述
3. 可选：设置 Token 预算
4. 点击"开始"

### 方法三：让智能体创建

智能体可以通过 `create_goal` 工具创建目标。这通常发生在智能体判断任务需要持续执行时。

## 目标状态

目标有以下几种状态：

| 状态 | 含义 | 颜色标识 |
|------|------|----------|
| 活跃 | 正在执行中 | 绿色 |
| 暂停 | 用户暂停了目标 | 黄色 |
| 预算受限 | Token 预算耗尽 | 红色 |
| 已完成 | 目标已完成 | 灰色 |

### 活跃状态

目标处于活跃状态时：

- 智能体会持续执行
- 每轮执行后会自动继续下一轮
- 形象显示"运行中"动画
- 消息列表显示目标分隔线

### 暂停状态

目标暂停后：

- 智能体停止执行
- 可以通过 `/goal resume` 恢复
- 或通过目标编辑器恢复

### 预算受限

当设置了 Token 预算且已耗尽时：

- 目标自动进入预算受限状态
- 智能体停止执行
- 需要用户干预（提高预算或清除目标）

### 已完成

智能体调用 `update_goal(status="complete")` 后：

- 目标标记为已完成
- 停止自动执行
- 显示完成时间

## Token 预算

### 设置预算

在目标编辑器中可以设置 Token 预算：

```
目标：完成用户认证模块
预算：100000 tokens
```

### 预算消耗

每轮执行后，系统会累加：

- 输入 Token 数
- 输出 Token 数
- 工具调用 Token 数

当累计消耗达到预算时，目标自动进入"预算受限"状态。

### 预算建议

根据任务复杂度设置合理的预算：

| 任务类型 | 建议预算 |
|----------|----------|
| 简单功能 | 10,000 - 30,000 |
| 中等功能 | 30,000 - 100,000 |
| 复杂功能 | 100,000 - 500,000 |
| 大型项目 | 500,000+ |

## 目标执行过程

### 自动延续

目标处于活跃状态时，智能体会自动延续执行：

1. 完成一轮执行
2. 检查目标状态和预算
3. 等待 1 秒冷却期
4. 注入延续提示词
5. 开始下一轮执行

### 等待用户输入

如果智能体需要用户输入才能继续：

- 目标不会自动延续
- 等待用户回复后继续

### 完成判定

智能体判断目标完成时，会调用 `update_goal` 工具：

```
智能体：我已经完成了用户认证模块的开发，包括：
1. 用户注册功能
2. 用户登录功能
3. 密码重置功能

调用 update_goal(status="complete")
```

## 退出目标态

### 正常退出

智能体完成任务后自动退出。

### 手动退出

点击聊天面板的停止按钮，或使用命令：

```
/stop
```

或

```
/cancel
```

### 注意事项

- 退出目标态会清除当前目标
- 已执行的操作不会回滚
- 可以重新设定新的目标

## 前端显示

### 目标分隔线

在消息列表中，目标开始位置会显示分隔线：

```
────────── 目标已开始：完成用户认证模块的开发 ──────────
```

### 目标状态徽章

聊天面板顶部显示当前目标状态：

- 活跃：🟢 目标进行中
- 暂停：🟡 目标已暂停
- 预算受限：🔴 预算已耗尽
- 已完成：⚪ 目标已完成

### 目标锁定

目标态期间：

- 不能切换到其他会话
- 不能切换智能体
- 形象显示运行状态

## 命令参考

| 命令 | 作用 |
|------|------|
| `/goal` | 打开目标编辑器 |
| `/goal 目标描述` | 直接设置目标 |
| `/goal --tokens 100000 目标描述` | 设置目标并指定预算 |
| `/goal pause` | 暂停目标（仅命令行） |
| `/goal resume` | 恢复目标（仅命令行） |
| `/goal clear` | 清除目标（仅命令行） |
| `/stop` 或 `/cancel` | 退出目标态 |

## 最佳实践

### 明确的目标描述

好的目标描述：

```
完成用户认证模块的开发，包括：
1. 用户注册（邮箱验证）
2. 用户登录（支持记住我）
3. 密码重置（邮件链接）
4. 会话管理（自动过期）
```

不好的目标描述：

```
帮我写代码
```

### 合理的预算

- 不要设置过低：可能导致任务无法完成
- 不要设置过高：失去预算控制的意义
- 根据任务复杂度调整

### 及时干预

- 监控智能体执行进度
- 发现偏离时及时纠正
- 必要时暂停目标调整方向

## 常见问题

### 目标一直不完成

可能原因：

- 目标描述不够明确
- 智能体判断还有工作要做
- 遇到了阻塞问题

解决方法：

- 检查智能体的输出
- 提供更多上下文
- 手动退出并重新设定更具体的目标

### 预算耗尽但任务未完成

解决方法：

- 重新设定目标，提高预算
- 简化任务范围
- 分阶段完成

### 目标执行中断

可能原因：

- 网络问题
- 服务重启
- 系统错误

解决方法：

- 检查网络连接
- 刷新页面
- 重新设定目标
