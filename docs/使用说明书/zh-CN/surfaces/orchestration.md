# 编排功能

编排功能是蜂群进入"持续推进模式"后的专用工作台。

普通蜂群页适合一次性协作，编排页适合需要**多轮推进、保留轮次记录、从旧轮次分支**的场景。

## 适合什么场景

- 母蜂需要持续统筹多个工蜂，而不是临时做一次任务
- 你想把每轮产物按目录沉淀下来，后续可以回看
- 你想从旧轮次重新继续，形成新分支，而不是覆盖旧结果

如果只是临时并行协作，普通蜂群页更轻量。

## 进入方式

1. 左栏打开蜂群区域
2. 中栏选择一个蜂群
3. 点击「编排」进入编排页
4. 右侧栏顶部点击「新建编排」

新建编排时，系统会为母蜂和所有工蜂各自建立新线程，并把蜂群切换到这套编排状态。

注意：只有第一次「新建编排」会创建新线程，后续轮次沿用同一组线程，保持上下文连续。

## 页面布局

### 画布

- 左侧：母蜂卡片
- 中间：工蜂卡片
- 右侧：产物容器卡片

节点可拖拽，支持连接线和规整排布。

### 右侧栏

控制当前编排 run：

- **新建编排**：创建新的编排 run
- **历史**：查看历史 run
- **启动 / 关闭**：控制当前 run 是否激活
- **态势设置**：预设每轮的背景信息

蜂群标题旁的状态灯：
- 灰色 = 未激活
- 绿色 = 已激活，可继续推进
- 蓝色 = 正在执行

### 底部时间线

时间线显示的是母蜂的用户轮次，支持：
- 收起与展开
- 横向滚动
- 分支可视化
- 点击轮次查看当时的消息与产物

## 基本工作流

1. 新建编排
2. 在「态势」窗口预设各轮背景信息
3. 向母蜂发送消息
4. 母蜂调用蜂群工具，唤起工蜂
5. 工蜂在本轮产物目录里产出文件
6. 母蜂汇总结果并回复
7. 时间线新增一个轮次，可继续下一轮

如果某一轮被你主动停止，不会计入正式轮次。

## 态势设置

态势是每轮的背景信息，系统会自动附加到消息上下文中。

- 态势按轮次保存
- 可以提前预设未来轮次
- 支持导入 `.txt` 文件批量写入（用 `#1`、`#2` 分隔轮次）

## 产物查看

选中时间线中的某一轮时，页面只显示该轮的产物文件，直观回答"第 N 轮产出了什么"。

## 历史、回看与分支

### 回看旧轮次

点击时间线中的旧轮次：
- 右侧栏显示那一轮的消息
- 产物区显示那一轮的文件
- 发送入口禁用（只能查看，不能修改）

### 从旧轮次继续

从旧轮次继续发送时，系统会自动创建新的分支 run，而不是覆盖原 run。

- 原 run 保留不变
- 新分支从该轮重新继续
- 时间线显示主干与分支关系

### 删除后续内容

在时间线轮次节点上可以删除"该轮之后的内容"。删除后，当前轮保留，后续轮次和分支一起移除。

## 编排态与聊天页的关系

蜂群处于激活编排态时，聊天页中的编排线程会带明显标识，且：

- 不能直接发送消息到编排线程
- 不能新建线程覆盖编排主线程
- 历史线程可以查看，但不能破坏编排态

要继续编排，请回到编排页操作。

## 启动与关闭

### 关闭

关闭后不会离开编排页，当前 run 保留为可查看的历史，蜂群切回普通主线程。

### 启动

打开未激活的历史 run 时，可以点击「启动」重新激活。这不是新建 run，而是重新启用当前编排。

## 使用建议

- 新建编排前，先确认蜂群成员与角色已配好
- 长流程任务建议先预设好几轮态势再开始
- 想比较两种走法时，从旧轮次继续生成分支，不要覆盖旧轮
- 编排执行中需要打断，用右侧栏停止，不要切到聊天页操作
