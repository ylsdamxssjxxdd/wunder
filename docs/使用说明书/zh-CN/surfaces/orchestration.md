# 编排功能

编排功能不是普通聊天页的“另一种视图”，而是蜂群进入编排态后的专用工作台。

在这里，你可以围绕一个蜂群持续推进任务，观察母蜂与工蜂的协作过程，并按轮次查看当时的消息、态势和产物。

## 适合什么场景

适合以下情况：

- 你希望母蜂持续统筹多个工蜂，而不是临时做一次蜂群任务
- 你希望把每轮产物按目录沉淀下来，后续可以回看
- 你希望从旧轮次重新继续，形成新的分支，而不是覆盖旧结果

如果只是临时并行协作，普通蜂群页更轻；如果是连续推进、要保留轮次与产物现实，就应进入编排页。

## 进入方式

典型入口如下：

1. 打开左栏里的蜂群区域。
2. 在中栏选择一个蜂群。
3. 点击“编排”进入编排页。
4. 在右侧栏顶部点击“新建编排”，创建这次编排 run。

新建编排时，系统会为母蜂与全部工蜂各自建立新的编排主线程，并把整个蜂群切换到这套编排现实中。

注意：

- 一次编排 run 里，只有第一次“新建编排”会强制全员新线程。
- 后续轮次会一直沿用这组编排线程，保持上下文连续。

## 页面布局

编排页延续蜂群页的整体交互壳层，但默认排布不同。

### 画布

画布能力与蜂群页保持一致：

- 节点卡片可拖拽
- 支持连接线
- 支持规整与全屏
- 侧栏、动画和状态行为与蜂群页对齐

默认排布语义为：

- 左侧区域放母蜂卡片
- 中间区域放工蜂卡片
- 右侧区域放产物容器卡片

这不是死的三列表格。规整时会按这套语义重新排布，但你仍然可以手动拖拽调整。

### 右侧栏

右侧栏负责当前编排 run 的控制与消息发送，顶部主要动作包括：

- 新建编排
- 历史
- 启动或关闭
- 态势设置

蜂群标题旁会显示一个状态灯：

- 灰色：当前 run 未激活
- 绿色：当前 run 已激活，可继续推进
- 蓝色：当前 run 正在执行

### 底部时间线

时间线位于画布内部底部栏，不是单独页面区块。

它表示的是母蜂的用户轮次，而不是普通会话线程列表。时间线支持：

- 收起与展开
- 横向滚动
- 分支可视化
- 点击轮次查看当时的消息与产物快照

## 基本工作流

一个常见的编排流程如下：

1. 新建编排。
2. 在“态势”窗口中预先填写各轮态势。
3. 在右侧栏向母蜂发送消息。
4. 母蜂在编排态中调用蜂群工具，唤起工蜂。
5. 工蜂在本轮产物目录里产出文件。
6. 母蜂汇总结果并回复。
7. 时间线新增一个正式轮次，可继续进入下一轮。

如果某一轮被你主动停止，这一轮不会计入正式编排轮次。

## 态势设置

态势是每个母蜂用户轮次的背景信息。

系统会在你发送给母蜂时，自动把该轮对应的态势附加到消息上下文中；当母蜂在编排态中唤起工蜂时，当前轮态势也会自动传递给工蜂。

当前手册层面你需要知道的是：

- 态势按轮次保存
- 可以提前预设未来轮次
- 可以在态势窗口里跳到任意轮次编辑
- 支持导入 `.txt` 文件批量写入态势

导入文本使用 `#1`、`#2` 这样的轮次分隔形式，适合一次性准备很多轮的态势草稿。

## 产物怎么看

编排页里的产物节点是纯容器卡片，用来展示文件格子。

你在时间线里选中某一轮时，页面只显示该轮对应目录中的产物文件，因此可以很直观地回答“第 N 轮当时到底产出了什么”。

要点如下：

- 产物展示按轮次快照切换
- 不额外依赖业务标签
- 主要依据目录结构组织
- 工蜂各自产物会显示在对应产物容器里

## 历史、回看与分支

### 回看旧轮次

点击时间线中的旧轮次后：

- 右侧栏只显示那一轮的消息切片
- 产物区只显示那一轮的文件快照
- 发送入口会禁用

这意味着你是在“查看当时状态”，不是把母蜂线程真的回滚过去。

### 从旧轮次继续

如果你切回旧轮次后想继续推进，有两种常见方式：

- 修改该轮态势后继续
- 不改态势，直接继续发送新消息

只要你从旧轮次重新继续发送，系统就会自动创建新的分支 run，而不是覆盖原 run。

因此：

- 原 run 保留不变
- 新分支从该轮重新继续
- 时间线会显示主干与分支关系

### 删除后续内容

在时间线轮次节点上可以呼出操作菜单，删除“该轮之后的内容”。

删除后：

- 当前轮会保留
- 其后的轮次会移除
- 从这些后续轮次继续长出的后代分支也会一起移除

## 编排态与聊天页的关系

当一个蜂群处于激活编排态时，母蜂与相关工蜂的编排线程会在聊天页中带明显标识。

此时聊天页允许查看，但不允许破坏这套编排现实。你会看到以下限制：

- 不能直接发送消息到该编排线程
- 不能新建线程覆盖当前编排主线程
- 不能切换线程或改主线程
- 历史线程入口可以打开，但内容不能用来破坏当前编排态

如果你要继续这场编排，请回到编排页操作。

## 启动与关闭

右侧栏提供“启动 / 关闭”动作，用来控制当前 run 是否处于激活编排态。

### 关闭

关闭后：

- 不会离开编排页
- 当前 run 会保留为可查看的历史现实
- 整个蜂群会切回新的普通主线程

### 启动

当你打开的是一个未激活的历史 run 时，可以点击“启动”，让它重新成为当前激活编排态。

这不是新建 run，而是重新启用当前这次编排现实。

## 历史列表

右侧栏的“历史”会打开该蜂群的编排历史列表。

你可以在这里：

- 查看不同 run 的时间信息
- 打开指定历史 run
- 删除某个历史条目

如果某个历史 run 被重新加载，系统会尽量复用原有母蜂线程；部分工蜂线程如果已经缺失，会在后续需要时补建。

## 使用建议

- 新建编排前，先确认蜂群成员与角色已经配置好。
- 长流程任务建议先把未来几轮态势预设好，再开始推进。
- 想比较两种走法时，不要回头覆盖旧轮，直接从旧轮次继续生成分支。
- 编排执行中如果需要强行打断，请用右侧栏停止，而不是切到聊天页操作。
