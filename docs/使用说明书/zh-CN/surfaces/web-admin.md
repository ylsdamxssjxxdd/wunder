# 管理端界面

Wunder 的管理端不是普通用户入口。

它更像治理和调试控制台，服务对象是：

- 管理员
- 平台维护者
- 联调开发者

## 代码位置

- `web/`

## 默认职责

管理端主要负责这些事情：

- 模型配置
- 工具管理
- MCP 与 A2A 服务管理
- 用户与组织治理
- 渠道接入与运行监控
- 预设智能体治理
- benchmark 与性能观测

## 为什么它和用户前端必须分开

用户前端追求的是“工作台体验”。

管理端追求的是：

- 看全局状态
- 改配置
- 做审计和排障
- 做平台治理

这两种目标天然不同，所以 Wunder 明确把它们拆成两套界面。

## 当前你会在管理端看到什么

- 系统和模型配置页
- 工具与 MCP/A2A 调试面板
- 渠道账号与运行监控
- 预设智能体和同步面板
- 文档分组中的使用说明书与接口入口
- benchmark、吞吐、性能采样等运维能力

## 用户管理里的 Token 账务

在“用户管理 -> 设置”中，普通用户现在支持两类 Token 操作：

- 直接设置余额
- 发放 Token / 扣除 Token

这几个入口的语义不同：

- 直接设置余额：覆盖当前余额，适合修正账面值
- 发放 Token：增加用户持有余额，同时计入累计获得
- 扣除 Token：减少用户持有余额，同时计入累计消耗

管理员账户不参与 Token 余额限制，因此相关控件会在界面中禁用。

## 使用说明书和管理端的关系

当前使用说明书产物也发布在 `web/docs/` 下，并通过 `/docs/` 对外提供静态访问。

这意味着管理端和使用说明书共享同一套静态资源发布方式，但两者职责不同：

- 管理端用于治理和调试
- 使用说明书用于说明和导航

当前管理员侧文档分组里已经直接嵌入：

- [使用说明书](/docs/)
- 管理端内置接口页

## 什么时候优先看管理端文档

适合这些场景：

- 你要接入或排查 MCP/A2A
- 你要调整模型、鉴权、组织和 Token 账户
- 你要看渠道账号状态、错误和批量治理能力
