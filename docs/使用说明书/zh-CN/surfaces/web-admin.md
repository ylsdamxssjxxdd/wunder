# 管理端界面

管理端是系统治理的后台，只有管理员能访问。它不是一个聊天界面，而是配置、管理、监控的工作台。

## 面板分组

管理端按职责分成几个分组：

| 分组 | 包含面板 | 职责 |
|------|----------|------|
| **系统** | 内部状态、系统设置、用户管理、单位管理、渠道监控 | 全局治理 |
| **智能体** | 模型配置、线程管理、预设智能体、工具管理、系统提示词 | 能力配置 |
| **调试** | 吞吐测试、性能测试、蜂群测试、能力评估、调试面板 | 工程验证 |
| **文档** | 使用说明书、接口参考 | 查阅入口 |

## 系统分组

### 内部状态

查看系统整体运行状态：
- 活跃会话数
- 系统负载
- 关键组件状态

### 系统设置

配置全局参数：
- 服务地址和端口
- 安全策略
- 存储设置
- 工具开关

### 用户管理

管理用户账户：
- 用户列表查看
- 创建新用户
- 编辑用户信息
- 重置密码
- Token 账户管理（发放、扣除、设置余额）

### 单位管理

管理组织结构：
- 创建和管理单位
- 单位层级关系
- 成员管理

### 渠道监控

监控外部渠道：
- 渠道账号状态（在线/离线）
- 消息收发统计
- 异常告警
- 绑定关系管理

### 线程管理

### 预设智能体

### 系统提示词

- 切换默认模板包
- 创建新的模板包

## Token 账户管理

在用户管理中，管理员可以操作用户的 Token 账户：

| 操作 | 说明 |
|------|------|
| **直接设置余额** | 覆盖当前余额，适合修正账面值 |
| **发放 Token** | 增加余额，记录累计获得 |
| **扣除 Token** | 减少余额，记录累计消耗 |

管理员账户不参与 Token 余额限制。

## 常用管理操作

### 新用户加入

1. 用户管理 → 创建用户
2. 填写用户名、邮箱、密码
3. 分配所属单位
4. 根据需要发放初始 Token

### 配置新模型

1. 模型配置 → 添加模型
2. 填写模型名称、API 密钥、接口地址
3. 设置是否为默认模型

### 接入新渠道

1. 渠道监控 → 添加账号
2. 填写渠道类型和凭证
3. 配置 Webhook 地址
4. 测试消息收发

### 处理异常线程

1. 线程管理 → 找到异常线程
2. 查看详情了解问题
3. 如需终止，点击取消
4. 查看日志定位原因

## 管理端访问

管理端通过独立地址访问：
- 浏览器打开管理端 URL
- 使用管理员账户登录
- 具体地址由部署配置决定

## 安全注意

管理端拥有系统最高权限：
- 只有授权的管理员才能访问
- 管理员密码要足够复杂
- 定期更换密码
- 重要操作做好记录
