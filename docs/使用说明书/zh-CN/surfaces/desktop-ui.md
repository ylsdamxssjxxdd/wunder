# 桌面端界面

桌面端是个人用户的主要使用入口。它不只是网页的"壳"，而是本地优先的完整工作台。

## 桌面端特点

### 本地优先

- 数据存本地，隐私性强
- 不依赖服务器
- 开箱即用，无需配置

### 本机文件访问

可以直接访问本机的文件系统（取决于安全设置），比纯工作区更灵活。

### 本地运行时

安装包自带 Python 运行时，减少首次配置成本。

### 一键重置

异常时可以一键重置工作状态，快速恢复到干净状态。

## 界面结构

桌面端复用用户端的三栏布局，额外增加桌面特有的功能：

```
┌────────────┬──────────────────┬────────────────────────────┐
│   左栏     │      中栏        │          右栏              │
│   导航     │      列表        │         工作区             │
├────────────┼──────────────────┼────────────────────────────┤
│ • 聊天     │ 会话列表         │ 对话内容                   │
│ • 文件     │ 文件列表         │ 文件预览                   │
│ • 智能体   │ 智能体列表       │ 智能体设置                 │
│ • 工具     │ 工具列表         │ 工具详情                   │
│ • 设置     │ 设置分类         │ 设置内容                   │
│   └桌面设置│                  │ 桌面运行时配置             │
│   重置     │                  │ 一键重置工作状态           │
└────────────┴──────────────────┴────────────────────────────┘
```

## 桌面特有功能

### 桌面运行时设置

配置本地运行环境：
- Python 运行时路径
- 本地工作目录
- 本地工具配置

### 一键重置工作状态

当系统状态异常时使用：

**适用场景**：
- 会话卡在运行中
- 蜂群任务状态不一致
- 工作区有残留的运行态文件
- 界面状态和实际不一致

**重置后会**：
- 停止所有运行中的会话
- 清空排队任务
- 终止蜂群任务
- 为所有智能体重建干净的线程
- 清理工作态目录

**保留的内容**：
- 你的长期资产（技能、知识）
- 用户配置
- 工作区文件
- 会话历史记录

所以重置是"清理工作态"，不是"恢复出厂设置"。

### 本地文件能力

桌面端可以让智能体：
- 读取本机文件（在安全边界内）
- 写入本机文件
- 执行本地命令

这是比网页端更强的能力，但也要注意安全边界。

## 什么时候用桌面端

**适合**：
- 个人日常使用
- 需要处理本机文件
- 不想依赖服务器
- 注重隐私和数据本地化

**如果需要 Server 能力**：
- 多用户协作
- 管理员后台
- 组织级部署
- 渠道接入

请直接用浏览器访问 Server 的网页端，而不是在桌面端切换连接。

## 桌面端安装

1. 下载安装包
2. 运行安装程序
3. 启动应用
4. 开始使用

不需要额外配置数据库或服务器。

## 使用技巧

### 工作目录管理

- 工作目录默认持久化
- 定期清理不需要的文件
- 不要把临时文件放在工作目录

### 状态异常处理

遇到异常时：
1. 先等几秒看是否自动恢复
2. 如果不行，尝试重置工作状态
3. 如果问题持续，重启应用

### 安全边界

虽然是本地模式，智能体仍然受安全限制：
- 不能随意访问任意目录
- 命令执行受控制
- 在安全设置中可以调整

## 常见问题

### 数据在哪里？

- 工作区：本地工作目录
- 会话历史：本地 SQLite 数据库
- 配置：本地配置文件

### 能迁移到其他电脑吗？

可以：
- 复制工作目录
- 复制数据库文件
- 在新电脑上配置路径

### 本地模式能联网吗？

可以。本地模式只是数据存本地，不影响联网访问 AI 模型或网页。
