# 用户侧前端

Wunder 的用户侧前端不是单页聊天框，而是一套统一的消息工作台。

它把这些东西放进同一套 Messenger 壳里：

- 智能体聊天
- 我的概况与账号资料
- 蜂群协作
- 文件与工作区
- 工具与知识配置
- 系统设置与帮助文档

## 代码位置

- `frontend/`

## 当前主要入口

- `/app/chat`
- `/app/profile`
- `/app/workspace`
- `/app/tools`
- `/app/settings`
- `/app/channels`
- `/app/cron`

这些入口共享同一套外层框架，不是彼此独立的产品。

## 界面怎么组织

当前仍然是三栏结构：

- 左栏：一级导航
- 中栏：会话、智能体、蜂群、联系人或资源列表
- 右栏：真正的工作区

右栏既承载聊天记录，也承载工作流、资料页、设置面板和内嵌帮助手册。

## 登录、注册与密码

登录页现在有三条常用入口：

- 登录
- 注册账号
- 重置密码

`重置密码` 按钮位于 `注册账号` 的右侧。展开后，只需要这三项：

- 用户名
- 邮箱
- 新密码

如果你已经登录，可以在“我的概况 -> 编辑资料”里同时处理：

- 修改用户名
- 修改邮箱
- 修改登录密码

修改密码时，前端会要求：

- 输入当前密码
- 输入新密码
- 再确认一次新密码

如果只是改用户名或邮箱，密码项可以留空。

## 我的概况现在能看什么

“我的概况”已经并入 Messenger 设置面板，不再依赖旧页面单独渲染。

当前资料区会展示：

- 头像、用户名、用户 ID、所属单位
- 等级标签
- 经验进度条
- 会话数、近 7 天活跃、工具调用数、智能体数、累计 tokens
- 今日额度环状图

其中等级相关有两个关键变化：

- 等级标签已经替换掉头像区原来的“正式账号”文案
- 经验条放在资料卡片区域底部，而不是整个页面最底部

额度展示也改成了环状图，中心显示剩余额度，外圈按剩余/警戒/已用分段显示。

## 聊天区支持什么

聊天输入区仍然是一个上下文装配器，不是纯文本框。

- 文本输入：支持普通提问，也支持 `/new`、`/stop`、`/compact`、`/help`
- 文件上传：支持回形针选择和拖拽上传
- 图片：直接作为多模态上下文
- 文档：会先转成 Markdown 文本
- 音频：上传后会先转写
- 视频：默认按 `1 FPS` 抽帧，可手动重抽

只要附件还在处理中，发送会被阻止。这是保护机制，不是前端卡死。

## 新建线程为什么有时会被禁用

当前聊天页在智能体运行中会禁用 `新建线程` 按钮。

这是刻意的保护，不是故障。目的是避免：

- 运行中的主线程还没结束就切走
- 会话状态和工作流错位
- 同一个智能体在前台同时抢占主线程

如果你要开新线程，先做其中之一：

- 等当前运行自然完成
- 先停止当前会话，再新建线程

## 蜂群页面现在怎么看状态

蜂群页的中栏条目已经和消息列表对齐：

- 只要某个蜂群仍有运行中的 mission，列表项会出现呼吸高亮

右侧和画布里的工作流展示也有了更明确的分工：

- 母蜂节点优先展示真实工具轨迹
- 工蜂节点会持续跟随自己的工作流更新
- 子智能体节点完成后仍保留已走过的工具轨迹

也就是说，蜂群工作流区域现在关注的重点是“调用了什么工具、走到了哪一步”，而不是只堆状态、会话号或摘要。

## 系统设置里的危险操作

系统设置页现在提供“一键重置工作状态”入口。

它会统一处理当前用户的工作态：

- 中止运行中的会话
- 清空排队任务
- 终止蜂群运行
- 为默认智能体和各用户智能体重建新的主线程
- 清理工作态目录

这适合用在这些场景：

- 会话卡死
- 线程状态和前端显示明显错位
- 蜂群运行异常后想整体回到干净状态

## 帮助手册入口

用户侧前端已经内置文档入口，不需要离开当前工作台。

- 入口位置：左栏“更多” -> “帮助手册”
- 打开方式：右栏内嵌 `/docs/?embed=user`

## 什么时候优先看这页

适合这些场景：

- 你要改用户体验或页面结构
- 你要判断某个能力应该落在聊天区、资料区还是设置区
- 你要确认用户当前在前端到底能看到什么
