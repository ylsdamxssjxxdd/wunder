# Desktop 本地模式

Desktop 本地模式是 Wunder 当前最贴近真实个人使用的形态。

## 先不要把它理解成什么

不要把它理解成：

- 小号 server
- 只少了管理员后台的 server
- 临时试玩模式

它更准确的定位是：

- 本地优先运行形态
- 桌面工作台
- 端云协同的前半段

## 本地模式最核心的特点

### 工作目录是持久的

本地模式下，工作目录不是临时 sandbox。

它按持久目录处理，不再依赖“24 小时自动清理”这种策略。

### 仍然保留容器语义

虽然是本地模式，但仍然区分：

- 用户私有容器
- 智能体运行容器

只是这些容器会映射到真实本地目录。

### 优先使用附带运行时

当前 desktop 本地模式优先使用安装包附带的 Python 运行时，而不是让用户先配置一套解释器。

这能显著降低初次使用门槛。

## 本地模式为什么适合作为第一入口

因为它一次性验证了几条主链路：

- 用户侧界面
- 对话执行
- 工具调用
- 工作区文件
- 智能体设置

也就是说，它比单纯的 demo 更接近真实产品形态。

## 本地模式下文件能力有什么特殊点

当前一个非常重要的约定是：

- 本地模式下，内置文件工具已放开本机绝对路径访问

这意味着它不再强制把所有操作限制在工作区内部。

但这也意味着你要更清楚自己的本地目录边界。

## 什么时候该切远端

如果你只是个人使用，本地模式通常已经够用。

当你开始需要这些能力时，再考虑接远端 server：

- 多用户治理
- 管理员后台
- 单位/组织控制
- 统一渠道与服务接入

## 渠道能力在本地模式下会怎样

本地模式不会自动失去渠道能力。

当前还会默认开启：

- `channels.outbox.worker_enabled=true`

这保证了即使没有管理员侧前端，`channel_tool.send_message` 也能正常投递。

## 最适合本地模式的使用者

- 个人用户
- 需要桌面工作台的人
- 本地文件智能体循环很重的人
- 先本地跑通，再考虑接远端的人

## 你最容易踩到的坑

- 把本地模式当作无状态运行
- 忘了本地目录也区分容器
- 以为改系统提示词会回写当前老线程
- 把它和远端治理能力混成一个概念
