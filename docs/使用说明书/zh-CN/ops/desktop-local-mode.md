# Desktop 本地模式

Desktop 本地模式是 Wunder 当前最贴近真实个人使用的形态。

## 先不要把它理解成什么

不要把它理解成：

- 小号 server
- 只少了管理员后台的 server
- 临时试玩模式

它更准确的定位是：

- 本地优先运行形态
- 桌面工作台
- 端云协同的前半段

## 本地模式最核心的特点

### 工作目录是持久的

本地模式下，工作目录不是临时 sandbox。

- `WUNDER_WORK/` 按持久目录处理
- 不走 24 小时自动清理

### 仍然保留容器语义

虽然是本地模式，但仍区分：

- 用户私有容器
- 智能体运行容器

只是这些容器会映射到真实本地目录。

### 优先使用附带运行时

Desktop 本地模式优先使用安装包附带的 Python 运行时，减少首次配置成本。

## 本地模式为什么适合作为第一入口

因为它一次性覆盖了完整主链路：

- 用户侧界面
- 对话执行
- 工具调用
- 工作区文件
- 智能体设置
- 蜂群协作

## 本地模式下文件能力有什么特殊点

当前一个关键约定是：

- 本地模式下，内置文件工具可以访问本机绝对路径

这比纯工作区限制更灵活，但也意味着你要更清楚本地目录边界。

## 一键重置工作状态是什么

本地模式现在提供 `一键重置工作状态`。

它适合这些场景：

- 会话卡在运行中
- 蜂群任务状态明显不一致
- 工作区里留下半截运行态文件
- 你想让默认智能体和各自定义智能体回到干净主线程

执行后会：

- 中止当前用户仍在运行的会话
- 清空排队任务
- 终止当前蜂群任务
- 为默认智能体和全部用户智能体重建主线程
- 清理对应工作态目录

## 重置后会保留什么

这次重置针对的是“工作态”，不是“长期资产”。

会保留的典型内容包括：

- `global`
- `skills`
- `knowledge`
- 用户长期配置

所以它更像“把运行现场清干净”，不是“恢复出厂设置”。

## 什么时候该切远端

如果你只是个人使用，本地模式通常已经够用。

当你开始需要这些能力时，再考虑接远端 server：

- 多用户治理
- 管理员后台
- 单位/组织控制
- 统一渠道与服务接入

## 本地模式容易踩的坑

- 把本地模式当作无状态运行
- 忘了本地目录仍区分容器
- 以为修改提示词会回写当前老线程
- 以为重置工作状态会顺手删除长期技能或知识
