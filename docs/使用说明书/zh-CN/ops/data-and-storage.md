# 数据与存储

## 四类数据

Wunder 的数据分为四类，各有各的存储方式和生命周期：

| 类型 | 内容 | 存储 | 生命周期 |
|------|------|------|----------|
| **业务数据** | 用户、会话、配置 | 数据库 | 持久 |
| **文件数据** | 用户文件、智能体产物 | 工作区 | 持久 |
| **向量数据** | 知识库检索 | 向量库 | 持久 |
| **临时数据** | 上传下载中转 | 临时目录 | 短期 |

**核心原则**：分清什么该持久化，什么只是临时中转。

## 数据库选择

### SQLite vs PostgreSQL

| | SQLite | PostgreSQL |
|--|--------|------------|
| **适合** | Desktop 本地模式 | Server 多用户模式 |
| **特点** | 轻量、无额外安装 | 专业级、支持高并发 |
| **部署** | 自动创建 | 需要单独部署 |

**选择建议**：
- 个人使用（Desktop）→ SQLite
- 团队使用（Server）→ PostgreSQL

这不是性能差异的问题，而是运行形态决定的。

### 数据库存什么

数据库存储所有业务核心数据：
- 用户账户和组织信息
- 会话记录和消息历史
- 智能体配置
- 渠道绑定和运行日志
- 系统配置

### 备份策略

- **PostgreSQL**：定期全量备份 + 日志增量
- **SQLite**：定期复制数据库文件
- **重要**：备份要验证可恢复性

## 工作区存储

### 工作区存什么

- 用户上传的文件
- 智能体生成的文件
- 技能和知识文件
- 智能体配置文件

### 存储要点

- **Server 模式**：工作区必须持久化（使用 NAS 或网络存储）
- **Desktop 模式**：工作区在本地目录，自动持久化
- **不要**把工作区放在临时目录

### 空间管理

- 定期检查工作区空间使用
- 不需要的文件及时清理
- 大文件考虑归档策略

## 向量知识库

### 作用

向量库用于知识库的语义检索功能，让智能体能快速找到相关知识内容。

### 注意

- 向量库**不是**主数据库的替代品
- 它只承担检索功能
- 备份时应与主数据库一起备份

## 临时目录

### 用途

临时目录是文件传输的中转区：
- 文件上传时的临时存放
- 文件下载时的转发
- 文档转换过程中的中间文件

### 注意

- **不要**在临时目录存放长期文件
- 临时文件会被定期清理
- 生产环境应配置自动清理策略

## 数据迁移

### 从 SQLite 迁移到 PostgreSQL

当团队规模增长需要从本地模式迁移到服务器模式时：
1. 导出 SQLite 数据
2. 配置 PostgreSQL 连接
3. 导入数据到 PostgreSQL
4. 迁移工作区文件
5. 验证数据完整性

### 工作区迁移

迁移工作区文件时：
1. 复制整个工作区目录
2. 保持目录结构不变
3. 更新配置中的路径指向
4. 验证文件可访问

## 常见误区

| 误区 | 正确理解 |
|------|----------|
| SQLite 和 PostgreSQL 只是性能不同 | 它们适合不同的运行形态 |
| 临时目录也能存长期文件 | 临时目录会被清理 |
| 向量库存所有数据 | 向量库只存检索数据 |
| 工作区放在 config 目录 | config 是运行时目录，不是数据目录 |
