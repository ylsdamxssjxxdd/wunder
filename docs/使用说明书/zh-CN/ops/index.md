# 运维概览

Wunder 的运维可以理解为**五条主线**：部署、存储、安全、渠道、监控。

目标不是"启动成功"，而是**让系统长期稳定运行**。

## 五条主线

### 1. 部署与运行

**目标**：正确选择运行形态，顺利启动，健康检查通过。

**关键决策**：
- 个人使用还是团队使用？
- 用 PostgreSQL 还是 SQLite？
- 工作区放在哪里？

### 2. 数据与存储

**目标**：不丢数据、快速恢复、存储性能足够。

**存储分层**：

| 层数据 | 用途 | 备份建议 |
|--------|------|----------|
| 数据库 | 用户、会话、配置 | 定期全量备份 |
| 工作区 | 用户文件和产物 | 文件系统备份 |
| 向量库 | 知识库检索 | 随数据库备份 |
| 临时目录 | 上传下载中转 | 定期清理，不需要备份 |

### 3. 账户与安全

**目标**：权限清晰、访问受控、审计可追溯。

**安全要点**：
- API 密钥足够复杂
- 路径和命令访问受限
- 审批模式按需配置

### 4. 渠道管理

**目标**：消息不丢、连接稳定、异常可发现。

**监控要点**：
- 账号是否在线
- 消息收发是否正常
- 出站是否卡在队列

### 5. 监控与评估

**目标**：提前发现问题、量化系统表现。

**观测重点**：
- 活跃会话数和队列深度
- 错误率和响应延迟
- Token 消耗趋势

## 排障优先级

系统出问题时，按这个顺序排查：

1. **入口是否可达**——能打开页面吗？能连接吗？
2. **账户权限是否正常**——登录正常吗？Token 过期了吗？
3. **依赖服务是否正常**——数据库正常吗？MCP 服务通吗？
4. **运行状态是否健康**——会话卡住了吗？队列积压了吗？
5. **指标是否异常**——错误率突增？延迟变大？

## 生产环境检查清单

### 必做项

- [ ] 使用 PostgreSQL（不要用 SQLite）
- [ ] 数据库自动备份已配置
- [ ] 工作区存储持久化（NAS/网络存储）
- [ ] HTTPS 证书配置
- [ ] API 密钥复杂度足够

### 推荐项

- [ ] CORS 策略已收紧
- [ ] 日志收集已配置
- [ ] 监控告警已设置
- [ ] 定期清理临时目录

## 按场景的运维建议

### 个人 Desktop 使用

基本不需要运维。关注：
- 定期更新应用
- 工作区文件别太大
- 异常时用重置功能

### 小团队 Server

关注：
- 数据库备份
- 用户管理和 Token 发放
- 基本监控

### 组织生产环境

需要全面运维：
- 完整的备份策略
- 监控告警体系
- 安全审计
- 渠道管理
- 性能调优

## 常见运维误区

| 误区 | 正确理解 |
|------|----------|
| 进程活着 = 服务可用 | 还要检查入口和依赖健康 |
| 渠道问题 = 模型问题 | 先查接入层，再查模型 |
| 临时目录可以存长期数据 | 临时目录会被清理 |
| 部署完就没事了 | 要监控、要备份、要维护 |
