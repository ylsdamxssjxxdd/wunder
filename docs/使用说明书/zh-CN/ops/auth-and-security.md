# 账户与安全

## 账户体系

Wunder 有多种身份类型，各有各的用途：

### 管理员账户

- 拥有系统治理权限
- 可以管理用户、配置系统
- 不受 Token 余额限制
- 负责安全策略的配置

### 普通用户账户

- 日常使用系统的用户
- 受 Token 余额限制
- 只能访问自己的数据
- 可以创建和管理自己的智能体

### 外部身份

从飞书、企业微信等外部平台映射过来的用户身份。不一定是注册用户，但仍然有独立的隔离空间。

## 安全分层

安全不是单一措施，而是分层防护：

```
┌─────────────────────────────────────┐
│   操作安全：工具审批、权限控制       │
├─────────────────────────────────────┤
│   接口安全：鉴权、限流、CORS        │
├─────────────────────────────────────┤
│   传输安全：HTTPS、加密通信          │
├─────────────────────────────────────┤
│   网络安全：网络隔离、容器安全       │
└─────────────────────────────────────┘
```

## 工具执行安全

### 路径控制

管理员可以配置：
- **允许的路径**：智能体可以操作的目录范围
- **禁止的模式**：禁止访问的文件类型或路径

**安全建议**：
- 不要无限制开放整个文件系统
- 明确指定允许访问的目录
- 禁止访问系统敏感目录

### 命令控制

- 控制智能体可以执行哪些命令
- 高风险命令需要审批
- 管理员可以设置白名单或黑名单

### 审批机制

敏感操作需要用户确认：
- 修改重要文件
- 执行敏感命令
- 访问受限路径

审批模式的严格程度可以配置。

## 数据隔离

### 用户级隔离

- 工作区：每个用户只能访问自己的文件
- 会话：每个用户只能看到自己的对话
- 配置：个人设置不影响他人

### 容器级隔离

同一用户内，个人文件和智能体文件也是分开的。

### 数据库隔离

所有数据访问都经过权限检查，不会出现跨用户数据泄露。

## 安全配置建议

### API 密钥

- 使用足够复杂的密钥（超过 16 位，混合字符）
- 定期更换密钥
- 不要在代码或配置中硬编码
- 不要把密钥提交到版本控制

### 传输安全

- 生产环境必须使用 HTTPS
- 配置 TLS 证书并自动续期
- WebSocket 使用 wss 协议

### CORS 策略

- 只允许必要的域名
- 不要使用通配符 `*`
- 按环境区分（开发/生产）

### 日志审计

- 记录关键操作（登录、配置变更、文件操作）
- 日志保留足够时间
- 敏感信息脱敏

## 常见安全问题与处理

### 密钥泄露

1. 立即更换密钥
2. 检查是否有异常访问
3. 审查日志确认影响范围

### 越权访问

1. 检查权限配置
2. 确认数据隔离正常
3. 必要时重置用户状态

### 暴力破解

1. 配置登录限流
2. 设置账户锁定策略
3. 监控异常登录尝试
