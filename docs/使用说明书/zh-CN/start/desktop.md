# Desktop 入门

如果你想**马上用起来**，不要先去搭 server，直接用 `wunder-desktop`。

它是面向个人用户和本地优先场景的主入口——打开就是一个完整的智能体工作台。

---

## 什么时候选 Desktop？

| 场景 | 选 Desktop |
|------|-----------|
| 你想直接开始用，不想先部署服务 | ✅ |
| 你需要操作本地文件、窗口、浏览器 | ✅ |
| 你更习惯图形界面，不想从命令行开始 | ✅ |
| 你后面可能会接远端，但今天先想跑起来 | ✅ |
| 你需要多用户治理和统一管理 | ❌ 选 Server |

---

## Desktop 是什么？

Desktop 不是「简化版 Server」，它是一个**本地优先的独立运行形态**：

```
┌─────────────────────────────────────┐
│     wunder-desktop                  │
│  ┌───────────────────────────────┐ │
│  │  用户侧前端（Vue3）           │ │
│  │  - 聊天界面                   │ │
│  │  - 智能体管理                 │ │
│  │  - 工作区文件                 │ │
│  │  - 系统设置                   │ │
│  └───────────────────────────────┘ │
│  ↓ 本地桥接                         │
│  ┌───────────────────────────────┐ │
│  │  Rust 核心引擎                │ │
│  │  - 调度器                     │ │
│  │  - 工具链                     │ │
│  │  - 本地工作区                 │ │
│  └───────────────────────────────┘ │
└─────────────────────────────────────┘
```

### 你会看到什么

Desktop 打开后，是一个统一的三栏消息工作台：

| 区域 | 功能 |
|------|------|
| **左侧** | 导航栏：聊天、智能体、用户世界、工作区、工具、设置 |
| **中间** | 列表视图：会话列表、智能体列表、联系人、文件等 |
| **右侧** | 主区域：聊天界面、智能体循环、设置面板、扩展功能 |

### 它不只是聊天

Desktop 承载的是一整套工作台：

- ✅ 用户与智能体会话
- ✅ 工作区与文件管理
- ✅ 桌面能力和本地目录映射
- ✅ 本地模式与远端接入切换
- ✅ 蜂群协作与多智能体
- ✅ 渠道接入（飞书、微信、QQ、XMPP 等）

## 5 步上手 Desktop

### 1. 下载安装

- 去项目 Releases 页面下载匹配你系统的安装包
- Windows: `.exe` 或 `.msi`
- macOS: `.dmg`
- Linux: `.AppImage` 或 `.deb`

### 2. 首次启动

启动后，系统会自动：
- 创建本地工作目录 `WUNDER_WORK/`
- 创建配置目录 `WUNDER_TEMPD/`
- 同步内置技能到本地

### 3. 配置模型

打开「系统设置」→「模型配置」：

```
必填项：
- API Key: 你的大模型 API Key
- Endpoint: API 端点地址
- Model Name: 模型名称（如 gpt-4o）

可选：
- 工具调用模式：tool_call / function_call / freeform_call
- 最大轮次：防止无限循环
```

点击「测试连接」确认可用后保存。

### 4. 发起第一次对话

回到聊天界面，输入：

```
帮我列出当前工作目录的文件
```

你会看到：
- 模型思考
- 调用「列出文件」工具
- 展示文件列表
- 给出最终回复

### 5. 探索更多能力

试试这些：

| 任务 | 输入示例 |
|------|----------|
| 写代码 | `写一个 Python 脚本，读取 CSV 并生成图表` |
| 编辑文件 | `打开 main.py，在第 10 行后添加日志` |
| 浏览网页 | `打开 https://example.com 并提取正文` |
| 多智能体 | `用蜂群协作：一个查资料，一个写稿，一个复核` |

## 聊天输入区常用操作

Desktop 的聊天输入区可以直接装配上下文，不只是打一段字。

- 回形针上传：支持图片、文档、音频、视频。
- 拖拽上传：把文件直接拖进输入框区域即可。
- 图片：直接作为视觉上下文发送。
- 文档：会先解析成 Markdown，再作为文本上下文发送。
- 音频：上传后会先转写，再进入模型上下文。
- 视频：上传后会先拆成图片序列和音频附件，默认按 `1 FPS` 抽帧。
- 视频抽帧控制：点击上传预览列表中的视频条目，可以展开 FPS 输入框并重新抽帧。允许范围是 `0.1 ~ 12 FPS`。如果视频很长，系统可能会自动降低实际 FPS。
- 处理中保护：只要附件还没处理完，发送会暂时被阻止。

## 本地模式下的额外输入能力

如果当前是 desktop 本地模式，聊天区通常还会看到额外按钮：

- 麦克风：直接录音并发送语音附件。
- 截图：支持全屏截图、区域截图，以及是否隐藏当前窗口。
- 这些按钮是否可用，取决于当前机器权限和桌面桥接能力，不是所有运行形态都一定有。

## 不会用时，直接打开帮助手册

Desktop 里已经内置文档入口：

- 左栏进入“更多”
- 中栏选择“帮助手册”
- 右栏会直接打开 wunder 文档站

这适合在使用过程中边操作边查，不需要离开当前桌面工作台。

## 本地模式的关键约定

### 工作区持久化

- 本地工作目录 `WUNDER_WORK/` 按**持久目录**处理
- **不会**被 24 小时自动清理
- 各容器目录可独立映射到本地

### 容器分层

| container_id | 用途 | 说明 |
|--------------|------|------|
| 0 | 用户私有文件 | `global/`、`skills/`、`agents/<agent_id>/` |
| 1~10 | 智能体运行空间 | 每个智能体可独立配置映射 |

### 智能体内见系统

用户私有容器（container_id=0）同时作为「智能体内见系统」根目录：

```
WUNDER_WORK/
├── global/
│   └── tooling.json          # 用户级工具配置
├── skills/
│   ├── builtin/              # 内置技能（只读）
│   └── custom/               # 用户自定义技能
└── agents/
    └── <agent_id>/
        ├── worker-card.json   # 智能体配置
        └── system_prompt.md  # 系统提示词
```

智能体可以直接读写这些配置文件！

## Desktop 特有能力

### 桌面控制（仅本地模式）

- **浏览器自动化**：打开网页、点击、输入、截图
- **桌面控制器**：操作窗口、截图、监视变化
- **读图工具**：视觉模型可分析本地图片

### 端云协同

Desktop 不仅能本地运行，还能：

- 接入远端 Gateway
- 使用远端 Server 的能力
- 在本地和远端之间切换

### 在线更新

Tauri 版本支持：
- 自动检测更新
- 一键安装更新
- 签名验证

## 常见问题

**Q: Desktop 需要联网吗？**  
A: 模型调用需要联网，但工具执行可以纯本地。

**Q: 我的文件安全吗？**  
A: 本地模式下，文件只在你电脑上，不会上传到服务器。

**Q: 视频为什么不是直接发给模型？**  
A: 当前聊天输入区会先把视频拆成图片序列和音频相关附件，再作为上下文发送。这样更稳定，也允许你在发送前手动调整抽帧频率。

**Q: 能和 Server 配合用吗？**  
A: 可以！Desktop 可以接入远端 Gateway，享受端云协同。

**Q: 内置技能会被覆盖吗？**  
A: 内置技能是只读的，重装时会自动更新到最新版。
