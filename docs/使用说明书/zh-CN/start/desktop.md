# Desktop 入门

如果你想马上用起来，不要先搭 server，直接用 `wunder-desktop`。

它面向个人用户和本地优先场景，打开就是一套完整的智能体工作台。

## 什么时候选 Desktop

| 场景 | 选 Desktop |
|------|-----------|
| 你想直接开始用，不想先部署服务 | ✅ |
| 你需要本地文件、窗口、浏览器能力 | ✅ |
| 你更习惯图形界面 | ✅ |
| 你后面可能接远端，但现在先跑起来 | ✅ |
| 你需要多用户治理和管理员后台 | ❌ 选 Server |

## Desktop 是什么

Desktop 不是“简化版 Server”，它是本地优先的独立运行形态：

```text
wunder-desktop
  ├─ 用户侧前端（聊天、资料、蜂群、设置）
  ├─ 本地桥接（截图、录音、桌面环境能力）
  └─ Rust 核心引擎（调度、工具、工作区）
```

## 你打开后会看到什么

Desktop 仍然是统一三栏：

- 左侧：导航
- 中间：会话、智能体、蜂群和资源列表
- 右侧：聊天区、工作流、资料页、设置页

它不只是聊天，还包含：

- 用户与智能体会话
- 工作区与文件管理
- 蜂群协作与多智能体
- 系统设置与本地运行偏好
- 我的概况与账号资料

## 5 步上手

### 1. 下载安装

- 从 Releases 下载对应系统安装包

### 2. 首次启动

启动后会自动：

- 创建本地工作目录 `WUNDER_WORK/`
- 创建配置目录 `WUNDER_TEMPD/`
- 准备内置技能和运行资源

### 3. 配置模型

进入“系统设置”配置：

- API Key
- Endpoint
- Model Name

保存前建议先点一次“测试连接”。

### 4. 处理账号资料

首次用自己的账号时，建议顺手完成这几件事：

- 在登录页确认是否需要注册或重置密码
- 进入“我的概况 -> 编辑资料”修改用户名或邮箱
- 如果这是长期使用账号，顺便设置新的登录密码

登录页的重置密码只需要：

- 用户名
- 邮箱
- 新密码

已登录后的修改密码则需要当前密码校验。

### 5. 发起第一次对话

回到聊天页，输入：

```text
帮我列出当前工作目录的文件
```

你会看到：

- 模型开始执行
- 工具工作流逐步出现
- 最终回复落到聊天区

## 两个你很快会遇到的交互约束

### 运行中不能直接新建线程

当当前智能体还在运行时，`新建线程` 按钮会被禁用。

这是为了避免运行中的主线程和新线程抢状态。正确做法是：

- 等它完成
- 或者先停止当前会话，再新建线程

### 蜂群运行会有持续高亮

如果某个蜂群还在活动中：

- 中栏的蜂群条目会持续呼吸高亮
- 画布上的母蜂、工蜂、子智能体节点会保留工作流轨迹

## 系统设置里的恢复入口

如果你把会话、蜂群或工作区跑乱了，可以用“系统设置”里的 `一键重置工作状态`。

它会：

- 中止当前运行中的会话和任务
- 终止蜂群运行
- 为默认智能体和各用户智能体重建主线程
- 清理工作态目录

它不会清空用户长期资产，例如：

- `global`
- `skills`
- `knowledge`

## 本地模式下的额外输入能力

如果当前是 Desktop 本地模式，聊天区通常还会看到：

- 麦克风按钮
- 截图按钮

这些按钮是否可用，取决于当前机器权限和桌面桥接能力。

## 不会用时，直接打开帮助手册

Desktop 已经内置文档入口：

- 左栏进入“更多”
- 中栏选择“帮助手册”
- 右栏直接打开文档站
