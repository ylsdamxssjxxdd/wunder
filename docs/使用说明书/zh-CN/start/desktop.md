# Desktop 入门

如果你想马上用起来，不要先搭 server，直接用 Desktop。

打开就是一套完整的智能体工作台，不需要服务器。

## 什么时候选 Desktop

| 场景 | 选 Desktop |
|------|-----------|
| 想直接开始用，不想先部署服务 | ✅ |
| 需要操作本地文件、窗口、浏览器 | ✅ |
| 更习惯图形界面 | ✅ |
| 后面可能接远端，但现在先跑起来 | ✅ |
| 需要多用户治理和管理员后台 | ❌ 选 Server |

## 你打开后会看到什么

Desktop 是统一三栏布局：

- **左栏**：导航（聊天、文件、智能体、工具、设置、帮助）
- **中栏**：列表（会话、文件、智能体列表等）
- **右栏**：工作区（聊天、文件预览、设置面板等）

## 5 步上手

### 1. 下载安装

从 Releases 下载对应系统的安装包，安装后启动。

### 2. 首次启动

启动后会自动创建本地工作目录和配置目录，不需要手动操作。

### 3. 配置模型

进入「系统设置」→「模型配置」，填入：

- API Key
- 接口地址
- 模型名称

保存前建议先点「测试连接」确认能用。

### 4. 完善账号资料

首次登录后建议：

- 去「我的概况 → 编辑资料」修改用户名或邮箱
- 如果是长期使用，顺便设置新的登录密码

### 5. 发起第一次对话

回到聊天页，输入：

```
帮我列出当前工作目录的文件
```

你会看到模型开始执行 → 工具逐步工作 → 最终回复落到聊天区。

## 两个常见交互约束

### 运行中不能新建线程

当前智能体还在运行时，「新建线程」按钮会被禁用。这是正常保护，等它完成或先停止当前会话再新建。

### 蜂群运行时会有高亮提示

如果某个蜂群还在活动中，中栏的蜂群条目会持续呼吸高亮，这是在提醒你它还在工作。

## 状态乱了怎么办？

如果你把会话、蜂群或工作区跑乱了，用「系统设置」里的**一键重置工作状态**。

它会清理运行中的状态，但不会删除你的技能、知识库等长期资产。

## 本地模式额外能力

Desktop 本地模式下，聊天区还会看到：

- 🎤 麦克风按钮（语音输入）
- 📷 截图按钮（截屏分析）

是否可用取决于当前机器权限。

## 不会用时，直接打开帮助手册

左栏 → 更多 → 帮助手册，可以直接在应用内查阅文档。
