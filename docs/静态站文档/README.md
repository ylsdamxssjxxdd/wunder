# 静态站文档目录说明

本目录用于存放 wunder 正式静态文档站的内容源文件。

## 目录约定

- 正式文档统一放在 `docs/静态站文档/`
- 中文文档放在 `docs/静态站文档/zh-CN/`
- 后续如需英文文档，放在 `docs/静态站文档/en-US/`
- `docs/方案/` 继续作为内部方案与 RFC，不直接作为静态站主内容
- `docs/wunder补全文档落地方案.md` 作为实施方案文档保留在 `docs/` 根目录

## 页面风格

静态站文档的布局、内容形式、说法方式尽量参考 OpenClaw 文档工程：

- 目录分层清晰，按 `start / concepts / tools / integration / help / reference` 组织
- 单篇页面只解决一个核心问题，避免写成超长总设计文档
- 页面开头先给出简短 summary，再进入正文
- 多使用“适用场景 / 操作步骤 / 相关文档”这种直读结构
- FAQ、troubleshooting、hub 页面单独组织，不把所有说明堆到首页

## 不建议放入本目录的内容

- 临时讨论稿
- 一次性实施清单
- 对标分析原稿
- 演示脚本与视频提示词
- 与静态站无关的内部备忘
