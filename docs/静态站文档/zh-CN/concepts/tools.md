# 工具体系

wunder 的核心不是“模型回答”，而是“模型调度工具完成任务”。

## 工具来源

当前工具来源至少包括：

- 内置工具
- MCP 工具
- Skills
- 知识库工具
- 用户工具

它们在能力来源上不同，但在调度层会被统一编排。

## 内置工具

内置工具用于最基础、最稳定的系统能力，例如：

- 读取文件
- 写入文件
- 应用补丁
- 搜索内容
- 列出文件
- 执行命令
- 线程控制
- 睡眠
- 图像读取

内置工具是 wunder 最基础的执行面。

## MCP

MCP 用于接入外部能力或额外服务。

它的作用不是替代内置工具，而是把 wunder 之外的系统能力纳入统一调度。

## Skills

Skill 更像流程化经验包。

它可以把：

- 一套方法
- 一组步骤
- 一批约束
- 某类任务最佳实践

封装成模型可调用的能力块。

## 知识库

知识库提供的是“受控上下文来源”，不是直接替代模型记忆。

它更适合：

- 固定制度
- 长文档
- 领域资料
- 本地知识沉淀

## 用户工具

用户工具允许不同用户按自己的工作场景扩展能力，但依然在系统治理边界内运行。

## 为什么要统一工具体系

如果不同来源的工具各自为政，会出现几个问题：

- 模型不知道该用哪类能力
- 前端无法统一展示
- 权限和治理无法统一
- 监控和排障会被打散

wunder 的做法是：来源不同，但调度和展示尽量统一。
