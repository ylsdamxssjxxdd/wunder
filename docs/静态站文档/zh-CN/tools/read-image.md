# 读图工具

`读图工具` 解决的是“把本地图片交给模型看”，不是“把图片文件当文本读出来”。

## 常用参数

- `path`
- `prompt`

其中：

- `path` 指向工作区或允许读取根目录里的图片文件。
- `prompt` 是附带给多模态模型的说明文字，可选。

## 它会做什么

在真正发给模型前，系统会先做几件事：

- 解析并校验路径
- 校验文件存在且是文件
- 校验大小上限
- 识别图片 MIME 类型
- 生成后续消息里的 `image_url`

当前实现会限制图片大小，避免把超大二进制直接塞进上下文。

## 适合哪些格式

当前内置检测会覆盖常见图片格式，例如：

- PNG
- JPEG
- GIF
- WebP
- BMP
- TIFF
- AVIF

## 和 `读取文件` 的区别

- `读取文件` 面向纯文本文件。
- `读图工具` 面向图片与多模态输入。

如果你把二进制图片路径交给 `读取文件`，通常不是正确路径。

## 什么时候该用它

- 让模型解释截图
- 让模型分析设计图或错误弹窗
- 让模型看本地生成的图片产物

这类场景都比直接贴文字描述更稳定。

## 你最需要记住的点

- `读图工具` 的核心不是返回像素内容，而是构造后续多模态消息。
- 它只适合图片，不适合一般二进制文件。
- 真正的文本文件仍然应该走 [文件与工作区工具](/docs/zh-CN/tools/workspace-files/)。
