# 技能调用

`技能调用` 的核心价值，是让技能从“一个名称”变成“模型可主动读取的操作手册”。

## 它解决什么

传入技能名后，工具会返回：

- 完整 `SKILL.md`
- 技能目录结构

所以模型不必反复猜某个技能到底怎么用。

## 常用参数

- `name`

它本身非常简单，但结果很重要。

## `{{SKILL_ROOT}}` 为什么重要

技能文档里建议用：

- `{{SKILL_ROOT}}`

来引用脚本、示例和工作流文件。

当 `技能调用` 返回正文时，这个占位符会被替换成当前可见的真实技能根目录路径。

这能避免：

- server、desktop、cli 下路径不一致
- 技能迁移后文档路径失效

## 为什么这比只在 prompt 里写技能名更好

因为很多复杂技能真正重要的是：

- 细节步骤
- 资源路径
- 约束条件

如果只把技能名丢给模型，模型很容易误用。

## 实施建议

- `技能调用` 是读取技能正文的正式工具入口。
- `{{SKILL_ROOT}}` 会在返回时被替换成真实路径。
- 技能不只是元信息，应该被看作模型可读取的执行手册。
