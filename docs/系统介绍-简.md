# wunder 系统介绍（极简）

## 简介
wunder 是一个以智能体为核心的调度平台：把大模型、工具、知识和工作区统一成可编排能力，通过 Rust（Axum）对外提供 `/wunder` 入口，支持流式返回中间过程与最终结果。

## 核心理念
- 对开发者：一切皆接口（模型、工具、知识、渠道、权限都可标准化接入）。
- 对大模型：一切皆工具（模型通过统一工具协议驱动系统能力）。
- 对组织：一切皆资产（会话、工具、知识、审计与配额可治理、可追踪）。

## 智能体核心能力（重点）
- 智能体应用化：每个智能体可独立配置系统提示词、工具集合、权限级别、共享状态与沙盒容器，支持按场景快速创建/复制/发布。
- 智能体线程调度：以 `user_id + agent_id` 维度维护主会话，支持忙时排队、会话锁与断点恢复，保障并发下稳定执行。
- 智能体动作循环：单轮用户输入可触发多轮模型动作（模型调用/工具调用/最终回复），并完整记录用户轮次、模型轮次与上下文 token 占用。
- 智能体协作执行：内置子智能体控制与蜂群（Swarm）机制，可将复杂任务拆成 TeamRun/TeamTask 并行协作后汇总。
- 智能体可扩展工具脑：内置工具链 + MCP + Skills + A2A + 知识库 + 用户自建/共享工具统一编排，按策略动态启停。
- 智能体长期运行能力：支持上下文压缩、长期记忆、工作区持久化与监控审计，面向长周期稳定运行。

## 核心功能
- 统一通信：WebSocket 优先，SSE 兜底；支持流式事件、取消与恢复。
- 多端体系：用户侧前端（Vue3）用于应用与会话；管理侧前端（web）用于配置、治理与运维。
- 治理运维：用户/组织/权限/配额管理，监控、压测、评估、性能采样与安全策略一体化。
- 开放接入：支持外部渠道接入与自托管 MCP（`/wunder/mcp`），便于跨系统集成。

## 对外宣传版（100字以内）
wunder：自托管智能体调度平台，统一入口编排模型/工具/知识，支持流式过程、多智能体协作、工作区沉淀与可治理运维。
