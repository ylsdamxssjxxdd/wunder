# 智能体实时分布式系统事件字典

## 1. 文档目标

- 固定智能体实时分布式系统中的事件命名、分类、优先级、最小字段。
- 统一当前 `chat / beeroom / user_world / mission` 多条实时链路的事件语义。
- 为后续 `thread / mission / projection_room` 三类 target 的统一协议做事件标准。

## 3. 统一事件模型

建议未来所有实时事件统一分成五大类：

| 类别 | 含义 | 典型 target |
| --- | --- | --- |
| Ingress Event | 消息或命令进入系统 | thread / mission |
| Runtime Event | 智能体或任务执行状态变化 | thread / mission |
| Projection Event | beeroom / user_world / session 的共享投影变化 | projection_room |
| Control Event | 握手、watch、ack、cancel、resync | thread / mission / projection_room |
| Telemetry Event | 使用量、上下文、性能、辅助观测 | thread / mission |

## 4. 统一 envelope 字段

所有标准事件建议至少包含以下字段：

```json
{
  "event_type": "string",
  "target_kind": "thread | mission | projection_room",
  "target_id": "string",
  "epoch": 1,
  "seq": 1001,
  "created_at": 1710000000.123,
  "priority": "P0 | P1 | P2 | P3 | P4",
  "payload": {}
}
```

建议附加字段：

- `source_kind`
  - `user | channel | mission | system | tool | agent`
- `source_id`
  - agent_id / user_id / tool_name / route_id
- `thread_id`
- `mission_id`
- `projection_realm`
- `public`
  - 是否属于公开投影事件

## 5. 标准事件字典

## 5.1 Ingress 事件

### `message_accepted`

- 含义：任意来源的消息已被 wunder 接管
- 目标：`thread`
- 优先级：`P1`
- 生产者：ingress router / thread runtime
- 最小 payload：

```json
{
  "logical_message_id": "msg_xxx",
  "source_kind": "user | channel | mission",
  "session_id": "sess_xxx",
  "thread_id": "thread_xxx",
  "content_type": "text",
  "preview": "..."
}
```

### `command_accepted`

- 含义：非消息型命令已被系统接管
- 目标：`thread | mission`
- 优先级：`P1`
- payload：

```json
{
  "command_type": "cancel | retry | start_mission | mark_read | control",
  "request_id": "req_xxx"
}
```

## 5.2 Thread Runtime 事件

### `thread_running`

- 含义：线程已开始推进
- 目标：`thread`
- 优先级：`P1`
- 当前兼容映射：`thread_status(status=running)`、部分 `progress`

### `thread_waiting`

- 含义：线程进入等待态
- 目标：`thread`
- 优先级：`P1`
- waiting reason 建议枚举：
  - `waiting_tool`
  - `waiting_approval`
  - `waiting_user`
  - `queued`

### `thread_output_delta`

- 含义：线程文本或 reasoning 的增量输出
- 目标：`thread`
- 优先级：`P2`
- 当前兼容映射：`llm_output_delta`
- payload：

```json
{
  "delta": "...",
  "reasoning_delta": "...",
  "model_round": 3,
  "user_round": 2
}
```

### `thread_output_final`

- 含义：线程本轮最终结构化输出
- 目标：`thread`
- 优先级：`P1`
- 当前兼容映射：`llm_output`、`final`

### `thread_failed`

- 含义：线程终止且失败
- 目标：`thread`
- 优先级：`P0`
- 当前兼容映射：`error`、`thread_closed(reason=failed)`、部分 `turn_terminal`

### `thread_completed`

- 含义：线程本轮正常完成
- 目标：`thread`
- 优先级：`P1`
- 当前兼容映射：`final`、`turn_terminal`

### `thread_closed`

- 含义：thread runtime 被关闭或 owner 失效
- 目标：`thread`
- 优先级：`P0`
- 当前兼容映射：`thread_closed`

## 5.3 Tool 事件

### `tool_started`

- 含义：工具开始执行
- 目标：`thread`
- 优先级：`P1`
- 当前兼容映射：`tool_call`
- payload：

```json
{
  "tool_name": "shell_command",
  "call_id": "call_xxx",
  "approval_required": false
}
```

### `tool_waiting_approval`

- 含义：工具执行因审批等待而阻塞
- 目标：`thread`
- 优先级：`P1`
- 当前兼容映射：`approval_request`

### `tool_finished`

- 含义：工具成功完成
- 目标：`thread`
- 优先级：`P0`
- 当前兼容映射：`tool_result`

### `tool_failed`

- 含义：工具执行失败
- 目标：`thread`
- 优先级：`P0`
- 当前兼容映射：`tool_result(ok=false)`、部分 `error`

## 5.4 Mission 事件

### `mission_started`

- 含义：mission 实例已启动
- 目标：`mission`
- 优先级：`P1`
- 当前兼容映射：`team_start`

### `task_claimed`

- 含义：某 agent 已领取 task
- 目标：`mission`
- 优先级：`P1`
- 当前兼容映射：`team_task_dispatch`

### `task_running`

- 含义：task 已进入执行态
- 目标：`mission`
- 优先级：`P1`
- 当前兼容映射：`team_task_update(status=running)`

### `task_progress`

- 含义：task 执行过程中公开进展
- 目标：`mission`
- 优先级：`P2`
- 当前兼容映射：`team_task_update`

### `task_result`

- 含义：task 产出公开结果
- 目标：`mission`
- 优先级：`P1`
- 当前兼容映射：`team_task_result`

### `task_failed`

- 含义：task 失败
- 目标：`mission`
- 优先级：`P0`
- 当前兼容映射：`team_task_result(status=failed)`、`team_error`

### `mission_merging`

- 含义：mission 正在汇聚多个 task 结果
- 目标：`mission`
- 优先级：`P1`
- 当前兼容映射：`team_merge`

### `mission_finalized`

- 含义：mission 已进入终态
- 目标：`mission`
- 优先级：`P0`
- 当前兼容映射：`team_finish`、`team_error`

## 5.5 Projection 事件

### `projection_summary_updated`

- 含义：共享摘要发生变化
- 目标：`projection_room`
- 优先级：`P3`
- 适用：
  - beeroom group summary
  - conversation summary
  - session summary

### `projection_message_appended`

- 含义：投影视图中新增一条公开消息
- 目标：`projection_room`
- 优先级：`P1`
- 当前兼容映射：
  - `chat_message`
  - `uw.message`

### `projection_message_cleared`

- 含义：消息列表被清空或重置
- 目标：`projection_room`
- 优先级：`P1`
- 当前兼容映射：`chat_cleared`

### `projection_presence_changed`

- 含义：共享在线态/忙闲态变化
- 目标：`projection_room`
- 优先级：`P2`
- 当前兼容映射：beeroom 中由 `team_task_*` 间接推导的 agent busy/idle

### `projection_resync_required`

- 含义：当前投影增量无法继续，需要重取快照
- 目标：`projection_room`
- 优先级：`P0`
- 当前兼容映射：`sync_required`

## 5.6 User World 专项事件

### `social_message_appended`

- 含义：社交会话新增消息
- 目标：`projection_room`
- 优先级：`P1`
- 当前兼容映射：`uw.message`

### `social_read_updated`

- 含义：已读状态更新
- 目标：`projection_room`
- 优先级：`P1`
- 当前兼容映射：`uw.read`

### `social_group_updated`

- 含义：群名、公告、成员数等摘要变化
- 目标：`projection_room`
- 优先级：`P2`
- 当前状态：代码层尚未形成统一实时事件，后续需新增

## 5.7 Control 事件

### `watching`

- 含义：watch 已建立
- 目标：任意
- 优先级：`P1`
- 当前已有：
  - beeroom
  - user_world
  - chat/watch 路径

### `ack`

- 含义：命令已接收并处理
- 目标：任意
- 优先级：`P1`
- 当前已有：user_world send/read

### `final`

- 含义：本次命令响应流结束
- 目标：任意
- 优先级：`P1`
- 当前已有：chat/user_world

### `backpressure`

- 含义：服务端要求客户端降级或重同步
- 目标：任意
- 优先级：`P0`
- 当前兼容映射：`slow_client(reason=queue_full_resume_required)`

### `resync_required`

- 含义：客户端当前 cursor 无法安全继续，需要重新同步
- 目标：任意
- 优先级：`P0`
- 当前兼容映射：`sync_required`

## 5.8 Telemetry 事件

### `usage_context_updated`

- 当前兼容映射：
  - `context_usage`
  - `quota_usage`
  - `round_usage`

### `workspace_changed`

- 当前兼容映射：`workspace_update`

### `plan_changed`

- 当前兼容映射：`plan_update`

### `question_panel_updated`

- 当前兼容映射：`question_panel`

这些事件默认不应作为最强实时保证对象，优先级通常是 `P2-P3`。

## 7. 字段约束

### 7.1 所有 P0/P1 事件必须带

- `event_type`
- `target_kind`
- `target_id`
- `epoch`
- `seq`
- `created_at`
- `priority`

### 7.2 所有 message 类事件必须带

- `logical_message_id`
- `source_kind`
- `thread_id` 或 `projection_room`
- `preview` 或结构化 message 内容

### 7.3 所有 tool 类事件必须带

- `tool_name`
- `call_id`
- `thread_id`
- `status`

### 7.4 所有 mission/task 类事件必须带

- `mission_id`
- `task_id`（task 类）
- `agent_id`（如有归属）
- `status`

## 8. 顺序与恢复规则

- 单 `thread` 内按 `epoch + seq` 严格应用
- 单 `mission` 内按 `epoch + seq` 严格应用
- `projection_room` 按自身 `epoch + seq` 应用
- 恢复顺序固定为：
  - `thread`
  - `mission`
  - `projection_room`
- 客户端必须支持幂等重放
- 旧 epoch 事件必须被丢弃
