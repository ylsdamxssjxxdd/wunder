# 渠道工具落地方案

## 1. 背景与目标

当前渠道层（含 XMPP）已经具备基础收发、文件入站落盘与出站链接改写能力，但模型侧缺少“可主动操作渠道”的统一工具，导致以下场景无法自助完成：

- 查询可联系对象（联系人/最近对话对象）。
- 将处理后的结果主动转发给指定渠道用户。
- 在回复中引用工作区文件，并自动转换为可下载链接后发送。

本方案新增内置工具 `渠道工具`（英文别名：`channel_tool`），为模型提供受控的渠道查询与发送能力。

## 2. 分期范围

## 2.1 P1（本次落地）

- `action=list_contacts`：基于渠道会话历史返回“可联系对象”。
- `action=send_message`：向指定渠道对象发送文本/附件（支持工作区文件引用转下载链接）。
- 权限控制：仅允许操作当前用户拥有绑定关系的渠道账号。
- 审计字段：统一写入 outbox meta，保留工具来源与会话上下文。

## 2.2 P2（后续增强）

- XMPP roster 真查询（`jabber:iq:roster`）并缓存。
- 联系人来源融合（roster + 历史会话），返回来源标签与可信度。
- 高风险转发策略（审批/白名单/频控）细化。

## 3. 能力节点设计

## 3.1 节点 A：联系人查询（list_contacts）

- 输入：`channel/account_id/keyword/offset/limit`（均可选，`offset/limit`用于分页）。
- 数据源：`channel_sessions`（按 `user_id` + 账号归属过滤）。
- 输出：
  - `items[]`：`channel/account_id/peer_kind/peer_id/thread_id/last_message_at/source`
  - `total/offset/limit`
- 说明：P1 的 `source` 固定为 `session_history`。

## 3.2 节点 B：渠道发送（send_message）

- 输入：
  - 最小必填：`text|content|attachments` 三选一（至少一个非空）
  - 常用选填：`to`（如不传，优先从当前会话唯一对端或账号默认目标补全）
  - 高级选填：`channel/account_id/peer_kind/thread_id/meta/wait/wait_timeout_s/contact`
- 执行：
  - 优先按 `channel/account_id` 显式入参发送；未显式传参时自动从“当前会话映射/账号默认配置”补全。
  - 校验账号归属与账号状态。
  - 构造 `ChannelOutboundMessage` 并写入 `channel_outbox`。
  - 由既有 outbox worker 发送，保持现有重试与适配器机制。
- 输出：
  - 基础：`outbox_id/status=pending`
  - 可选等待：`delivery.status(sent|failed|pending)` 与错误信息

## 3.3 节点 C：文件转发语义

- 模型在 `text/content` 中可直接引用 `/workspaces/...` 或 `![](/workspaces/...)`。
- 出站发送前复用已有链路：`append_temp_dir_links_for_outbound`。
- 对象渠道收到的是可下载链接，不暴露工作区真实路径。

## 3.4 节点 D：权限与审计

- 账号权限：
  - 以 `channel_user_bindings` 判断用户是否拥有该账号。
  - 兼容 legacy 账号 ID 与 `owner_user_id` 回退。
- 审计元数据：
  - `tool=channel_tool`
  - `tool_action`
  - `session_id/user_id/agent_id`
  - 透传调用方 `meta`（与系统字段合并）

## 4. 工具接口（模型侧）

工具名：`渠道工具`  
英文别名：`channel_tool`

统一入参：

- `action`：
  - `list_contacts`
  - `send_message`

`list_contacts` 主要参数：

- `channel`（可选）
- `account_id`（可选）
- `keyword`（可选）
- `offset`（可选，默认 0）
- `limit`（可选，默认 20，最大 200）

`send_message` 主要参数：

- `text` 或 `content` 或 `attachments`（三选一，至少一个非空）
- `to`（可选，当前会话存在唯一对端或账号已配置默认目标时可省略）
- `contact`（可选，对象；可直接复用 `list_contacts` 返回项，一次性带入 channel/account_id/to/peer_kind/thread_id）
- `channel`（可选）
- `account_id`（可选）
- `peer_kind`（可选，默认 `user`）
- `thread_id`（可选）
- `attachments`（可选，`ChannelAttachment[]`）
- `meta`（可选，对象）
- `wait`（可选，默认 false）
- `wait_timeout_s`（可选，默认 8，范围 1~30）

## 5. 关键时序

1. 模型调用 `channel_tool.send_message`。
2. 工具层做账号归属/参数校验。
3. 写入 `channel_outbox`（`pending`）。
4. outbox worker 投递到渠道适配器（XMPP/飞书等）。
5. 发送前执行工作区路径改写，生成临时下载链接。
6. 发送结果回写 outbox 状态（`sent/failed`）。

## 6. 失败与回退策略

- 参数缺失或非法：直接返回 `ok=false + error`。
- 账号无权限：拒绝发送并返回 `forbidden` 类错误。
- 账号不存在或禁用：拒绝发送并返回明确错误。
- outbox 入队成功但投递失败：保持重试机制，不阻断工具返回。
- `wait=true` 超时：返回 `pending`，由后台继续投递。

## 7. 兼容性与风险

- 与现有渠道入站流程兼容，不改 inbound 协议。
- 与现有 outbox/适配器兼容，不新增渠道专属队列。
- 主要风险：
  - 账号权限边界不清导致越权发送。
  - 文本内路径改写覆盖不到边界格式。
- 对策：
  - 强制账号归属校验 + 结构化 meta 审计。
  - 复用已在线上的统一链接改写函数，避免重复实现。

## 8. 验收标准

- 模型可通过 `channel_tool` 列出联系人并按关键词筛选。
- 模型可向指定 XMPP 对象发送消息。
- 文本中的工作区文件引用可被自动改写为可下载链接。
- `cargo check` 通过，关键路径有测试覆盖，文档同步更新。

## 9. P2 进展（2026-03-11）

- 已实现 XMPP roster 真查询（`jabber:iq:roster`），通过短连接 IQ get 拉取联系人。
- 已实现账号级 roster 缓存（短 TTL），并支持 `list_contacts.refresh=true` 强制刷新。
- `list_contacts` 已从“仅会话历史”升级为“会话历史 + roster 融合”。
- 返回 `source` 字段区分来源：`session_history` / `roster` / `session_history+roster`。

## 10. 测试开放态（2026-03-11）

- 目标：为内部联调先放开权限与审批，优先验证跨渠道转发链路。
- 权限策略：
  - `CHANNEL_TOOL_OPEN_ACCESS_FOR_TEST=true` 时，`channel_tool.send_message` 不再做账号归属拦截。
  - `channel_tool.list_contacts` 在该模式下可枚举系统内已配置账号（支持 `channel/account_id` 过滤）。
- 审批策略：
  - `CHANNEL_OPEN_APPROVAL_FOR_TEST=true` 时，ChannelHub 为渠道请求注入 `security.approval_mode=full_auto` 与 `security.exec_policy_mode=allow`。
  - 同时不再挂载渠道审批回传通道，渠道内 `1/2/3` 审批提示默认不会出现。
- 风险提示：当前模式仅用于测试环境，回归生产前需恢复配置化开关并按账号/环境细化权限边界。

## 11. 参数简化（2026-03-11）

- `send_message` 不再强制要求 `channel/account_id/to` 同时传入；系统会自动补全可推断参数。
- 自动补全顺序：
  - 账号：`显式入参` -> `当前会话映射` -> `唯一 active 账号` -> `channel_tool_default=true` 的账号。
  - 目标：`显式入参`/`contact` -> `当前会话唯一对端` -> `账号默认目标配置`。
- `list_contacts` 新增 `contact` 字段，模型可直接将该对象传回 `send_message`，减少字段拼装。
- 账号可预设配置（写入渠道账号 config）：
  - `channel_tool_default: true`
  - `channel_tool_default_users: ["user_a", "user_b"]`（可选，按用户生效）
  - `channel_tool_default_to: "someone@example.com"`
  - `channel_tool_default_peer_kind: "user"`
  - `channel_tool_default_thread_id: "thread-123"`
  - 以上键均支持 `*_by_user` 映射写法（如 `channel_tool_default_to_by_user`）。
