## 1. 全局改造约束

### 1.1 Agent First 约束

- `thread system prompt` 必须冻结，后续任何 realtime/projection 状态不得反写。
- 长期记忆只允许在线程初始化时注入一次，后续只允许通过普通消息或结构化事件影响执行。
- `Kernel Plane > Mission Plane > Projection Plane` 的优先级必须写入实现，不允许 UI 反向主导 runtime。
- 背压时优先保护 `cancel / tool_result / mission_finalize / first_token`，再保护摘要面板。

### 1.2 兼容与灰度

- 保留当前 `/wunder/chat/ws`、`/wunder/beeroom/ws`、`/wunder/user_world/ws` 一段时间，先作为 adapter。
- 旧的 HTTP detail/list 接口先保留，只把状态源从“即时聚合”切到“projection snapshot”。
- 所有新 runtime 需要有 `legacy | hybrid | runtime` 三档切换。

### 1.3 代码组织

- 不再向 [tools.rs](C:/Users/sjxx/Desktop/wunder/src/services/tools.rs) 堆新能力。
- 不再向 [MessengerView.vue](C:/Users/sjxx/Desktop/wunder/frontend/src/views/MessengerView.vue) 堆运行时逻辑。
- 后端新增模块优先放在：
  - `src/services/runtime/`
  - `src/services/projection/`
  - `src/services/directory/`
  - `src/services/presence/`
- 前端新增模块优先放在：
  - `frontend/src/realtime/agentWorld/`
  - `frontend/src/stores/agentWorld/`
  - `frontend/src/views/messenger/*` 的拆分目录
