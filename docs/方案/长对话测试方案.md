# 长对话测试方案

## 1. 目标
验证在长对话与大上下文注入场景下，系统是否满足以下要求：
- 能触发并完成上下文压缩（`compaction` 事件可观测）。
- 遇到模型上下文超限后可自动恢复（`overflow_recovery` 路径）。
- 会话不中断，后续轮次仍可持续返回 `final + turn_terminal`。

## 2. 前置条件
- 服务地址：`http://127.0.0.1:18000`
- 鉴权：`Authorization: Bearer ylsdamxssjxxdd`
- 使用同一个 `session_id` 连续压测。
- 默认模型为小上下文模型（例如 32768）。

建议先确认模型实际请求地址在容器内可达（避免把容器内 `127.0.0.1` 误当宿主机）。

## 3. 关键观测事件
重点看以下 SSE 事件：
- `context_usage`：当前上下文与预计请求 token。
- `compaction`：压缩触发与结果（`reason/status/...`）。
- `error`：是否出现 `CONTEXT_WINDOW_EXCEEDED` 等异常。
- `final`：本轮是否产出最终响应。
- `turn_terminal`：本轮是否正常闭环结束。
- `llm_request`：确认模型、base_url、provider 是否正确。

## 4. 快速健康检查（1轮）
先做 1 轮小请求，确认链路通畅：

```powershell
$env:PYTHONIOENCODING='utf-8'; @'
import json, time, requests
url='http://127.0.0.1:18000/wunder'
headers={'Authorization':'Bearer ylsdamxssjxxdd','Content-Type':'application/json'}
payload={
  'user_id':'long_chat_probe_user',
  'session_id':f'sess_quick_{int(time.time())}',
  'question':'快速健康检查：请回复 OK。',
  'stream':True,
  'skip_tool_calls':True,
  'allow_queue':False,
}
r=requests.post(url,headers=headers,json=payload,stream=True,timeout=(10,120))
print('http', r.status_code)
event='message'; buf=[]; final=False; terminal=False; err=False
if r.status_code==200:
  for raw in r.iter_lines(decode_unicode=True):
    if raw is None: continue
    if raw=='':
      if buf:
        if event=='final': final=True
        if event=='turn_terminal': terminal=True
        if event=='error': err=True
        buf=[]
      event='message'; continue
    if raw.startswith('event:'): event=raw[6:].strip()
    elif raw.startswith('data:'): buf.append(raw[5:].lstrip())
print({'final':final,'turn_terminal':terminal,'error':err})
'@ | python -
```

通过标准：`http=200` 且 `final=True` 且 `turn_terminal=True`。

## 5. 压缩触发与恢复测试（核心）
说明：
- 使用同一个 `session_id` 连续发问。
- 在指定轮次注入大文本，强制制造上下文压力。
- 建议先用中等注入（4万~6万字符）；若未触发，再逐步增大。

```powershell
$env:PYTHONIOENCODING='utf-8'; @'
import json, time, requests, random, string, warnings
warnings.filterwarnings('ignore')

BASE='http://127.0.0.1:18000/wunder'
HEADERS={'Authorization':'Bearer ylsdamxssjxxdd','Content-Type':'application/json'}
USER_ID='long_chat_probe_user'
SESSION_ID=f'sess_long_{int(time.time())}'


def make_big_block(chars:int, seed:int)->str:
    rnd=random.Random(seed)
    parts=[]
    total=0
    while total < chars:
        tok=''.join(rnd.choice(string.ascii_letters+string.digits) for _ in range(48))
        line=f'k={tok} v={tok[::-1]} marker=CTX\\n'
        parts.append(line)
        total += len(line)
    return ''.join(parts)[:chars]


def iter_sse(resp):
    event='message'; buf=[]
    for raw in resp.iter_lines(decode_unicode=True):
        if raw is None:
            continue
        if raw=='':
            if buf:
                yield event, '\\n'.join(buf)
                buf=[]
            event='message'
            continue
        if raw.startswith('event:'):
            event=raw[6:].strip()
        elif raw.startswith('data:'):
            buf.append(raw[5:].lstrip())


heavy_turns={4:45000, 8:55000}
max_turns=12
results=[]

for turn in range(1, max_turns+1):
    if turn in heavy_turns:
        block=make_big_block(heavy_turns[turn], seed=turn*101)
        q=(f'第{turn}轮大注入测试：请仅回复 ACK-{turn}。\\n'
           f'BEGIN-{turn}\\n{block}\\nEND-{turn}')
    else:
        q=f'第{turn}轮常规追问：请回复 ACK-{turn}。'

    payload={
        'user_id':USER_ID,
        'session_id':SESSION_ID,
        'question':q,
        'stream':True,
        'skip_tool_calls':True,
        'allow_queue':False,
    }

    r=requests.post(BASE,headers=HEADERS,json=payload,stream=True,timeout=(10,360))
    row={
        'turn':turn,
        'http':r.status_code,
        'events':0,
        'final':False,
        'turn_terminal':False,
        'errors':0,
        'compactions':0,
        'last_context_tokens':None,
        'last_projected_tokens':None,
    }

    if r.status_code==200:
        for event, data in iter_sse(r):
            row['events'] += 1
            try:
                obj=json.loads(data)
            except Exception:
                obj={}
            inner=obj.get('data') if isinstance(obj,dict) else None

            if event=='final':
                row['final']=True
            elif event=='turn_terminal':
                row['turn_terminal']=True
            elif event=='error':
                row['errors'] += 1
            elif event=='compaction':
                row['compactions'] += 1
            elif event=='context_usage' and isinstance(inner,dict):
                row['last_context_tokens']=inner.get('context_tokens')
                row['last_projected_tokens']=inner.get('projected_request_tokens')

    results.append(row)
    print(json.dumps(row, ensure_ascii=False))

summary={
  'session_id':SESSION_ID,
  'turns':len(results),
  'http_failures':sum(1 for x in results if x['http']!=200),
  'turns_with_final':sum(1 for x in results if x['final']),
  'turns_with_terminal':sum(1 for x in results if x['turn_terminal']),
  'turns_with_error':sum(1 for x in results if x['errors']>0),
  'total_compactions':sum(x['compactions'] for x in results),
  'compaction_turns':[x['turn'] for x in results if x['compactions']>0],
}
print('===SUMMARY===')
print(json.dumps(summary, ensure_ascii=False))
'@ | python -
```

## 6. 判定标准
通过（建议）：
- `http_failures == 0`
- 至少一次出现 `compaction`（`total_compactions > 0`）
- 压测后续轮次仍有 `final + turn_terminal`
- 即使出现 `CONTEXT_WINDOW_EXCEEDED`，后续轮次能恢复并继续对话

关注但不判死：
- 大注入轮耗时显著上升（小模型常见）
- 模型只输出 reasoning，最终内容被系统兜底

失败（需修复）：
- 服务无响应或频繁非 200
- 触发超限后会话持续不可恢复
- 连续多轮无法拿到 `turn_terminal`

## 7. 实战经验与调参建议
- 若单轮耗时过长先降低注入量（如从 9 万降到 4~6 万字符）。
- 先中压触发，再做高压复测，避免一次压测卡住观察窗口。
- 每轮完整消费 SSE 到连接结束，不要在 `final` 后立即中断读取，避免统计偏差。
- 若压测脚本中断，先清理残留 `python` 进程再复测。

## 8. 本方案回归目标
目标不是“每轮都高质量回答”，而是验证：
- 不管用户如何连续提问或注入大上下文，会话都能持续运行；
- 系统能自动压缩、自动恢复，不因上下文超限把线程跑死。
