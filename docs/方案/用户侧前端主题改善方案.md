# 用户侧前端主题统一方案

## 1. 现状问题

当前有 5 个调色板 + 1 个 dark/light 开关，结构混乱：

| 调色板 | base.css 变量 | messenger 覆写 | shell 覆写 | 专属文件 |
| --- | --- | --- | --- | --- |
| eva-orange | 完整(14变量) | 仅 .messenger-view(25行) | 无 | 无 |
| claw-orange | 完整(14变量) | **无** | 无 | 无 |
| hula-green | 完整(14变量) | 仅 .messenger-view(25行) | 无 | 无 |
| minimal | 完整(14变量) | .messenger-view + 36个组件覆写(168行) | 无 | 无 |
| tech-blue | 完整(14变量) | 无 | 全部硬编码蓝黑色 | tech-blue-dark.css(2981行) |

另外 `light.css` 仅覆盖 chat-shell，`data-user-theme` dark/light 开关只有 tech-blue 完整支持。

**核心问题：覆盖极度不均，dark/light 开关是伪命题，shell.css 全量硬编码蓝色。**

## 2. 设计原则

1. **一个主题 = 一套完整样式**，不区分 dark/light，取消 `data-user-theme` 属性
2. **以 hula-green 为标准模板** — 所有主题覆盖层级与变量数量保持一致
3. **三级覆盖架构**：base.css 定义 accent 变量 → messenger.css 覆写 messenger 组件变量 → shell.css 使用变量不硬编码
4. **tech-blue 天然是暗色主题**，不需要 dark/light 开关，直接作为暗色主题存在
5. **消除所有专属主题文件**，将 light.css 和 tech-blue-dark.css 的内容归入各主题的标准覆盖块

## 3. 标准覆盖模板（以 hula-green 为准）

每个主题必须在以下位置提供完整且一致的覆盖：

### 3.1 base.css — accent 变量块（已有，保持 14 变量不变）

```css
:root[data-user-accent='<theme>'] {
  --ui-accent / --ui-accent-rgb
  --ui-accent-secondary / --ui-accent-secondary-rgb
  --ui-accent-deep
  --ui-accent-soft / --ui-accent-soft-2 / --ui-accent-soft-3 / --ui-accent-soft-4 / --ui-accent-soft-5
  --ui-accent-gradient-start / --ui-accent-gradient-mid
  --ui-accent-hover / --ui-accent-disabled / --ui-accent-link-hover
}
```

### 3.2 base.css — 全局背景变量块（新增）

每个主题增加全局背景色定义，取代 `--dark-bg` 等通用变量：

```css
:root[data-user-accent='<theme>'] {
  /* 已有的 14 个 accent 变量 ... */
  --theme-bg: <主背景色>;
  --theme-panel: <面板/卡片背景>;
  --theme-border: <边框色>;
  --theme-text: <正文色>;
  --theme-muted: <次要文字色>;
}
```

删除 `:root` 中默认的 `--dark-bg / --dark-panel / --dark-border / --dark-text / --dark-muted`，改为 `var(--theme-*)`。

### 3.3 messenger.css — .messenger-view 变量覆写（hula-green 的 25 个变量为标准）

```css
:root[data-user-accent='<theme>'] .messenger-view {
  --hula-left-bg: ...;
  --hula-left-bg-deep: ...;
  --hula-main-bg: ...;
  --hula-center-bg: ...;
  --hula-list-hover: ...;
  --hula-border: ...;
  --hula-muted: ...;
  --hula-text: ...;
  --hula-accent: ...;
  --hula-accent-soft: ...;
  --messenger-middle-pane-bg: ...;
  --messenger-chat-bg: ...;
  --messenger-right-dock-bg: ...;
  --messenger-search-bg: ...;
  --messenger-search-border: ...;
  --messenger-control-bg: ...;
  --messenger-control-border: ...;
  --messenger-control-text: ...;
  --messenger-composer-surface-bg: ...;
  --messenger-composer-box-bg: ...;
  --messenger-composer-panel-bg: ...;
  --messenger-composer-subtle-bg: ...;
  --messenger-composer-border: ...;
  background: ...;
}
```

### 3.4 shell.css — 通过变量驱动，不硬编码

`.chat-shell` 的默认值改为中性变量，各主题通过 `:root[data-user-accent='<theme>'] .chat-shell` 覆写：

```css
/* 默认值（中性灰） */
.chat-shell {
  --chat-bg: var(--theme-bg, #f1f1f1);
  --chat-text: var(--theme-text, #18181c);
  --chat-muted: var(--theme-muted, #808080);
  --chat-primary: var(--ui-accent);
  --chat-primary-rgb: var(--ui-accent-rgb);
  /* ... */
}

/* 各主题覆写 */
:root[data-user-accent='tech-blue'] .chat-shell {
  --chat-bg: #0b1426;
  --chat-surface: #111a2f;
  --chat-text: #f1f5f9;
  --chat-muted: #b3bccb;
  --chat-primary: #6ad9ff;
  /* ... */
}
```

## 4. 实施步骤

### 第一步：基础架构调整

1. **base.css** — 为每个主题增加 `--theme-bg/panel/border/text/muted` 5 个变量；删除 `:root` 默认的 `--dark-*` 变量块和 `data-user-theme='dark'` 块
2. **base.css** — `body { background: var(--dark-bg) }` 改为 `body { background: var(--theme-bg) }`
3. 全局搜索替换 `var(--dark-bg)` → `var(--theme-bg)` 等引用

### 第二步：shell.css 变量化

1. `.chat-shell` 默认变量改为引用 `var(--theme-*)` 和 `var(--ui-accent-*)`
2. 所有硬编码的蓝色（`#6ad9ff`、`rgba(77, 216, 255, ...)`、`#0b1426` 等）改为变量引用
3. 为 tech-blue 增加 `.chat-shell` 覆写块（保留暗蓝色风格）
4. 为 eva-orange / claw-orange / hula-green / minimal 各增加 `.chat-shell` 覆写块
5. 删除 `light.css`，其内容已归入各主题的 shell 覆写块

### 第三步：messenger.css 补齐

1. 为 **claw-orange** 补齐 `.messenger-view` 的 25 个变量覆写（参照 hula-green 的结构和色系）
2. 确认 eva-orange / hula-green / minimal 的覆写完整（均已存在，检查变量数量是否对齐）
3. 为 **tech-blue** 增加 `.messenger-view` 变量覆写（从 tech-blue-dark.css 中提取 messenger 相关部分）
4. minimal 的 36 个额外组件覆写保留不动（它们是 minimal 风格特有的"去装饰"处理）

### 第四步：清除 tech-blue-dark.css

1. 将 tech-blue-dark.css 中的有用覆写按功能拆分归入对应文件：
   - messenger 部分 → `messenger.css` 的 tech-blue 块
   - shell 部分 → `shell.css` 的 tech-blue 块
   - 其他页面（auth、portal 等）→ 对应页面 CSS 的 tech-blue 块
2. 删除 tech-blue-dark.css 文件
3. 删除 light.css 文件

### 第五步：清理前端代码中的 data-user-theme 引用

1. 搜索所有 `data-user-theme` 的设置和读取代码，移除 dark/light 切换逻辑
2. 移除主题切换按钮（或改为纯调色板切换）
3. 清理 Vue 组件中残留的硬编码颜色值

### 第六步：验证与调整

1. 逐主题验证所有页面（messenger 四栏、chat shell、auth 登录页、portal 门户页）
2. 确保每个主题切换后视觉完整、无遗漏区域
3. 检查对比度是否满足 WCAG AA

## 5. 各主题定位

| 主题 | 风格 | 背景基调 |
| --- | --- | --- |
