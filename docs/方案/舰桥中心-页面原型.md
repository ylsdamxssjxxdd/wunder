# 舰桥中心页面原型

## 1. 当前页面形态

舰桥中心已经落地为“监控主页 + 弹窗配置”，而不是 Tab 式重配置页面。

当前结构：

- 左侧：舰桥节点列表
- 右侧：节点概览、自动分配路由、最近投递日志
- 弹窗 1：中心配置
- 弹窗 2：渠道设置

---

│ 舰桥节点列表          │ 当前舰桥节点                                    │
│                       │                                                  │

│ active                │ [中心配置] [渠道设置]                            │
│ 渠道 2 | 路由 18      │                                                  │
│                       │ 概览指标：                                       │
│ 售后服务节点          │ - 节点状态                                       │

## 3. 中心配置弹窗

字段：

- 舰桥节点名称
- 状态
- 默认预设智能体
- 目标单位
- 说明

特征：

- 不展示 `中心编码`
- 不展示 `用户名策略`
- 不展示 `默认身份策略`
- 保存后即可继续配置节点渠道

原型：

```text
┌─ 中心配置 ───────────────────────────┐
│ 节点名称 [售前咨询节点__________]     │
│ 状态     [active▼]                   │
│ 默认预设 [客服助手▼]                 │
│ 目标单位 [默认不指定▼]               │
│ 说明     [______________________]    │
│                                      │
│              [取消] [保存节点]       │
└──────────────────────────────────────┘
```

---

## 4. 渠道设置弹窗

这是管理员侧单独实现的一套 JS 表单，不复用用户侧 Vue 页面。

表单结构：

- 绑定规则：每个舰桥节点只允许绑定一个渠道账号
- 表单字段：
  - 渠道
  - 账号 ID
- Weixin 特例：
  - 当选择 `Weixin iLink (New)` 时，弹窗内直接显示二维码区域
  - 点击“生成二维码”后自动拉起扫码等待
  - 扫码确认后自动创建或更新渠道账号并绑定当前节点
- 无 JSON 配置、无启用开关、无预设覆盖、无线程策略、无状态说明、无能力说明块

原型：

```text
┌─ 渠道设置 ──────────────────────────────────────────────────────┐
│ 渠道设置                                                        │
│ 每个节点只绑定一个渠道账号                                      │
│                                                                 │
│ 渠道        [xmpp▼]                                             │
│ 账号 ID     [shared_xmpp_01▼]                                   │
│                                                                 │
│ Weixin iLink (New) 扫码                                         │
│ [生成二维码]                                                    │
│ [二维码图片]                                                    │
│ 会话 Key / 状态 / 提示消息                                      │
│                                                                 │
│                 [移除绑定] [关闭] [保存渠道]                    │
└─────────────────────────────────────────────────────────────────┘
```

---

## 5. 关键体验要求

当前页面已经按以下要求收敛：

- 主页优先展示监控，不堆复杂配置。
- 配置都通过弹窗进入，避免增加心智负担。
- 默认预设智能体必须能正常拉到已有预设数据。
- 渠道设置支持所有渠道，不只支持 XMPP。
- 每个舰桥节点只绑定一个渠道账号。
- 管理端渠道设置是单独的原生 JS 实现，不再依赖用户侧前端结构。
- `Weixin iLink (New)` 必须在管理员侧弹窗内支持二维码扫码，不要求管理员跳去用户侧页面。
