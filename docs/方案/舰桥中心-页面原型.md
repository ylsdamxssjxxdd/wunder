# 舰桥中心页面原型

## 1. 当前页面形态

舰桥中心已经落地为“监控主页 + 弹窗配置”，而不是 Tab 式重配置页面。

当前结构：

- 左侧：舰桥节点列表
- 右侧：节点概览、接入渠道监控、自动分配路由、最近投递日志
- 弹窗 1：中心配置
- 弹窗 2：接入渠道

---

## 2. 页面原型

```text
┌─ 舰桥中心 ───────────────────────────────────────────────────────────────┐
│ [刷新] [新建节点] [删除节点]                                            │
│ 管理员只需要创建舰桥节点、设置默认预设、配置节点专属接入渠道。           │
├───────────────────────┬────────────────────────────────────────────────┤
│ 舰桥节点列表          │ 当前舰桥节点                                    │
│                       │                                                  │
│ 售前咨询节点          │ 名称 / 创建人 / 更新时间                         │
│ active                │ [中心配置] [接入渠道]                            │
│ 渠道 2 | 路由 18      │                                                  │
│                       │ 概览指标：                                       │
│ 售后服务节点          │ - 节点状态                                       │
│ paused                │ - 默认预设                                       │
│ 渠道 1 | 路由 0       │ - 目标单位                                       │
│                       │ - 接入渠道数                                     │
│                       │ - 自动路由数                                     │
│                       │ - 活跃路由数                                     │
│                       │                                                  │
│                       │ 接入渠道表：                                     │
│                       │ 渠道 / 账号 / 显示名 / 配置状态 / 路由数 / 操作   │
│                       │                                                  │
│                       │ 自动分配路由表：                                 │

│                       │                                                  │
│                       │ 最近投递日志：                                   │
│                       │ 方向阶段 / 状态 / 摘要 / 时间                    │
└───────────────────────┴────────────────────────────────────────────────┘
```

---

## 3. 中心配置弹窗

字段：

- 舰桥节点名称
- 状态
- 默认预设智能体
- 目标单位
- 说明

特征：

- 不展示 `中心编码`
- 不展示 `用户名策略`
- 不展示 `默认身份策略`
- 保存后即可继续配置接入渠道

原型：

```text
┌─ 中心配置 ───────────────────────────┐
│ 节点名称 [售前咨询节点__________]     │
│ 状态     [active▼]                   │
│ 默认预设 [客服助手▼]                 │
│ 目标单位 [默认不指定▼]               │
│ 说明     [______________________]    │
│                                      │
│              [取消] [保存节点]       │
└──────────────────────────────────────┘
```

## 4. 接入渠道弹窗

这是管理员侧单独实现的一套 JS 表单，不复用用户侧 Vue 页面。

表单结构：

- 顶部提示区：显示当前渠道名称、描述、回调提示、能力快照、是否有适配器
- 操作按钮：`新增渠道`、`填入模板`、`清空为 {}`
- 公共字段：
  - 渠道
  - 账号 ID
  - 显示名称
  - 默认会话类型
  - 预设覆盖
  - 线程策略
  - 状态说明
  - 启用开关
- 渠道专属字段：统一写在 JSON 配置框中

原型：

```text
┌─ 接入渠道 ──────────────────────────────────────────────────────┐
│ 编辑接入渠道                                                    │
│ 直接在这里配置节点专属渠道账号                                  │
│                                                                 │
│ 渠道说明：XMPP (xmpp)   模式 runtime+generic   适配器已接线      │
│ 说明文字 / docs_hint / provider_caps                           │
│                                       [填入模板] [清空为 {}]    │
│                                                                 │
│ 渠道        [xmpp▼]                                             │
│ 账号 ID     [shared_xmpp_01____________]                        │
│ 显示名称     [客服主账号________________]                        │
│ 默认会话类型 [group▼]                                           │
│ 预设覆盖     [沿用节点默认预设▼]                                 │
│ 线程策略     [main_thread▼]                                     │
│ 状态说明     [____________________________]                      │
│ [x] 启用该接入渠道                                              │
│                                                                 │
│ JSON 配置                                                      │
│ {                                                               │
│   "xmpp": {                                                    │
│     "jid": "bot@example.com",                                 │
│     "password": ""                                            │
│   }                                                             │
│ }                                                               │
│                                                                 │
│                   [删除] [关闭] [保存渠道]                      │
└─────────────────────────────────────────────────────────────────┘
```

## 5. 关键体验要求

当前页面已经按以下要求收敛：

- 主页优先展示监控，不堆复杂配置。
- 配置都通过弹窗进入，避免增加心智负担。
- 默认预设智能体必须能正常拉到已有预设数据。
- 接入渠道支持所有渠道，不只支持 XMPP。
- 管理端接入渠道是单独的原生 JS 实现，不再依赖用户侧前端结构。
