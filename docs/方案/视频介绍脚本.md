> 语气：专业、清晰、可信赖
> 语速建议：每分钟 200~230 字，整体约 2200~2500 字

- 6:30-7:40 管理侧：用户与权限治理

## 逐段脚本

### 0:00-0:40 开场与一句话定位
- 画面：品牌封面 + 关键词闪现（统一入口、多模型、多工具、可治理）。
- 旁白：
  “大家好，欢迎了解 wunder。它是一个智能体调度平台，用一个入口对接多模型、多工具、多流程，让智能体真正成为企业生产力。接下来用 10 分钟，带你看清它的理念、功能与管理价值。”

### 0:40-1:40 设计理念与价值主张
- 画面：三条核心理念依次出现，配合简洁图标。
- 旁白：
  “wunder 的设计理念可以用三句话概括。对开发者来说，一切都是接口，业务能力通过清晰的 API 与配置快速接入；对大模型来说，一切皆工具，模型只需调用工具就能完成复杂任务，减少不可控的自由生成；对企业来说，一切可治理，过程可追溯、成本可量化、风险可控制。这样，智能体不只是‘会回答’，而是能稳定落地、可持续运营。”

### 1:40-2:40 架构与运行流程（概览）
- 画面：简化架构图，/wunder 入口指向调度引擎、工具体系与存储监控。
- 旁白：
  “在架构上，wunder 以 Rust + Axum 提供统一的 `/wunder` 入口，外部系统只需传入 user_id 与问题即可发起请求。调度引擎负责模型选择、工具编排与流程控制，并将执行过程以 SSE 流式事件返回。核心组件包含 Prompt Builder、工具执行层、存储与监控体系，既保证速度，也保证长期稳定运行。”

### 2:40-4:10 用户侧：新建智能体与能力配置
- 画面：用户侧功能广场 `/home`，首卡“新建智能体应用”，配置面板展开。
- 旁白：
  “在用户侧，入口是功能广场。首卡就是‘新建智能体应用’，用户可以快速创建自己的智能体。创建时可填写名称与描述，配置系统提示词；选择需要的工具、技能与知识库，并决定是否共享给同等级用户。完成配置后即可一键启动，进入专属会话。这样，业务人员无需开发，也能把常用流程沉淀成可复用的智能体应用。”

### 4:10-5:20 用户侧：多智能体并行与工作区
- 画面：多个智能体同时运行，左右并行展示会话与结果，底部展示工作区文件。
- 旁白：
  “wunder 的并行能力来自 `user_id + agent_id` 的隔离策略。同一用户可以同时运行多个智能体应用，互不干扰、互不串话，每个智能体拥有独立会话和工作区。用户可以并行处理多个任务，例如一个智能体做资料整理，另一个智能体做报告生成。多智能体同时运行，真正把协作效率拉满。”

### 5:20-6:30 用户侧：工具/知识库/技能与流式过程
- 画面：工具管理、知识库检索、技能调用、流式事件输出示意。
- 旁白：
  “用户可以在工具管理中配置 MCP 工具、技能与知识库，把能力变成可编排的工具集。对话中，系统会流式返回中间过程与最终回复，工具调用与结果会清晰展示。配合计划面板、问询面板与 A2UI 输出，用户能看到模型的步骤、可选路径与可视化结果，理解‘为什么这么做’并能及时干预。”

### 6:30-7:40 管理侧：用户与权限治理
- 画面：管理端用户管理、权限控制、额度配置、工具授权。
- 旁白：
  “管理侧提供完整的用户治理能力：账号创建与禁用、权限等级控制、工具与智能体访问策略，以及每日额度的配置与管理。管理员还能统一维护 MCP、技能与知识库，并为不同用户设置可用范围。这让智能体能力不仅可用，更可控、可管、可扩展。”

### 7:40-8:40 管理侧：监控、成本与性能
- 画面：监控面板、线程详情、工具使用统计、性能采样报表。
- 旁白：
  “系统内置监控与评估能力，支持会话状态追踪、工具调用统计与异常回放。成本统计以‘上下文占用’为口径，配合额度展示，让消耗可见、可算。性能方面提供采样与压测接口，帮助定位瓶颈并持续优化，保证长期稳定运行。”

### 8:40-9:30 生态与集成（MCP/A2A/API）
- 画面：MCP 服务、A2A 接口、外部系统对接箭头。
- 旁白：
  “wunder 能对接 MCP 工具服务，也支持 A2A 标准协作接口，让跨智能体协作成为可能。对外系统只需调用 `/wunder`，即可获得多模型、多工具的统一能力入口。无论是内部系统集成，还是对外能力开放，都能快速落地。”

### 9:30-10:00 总结与收尾
- 画面：品牌收尾 + 关键价值回顾。
- 旁白：
  “wunder 不是聊天机器人，而是企业级智能体调度平台。它让能力可集成、流程可沉淀、成本可治理、风险可控制。现在，就用一个入口，把智能体能力真正落地到业务现场。”
