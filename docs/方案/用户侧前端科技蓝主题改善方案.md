# 用户侧前端主题统一与优化方案

## 1. 现状分析

### 1.1 主题体系

当前系统有 5 个调色板 + 2 个明暗模式：

| 调色板 | base.css 变量 | messenger 组件覆写 | 专属主题文件 | dark 模式 | light 模式 |
| --- | --- | --- | --- | --- | --- |
| eva-orange | 完整 | 仅 .messenger-view (15行) | 无 | 跟随全局 | 跟随全局 |
| claw-orange | 完整 | 无 | 无 | 跟随全局 | 跟随全局 |
| hula-green | 完整 | 仅 .messenger-view (15行) | 无 | 跟随全局 | 跟随全局 |
| minimal | 完整 | 36 个组件覆写 | 无 | 跟随全局 | 跟随全局 |
| tech-blue | 完整 | 无（全在 dark 文件里） | tech-blue-dark.css (2933行) | 强制锁定 dark | 不支持 |

### 1.2 核心问题

1. **覆盖极度不均** — tech-blue 有 2933 行专属覆写，claw-orange 只有 base 变量、零组件覆写
2. **chat-shell 硬编码** — `.chat-shell` 的 `--chat-bg: #0b1426` 等颜色固定为暗蓝，不跟随调色板切换
3. **dark 模式仅 tech-blue 可用** — 其他 4 个调色板切 dark 时大量组件失色或对比度不足
4. **light 模式覆盖有限** — `light.css` 只覆盖了 chat-shell 相关组件，messenger 侧栏/导航等区域未覆盖
5. **messenger 四栏配色不统一** — 左栏用 `--ui-accent-secondary`、中间用固定灰、右栏用 `--hula-main-bg`，切换调色板时视觉割裂
6. **Vue 组件内散落 19 处内联主题样式** — 部分 Vue 文件里有硬编码颜色值，绕开了主题变量

### 1.3 设计原则

- **tech-blue 保持仅 dark** — 它本身就是深色主题，不提供 light 变体
- **其他 4 个调色板均支持 dark + light** — 通过统一的变量和覆写文件实现
- **变量优先** — 所有颜色走 CSS 变量，消灭硬编码色值
- **分层治理** — base.css 定变量 → 各组件用变量 → 主题文件覆写变量值

## 2. 改善方案

### A. 配色架构统一

#### A1. chat-shell 配色改为变量驱动

**问题**：`chat/shell.css` 中 `--chat-bg: #0b1426` 等颜色硬编码为暗蓝。

**改动**：
- `.chat-shell` 默认变量改为中性暗色（非特定蓝），作为 dark 模式的默认值
- 各调色板通过 `:root[data-user-accent='xxx'] .chat-shell` 覆写为自己的色系
- `light.css` 已有的覆写保持不变
- 新增其他调色板的 dark 模式 chat-shell 覆写

**涉及文件**：
- `frontend/src/styles/chat/shell.css` — 默认值改为中性暗色
- `frontend/src/styles/base.css` — 无需改动（调色板变量已完整）
- `frontend/src/styles/theme/light.css` — 保持现有
- 新增组件级覆写块

#### A2. 为 eva-orange / claw-orange / hula-green 补齐 messenger 组件覆写

**问题**：这三个调色板只有 base 变量，切换后 messenger 四栏、按钮、气泡等不跟随。

**改动**：在 `messenger.css` 中为这三个调色板补齐组件覆写，参照 minimal 的 36 个覆写点：

覆盖区域：
- `.messenger-view` 四栏背景色
- `.messenger-left-rail` 导航栏
- `.messenger-middle-pane` 中间面板
- `.messenger-chat` 聊天区
- `.messenger-right-dock` 右栏
- `.messenger-avatar-btn` 头像按钮
- `.messenger-left-nav-btn` 导航按钮
- `.messenger-message-bubble` 消息气泡
- `.messenger-list-item` 列表项
- `.messenger-world-composer` 输入区
- `.messenger-scroll-top-btn` / `scroll-bottom-btn` 滚动按钮

**涉及文件**：`frontend/src/styles/messenger.css`

#### A3. 为 eva-orange / claw-orange / hula-green / minimal 补齐 dark 模式

**问题**：只有 tech-blue 有 dark 模式覆写，其他调色板切 dark 时大量区域失色。

**改动**：在 `base.css` 中为每个调色板增加 dark 模式变量块：

```
:root[data-user-theme='dark'][data-user-accent='eva-orange'] { ... }
:root[data-user-theme='dark'][data-user-accent='claw-orange'] { ... }
:root[data-user-theme='dark'][data-user-accent='hula-green'] { ... }
:root[data-user-theme='dark'][data-user-accent='minimal'] { ... }
```

每个 dark 块定义：
- `--dark-bg`、`--dark-panel`、`--dark-border`、`--dark-text`、`--dark-muted`
- `--ui-accent-soft` 系列改为 rgba 透明值（适配暗背景）

同时在 `messenger.css` 中为每个调色板增加 dark 组件覆写。

**涉及文件**：
- `frontend/src/styles/base.css`
- `frontend/src/styles/messenger.css`

#### A4. 清理 Vue 组件内联硬编码颜色

**问题**：19 个 Vue 文件中有硬编码色值，绕开了主题系统。

**改动**：
- 逐文件将硬编码颜色替换为 `var(--ui-accent)` 或对应的 CSS 变量
- 对确实需要固定颜色的地方（如文件夹图标黄色），提取为主题变量

**涉及文件**：19 个 Vue 文件（需逐个排查）

### B. 布局与视觉优化

#### B1. Messenger 网格比例微调

**问题**：四栏比例 `60px / 220px / minmax(460px,1fr) / 284px` 在小屏下右栏被过度压缩。

**改动**：
- middle pane 从 220px 收窄到 200px
- chat min-width 从 460px 降到 420px
- 右栏在小屏（<=1280px）默认折叠
- 增加中间面板收起时 chat 自动撑满的响应式规则

**涉及文件**：`frontend/src/styles/messenger.css`

#### B2. Chat 空态优化

**改动**：
- 新会话欢迎语改为更简洁的引导文案
- 空态改为轻量 SVG 插图
- 添加快捷入口提示

**涉及文件**：`frontend/src/views/MessengerView.vue` 或其子组件

#### B3. 消息气泡视觉改善

**改动**：
- AI 消息和用户消息视觉区分更明显
- 气泡增加微妙的 glass 效果
- 工具调用折叠/展开交互更清晰
- thinking 区域增加折叠动画

**涉及文件**：
- `frontend/src/styles/chat/messages.css`
- `frontend/src/styles/chat/workflow.css`

- `frontend/src/styles/messenger.css`

## 5. 注意事项

- tech-blue 保持仅 dark，不做 light 变体
- 所有颜色变更保证 WCAG AA 级对比度（4.5:1 正文，3:1 大字）
- 不引入新的 npm 依赖
- 保持现有 CSS 变量架构和命名体系
- 动效使用 CSS transition/animation，不引入 JS 动画库
- 改动需在 Chrome、Firefox、Safari 上验证
