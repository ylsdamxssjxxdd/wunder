## 1. 统计口径

- 本表只统计已经进入当前代码主线的内容，不把“仅在文档中提出”“仅口头约定”“未落代码的草图”计入完成度。
- “已完成”至少要满足以下之一：代码已落地、接口已可调用、测试已覆盖、文档已与代码对齐。
- “部分完成”表示主骨架已进入代码，但关键链路、持久化、多节点、压测或前端消费尚未闭环。
- “未开始”表示方案已明确，但仓库内还没有形成可复用的实现骨架。
- 完成度为工程估算值，用于排期和跟踪，不代表功能对外承诺。

## 6. 当前主要缺口

### 6.1 还缺真正的 Projection Plane

- 缺 session room
- 缺 user_world room
- 缺 projection publisher
- 缺 projection snapshot store
- 缺统一 replay / delta / resync 语义

### 6.2 还缺真正的多节点分布式能力

- lease 未持久化
- owner 抢占与 fencing 未形成数据库级语义
- gateway 未按 `thread / mission / projection` 做跨节点切流

### 6.3 还缺前端统一消费层

- 没有统一的 thread runtime view
- 没有统一的 mission runtime view
- 没有统一的 projection room runtime
- chat / beeroom / user_world 仍各自维护局部状态机

### 6.4 还缺压测和容量线

- hot thread
- hot mission
- high fanout projection
- reconnect storm
- slow client backlog
- channel ingress burst

## 8. 跟踪规则

- 每推进一个 R 阶段或一个关键骨架，需要同步更新本表。
- 每新增“已完成”项，应同时补充代码证据或测试证据。
- 若出现“方案已改口但代码未更新”，以代码现状为准，先将本表标为“部分完成”或“未开始”。
- 若某一阶段进入稳定切流或清理旧链路，需要把状态从“部分完成”升级为“已完成”。
