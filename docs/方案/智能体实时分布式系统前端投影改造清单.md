# 智能体实时分布式系统前端投影改造清单

## 1. 当前前端代码判断

- [chat.ts](C:/Users/sjxx/Desktop/wunder/frontend/src/stores/chat.ts) 体量很大，已包含 socket、history、workflow、feedback、subagent 等多种职责，不适合继续扩张。
- [beeroom.ts](C:/Users/sjxx/Desktop/wunder/frontend/src/stores/beeroom.ts) 已支持 `applyRealtimeEvent`，但仍依赖 `loadGroups/loadActiveGroup` 和静默刷新兜底。
- [userWorld.ts](C:/Users/sjxx/Desktop/wunder/frontend/src/stores/userWorld.ts) 自带 transport 选择、watch runtime 和会话状态，职责过重。
- [MessengerView.vue](C:/Users/sjxx/Desktop/wunder/frontend/src/views/MessengerView.vue) 已非常庞大，不能再继续堆 runtime 逻辑。

## 2. 前端改造目标

- 将 thread/mission/projection 的运行时从 store 中拆出。
- store 只负责视图态和投影缓存，不直接承担完整 realtime 协议机。
- 页面 steady state 依赖 snapshot + delta，而不是整包 refresh 修正状态。
