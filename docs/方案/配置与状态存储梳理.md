## 1. 配置分层与生效顺序

### 1.1 Base 配置

- 默认读取：`config/wunder.yaml`
- 可通过 `WUNDER_CONFIG_PATH` 指定
- 若目标文件不存在且文件名为 `wunder.yaml|wunder.yml`，会自动回退同目录 `wunder-example.yaml|wunder-example.yml`
- 支持 `.yaml/.yml` 互换查找

### 1.2 Override 配置

- 默认路径：`data/config/wunder.override.yaml`
- 可通过 `WUNDER_CONFIG_OVERRIDE_PATH` 指定
- `ConfigStore` 写入时采用“base vs updated 的差异”落盘
- 若差异为空会删除 override 文件

### 1.3 合并与插值规则

- 合并顺序：`base -> override -> 环境变量插值`
- 支持 `${ENV}` 与 `${ENV:-default}`
- 空字符串与 `null` 不会覆盖已有值（避免把 base 清空）

### 1.4 运行时环境变量兜底

以下变量在运行时直接覆盖配置字段：

- `WUNDER_HOST` / `WUNDER_PORT` / `WUNDER_SERVER_MODE`
- 鉴权相关：
  - `WUNDER_API_KEY`
  - `WUNDER_EXTERNAL_AUTH_KEY`
  - `WUNDER_EXTERNAL_EMBED_PRESET_AGENT_NAME`
  - `WUNDER_EXTERNAL_EMBED_JWT_SECRET`
  - `WUNDER_EXTERNAL_EMBED_JWT_USER_ID_CLAIM`

## 2. 三种运行形态：配置与状态落盘

### 2.1 Server（云端服务）

固定配置（随仓库/镜像发布）：

- `config/wunder.yaml`
- `config/wunder-example.yaml`
- `config/i18n.messages.json`
- `prompts/**`（default prompt pack）

运行期可写：

- override：`data/config/wunder.override.yaml`
- 管理端 prompt packs：`data/prompt_templates/<pack_id>/prompts/**`
- 用户工具配置：`data/user_tools/<safe_user_id>/config.json`
- 用户工作区：`workspace.root`（默认 `${WUNDER_WORKSPACE_ROOT:-./workspaces}`）

存储后端：

- 由 `storage.backend` 决定，`auto` 当前走 PostgreSQL
- DSN：`storage.postgres.dsn`（可由 `WUNDER_POSTGRES_DSN` 注入）

### 2.2 CLI（本地命令行）

默认目录：

- `temp_root`：`<launch_dir>/WUNDER_TEMP`（可通过启动参数覆盖）
- `WUNDER_HOME`：优先环境变量，否则 `~/.wunder`（无 HOME 则 `<launch_dir>/.wunder`）

运行时自动设置：

- `WUNDER_CONFIG_PATH`
- `WUNDER_CONFIG_OVERRIDE_PATH`
- `WUNDER_I18N_MESSAGES_PATH`（存在时）
- `WUNDER_PROMPTS_ROOT`（未设置时自动指向 repo root）
- `WUNDER_SKILL_RUNNER_PATH`（存在时）
- `WUNDER_HOME`
- `WUNDER_USER_TOOLS_ROOT`（`<wunder_home>/user_tools`）
- `WUNDER_VECTOR_KNOWLEDGE_ROOT`（`<wunder_home>/vector_knowledge`）
- `WUNDER_WORKSPACE_SINGLE_ROOT=1`

CLI 默认会强制：

- `server.mode=cli`
- `storage.backend=sqlite`
- `storage.db_path=<temp_root>/wunder_cli.sqlite3`
- `workspace.root=<launch_dir>`

### 2.3 Desktop（本地桌面）

默认目录（按 app_dir）：

- `temp_root`：`<app_dir>/WUNDER_TEMPD`
- `workspace_root`：`<app_dir>/WUNDER_WORK`
- `settings`：`<temp_root>/config/desktop.settings.json`

运行时自动设置：

- `WUNDER_CONFIG_PATH`
- `WUNDER_CONFIG_OVERRIDE_PATH`
- `WUNDER_I18N_MESSAGES_PATH`（存在时）
- `WUNDER_PROMPTS_ROOT`（未设置时自动指向 repo root）
- `WUNDER_SKILL_RUNNER_PATH`（存在时）
- `WUNDER_USER_TOOLS_ROOT`（`<temp_root>/user_tools`）
- `WUNDER_ADMIN_CUSTOM_SKILLS_ROOT`（`<temp_root>/admin_skills`）
- `WUNDER_BUILTIN_SKILLS_ROOT`（`<repo_root>/skills`）
- `WUNDER_VECTOR_KNOWLEDGE_ROOT`（`<temp_root>/vector_knowledge`）
- `WUNDER_DESKTOP_SETTINGS_PATH`
- `WUNDER_DESKTOP_APP_DIR`
- `WUNDER_DESKTOP_DEFAULT_WORKSPACE_ROOT`
- `WUNDER_DESKTOP_USER_ID`
- `WUNDER_WORKSPACE_SINGLE_ROOT=1`

Desktop 默认会强制：

- `server.mode=desktop`
- `storage.backend=sqlite`
- `storage.db_path=<temp_root>/wunder_desktop.sqlite3`
- `workspace.root=<workspace_root>`
- `channels.enabled=true`（保留本地渠道能力）
- `cron.enabled=true`
- `gateway.enabled=false`
- `agent_queue.enabled=false`
- `sandbox.mode=local`

Desktop override 自动播种（首次启动 `override` 不存在时）来源顺序：

1. `WUNDER_DESKTOP_PRECONFIG_PATH`
2. AppImage 同目录候选文件（`wunder.override.yaml|yml`、`预配置文件.yml|yaml`）
3. `app_dir` 同名候选文件
4. repo 内置候选：
   - `config/wunder.desktop.preconfig.yaml`
   - `预配置文件.yml`
   - `docs/分发/预配置文件.yml`

## 4. 活动配置与可写状态（运行期）

### 4.1 系统级活动配置

- override 文件（server/cli/desktop 各自路径）
- prompt pack 内容（`data/prompt_templates/**`）
- Desktop 本地设置（`desktop.settings.json`）

### 4.2 用户级活动配置

- 用户工具：`WUNDER_USER_TOOLS_ROOT/<safe_user_id>/config.json`
- 用户技能目录：`.../<safe_user_id>/skills/**`
- 用户知识库目录：`.../<safe_user_id>/knowledge/**`
- 用户模板包：
  - 根：`<prompts_root>/data/user_prompt_templates/<safe_user_id>/`
  - 包：`packs/<pack_id>/prompts/{zh|en}/system/*.txt`
  - 激活状态：`settings.json`

### 4.3 文件产物与工作区

- 业务工作区：`workspace.root/<safe_user_id>/**`
- 向量知识迁移目录（兼容旧数据）：`WUNDER_VECTOR_KNOWLEDGE_ROOT`
- 临时目录：`WUNDER_TEMP_DIR_ROOT`（默认 `<cwd>/temp_dir`）

## 7. Source of Truth（当前建议）

- 声明式配置（Git 管理）：`config/**`、`prompts/**`、compose 文件
- 运行期覆盖配置：override + prompt packs + user_tools
- 业务权威：数据库（表结构见 `docs/数据库设计.md`）
- 客户端本地偏好：Desktop settings 与浏览器 localStorage

> 团队约定强调 `data/` 不作为长期可靠归档。若用于生产，请将 `data/`、`workspace`、数据库目录映射到持久卷，或用环境变量改到专用持久路径。

## 8. 备份与迁移最小建议

- 配置快照：`config/wunder.yaml` + override + prompt packs + user_tools
- 数据快照：数据库逻辑备份（PostgreSQL 建议 `pg_dump`）
- 文件快照：`workspace.root` 与必要附件目录
- Docker 部署可直接使用：
  - `scripts/backup_persist_volume.ps1`
  - `scripts/restore_persist_volume.ps1`
