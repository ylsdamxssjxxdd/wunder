# 自我状态工具落地方案

## 1. 目标与范围

### 1.1 目标
- 新增内置工具：`自我状态`（英文别名：`self_status`）。
- 允许模型在会话内查询自身运行状态，包含：
  - 当前会话最近一次实际模型调用信息（provider/model/base_url/stream）。
  - 当前上下文占用（强调“占用量”，不是账单总消耗）。
  - 线程（会话）轮次、工具调用、审批等待、线程运行态统计。
- 工具为只读，不产生业务副作用。
- **工具可被开放（可选），但默认不启用**。

### 1.2 非目标
- 不扩展为跨用户/跨会话观测工具。
- 不改变线程 system prompt 冻结策略。
- 不新增数据库表，优先复用现有 monitor/workspace/orchestrator 运行态数据。

---

## 2. 关键语义澄清（本次修订重点）

## 2.1 desktop 的“开放”与“启用”是两层语义
- `collect_available_tool_names` 在 desktop 模式下返回全量 builtin，表示“可开放/可选能力集合”。
- 真正“本轮启用的工具集合”在实际聊天链路中由以下步骤决定：
  - 基础 allowed：`compute_allowed_tool_names(...)`。
  - 再按会话/智能体覆盖收敛：`apply_tool_overrides(...)`。
  - 最终 `tool_names` 才进入 orchestrator 执行。

结论：
- desktop 下“全开放”并不等于“默认全启用”。
- 你的要求应按“默认选择集不包含 self_status”实现，而不是改掉 desktop 全开放机制。

## 2.2 默认不启用的正确口径
- `self_status` 需要做到：
  - 在可选能力里可见（可开放）。
  - 默认选择集里不出现（默认不启用）。
- 显式启用后可进入 `tool_names` 并实际可调用。

## 3. 现状评估（代码基线）

## 3.1 工具注册与执行链路
- `src/services/tools/catalog.rs`
  - `builtin_tool_specs_with_language`
  - `builtin_aliases`
  - `collect_available_tool_names`
- `src/services/tools/dispatch.rs`
  - `execute_builtin_tool` 的内置工具分发。
- `src/services/tools/context.rs`
  - `ToolContext` 已具备 `monitor/workspace/orchestrator/storage` 访问能力。

## 3.2 “默认启用集”来源链路
- 聊天入口会生成最终 `tool_names`（`src/api/chat.rs`）：
  - `compute_allowed_tool_names`（可用集）
  - `resolve_session_tool_overrides / resolve_agent_tool_defaults`
  - `apply_tool_overrides`
  - `finalize_tool_names`
- default agent 的初始工具集来自 `curated_default_tool_names`（`src/services/default_tool_profile.rs`，并在 default_agent_sync / inner_visible 构建默认配置时使用）。

## 3.3 可复用数据源
- `src/ops/monitor.rs`
  - `SessionRecord`：`user_rounds/context_tokens/context_tokens_peak/events`
  - `monitor.get_record(session_id)` 可拿完整记录。
  - 事件含 `llm_request/token_usage/context_usage/round_usage/thread_status/approval_*`。
- `src/services/workspace.rs`
  - `load_session_context_tokens(_async)`
  - `load_session_context_overflow(_async)`
- `src/orchestrator/thread_runtime.rs` 与 `src/orchestrator/turn_state.rs`
  - 有实时线程状态与 pending approval 信息，但当前为 `pub(super)`，需 orchestrator 暴露只读接口给工具层使用。

## 4. 默认不启用策略（最终决策）

## 4.1 决策
不改 desktop “全开放”机制；通过“默认选择集不包含 self_status”实现默认不启用。

## 4.2 落地规则
1. 将 `self_status` 加入 builtin spec 与 alias（可见、可选）。
2. 不加入任何默认启用清单：
- 不加入 `config/wunder.yaml` 与 `config/wunder-example.yaml` 的 `tools.builtin.enabled`。
- 不加入 `src/services/default_tool_profile.rs` 的 `DEFAULT_BUILTIN_TOOL_NAMES`。
3. 保持覆盖链路收敛逻辑不变：
- desktop 下默认由 default agent / session 的 `tool_overrides` 决定是否启用。
- API/server 下仍需显式进 `tools.builtin.enabled` 才可进入 allowed。

## 4.3 显式启用方式
- 管理端/用户端把 `self_status` 加到智能体工具选择（`tool_overrides` / declared 工具）。
- 或在非 desktop 配置中加入 `tools.builtin.enabled`。

## 5. 工具协议设计

```json

  "type": "object",
  "properties": {
    "detail_level": {
      "type": "string",

    "include_events": {
      "type": "boolean"
```

## 7. 聚合逻辑细节

## 7.1 模型信息
- 主来源：最近 `llm_request` 事件。
- 字段：`provider/model/base_url/stream`。

## 7.2 上下文占用
- 优先级：
  1. 最近 `token_usage.input_tokens`
  2. 最近 `context_usage.context_tokens`
  3. `workspace.load_session_context_tokens`
- `peak_tokens` 取事件峰值与 `session.context_tokens_peak` 的较大值。
- `overflow_flag`：`workspace.load_session_context_overflow`。

## 7.3 线程运行态
- 从 orchestrator 新只读接口读取：
  - `status/subscriber_count/active_turn_id`
  - `waiting_for_user_input/pending_approval_ids`
- orchestrator 不可用时降级用 monitor 记录，并写 warning。

## 7.4 轮次与调用统计
- `user_rounds`：`SessionRecord.user_rounds`
- `model_round_current_user`：当前 user_round 的最大 model_round
- `observed_model_round_total`：按 `(user_round, model_round)` 去重计数
- `tool_calls_*`：`tool_call` 事件计数
- `approvals_*`：`approval_request/approval_result/approval_resolved` 计数

## 10. 风险与回滚

## 10.1 风险
- monitor 事件窗口受 `event_limit` 影响，历史统计属于“观测窗口统计”。
- 若 `monitor_drop_event_types` 配置不当，部分字段会为空。
- 若某些入口绕过 `tool_overrides`，可能出现 desktop 下工具默认启用扩散，需要回归确认。

## 10.2 回滚
- 立即从选择集移除 `self_status`（配置/智能体覆盖）。
- 下线分发入口（catalog/dispatch 接线回退）。
- 无数据库 schema 变更，回滚成本低。

---

## 11. 验收标准
- `self_status` 在 desktop 中“可选可见”，但默认会话 `tool_names` 不包含它。
- 显式启用后可调用，并返回模型信息、上下文占用、线程统计。
- 输出口径遵循“上下文占用量”定义，不误导为总消耗。
- 工具执行只读、稳定、低开销。
