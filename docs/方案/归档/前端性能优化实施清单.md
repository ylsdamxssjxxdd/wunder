# 前端性能优化实施清单

## 目标

- 让用户侧前端更快出现可交互骨架，减少首次进入时的白屏/长时间 loading。
- 让智能体设置页优先显示基础信息，并尽快补齐工具信息、模型信息等次级数据。
- 降低消息器中的非必要请求、重复请求和大范围重渲染，提升主观流畅度。

## 本轮已开始实施

- [x] 收窄 Messenger bootstrap 关键路径。
  - 非消息页不再强制等待会话列表。
  - 非消息/智能体/蜂群/文件页不再强制等待智能体列表。
- [x] 为用户工具相关数据增加前端单飞缓存。
  - 缓存 `user_tools/catalog`
  - 缓存 `user_tools/tools`
  - 缓存 `user_tools/skills`
  - 工具或技能发生更新时主动失效缓存
- [x] 按页面需求预热工具数据。
  - 进入智能体设置页时预热工具目录
  - 进入右侧技能区时预热工具摘要和技能目录
  - 不再在 Messenger 全局挂载时无条件强拉这些数据
- [x] 拆开智能体设置页的串行阻塞链路。
  - Agent 详情优先加载
  - 工具目录异步补齐
  - 模型列表异步补齐
  - 蜂群分组异步补齐
- [x] 优化窄屏中栏 overlay 的首次交互成本。
  - 增加 hover intent，避免鼠标扫过立即触发整块刷新
  - 中栏改为隐藏不立即销毁，首次打开后复用已挂载内容
  - 在 overlay 模式下做轻量预热，尽量把首轮挂载成本挪到空闲时段
- [x] 将消息区工作流布局刷新改为定点测量。
  - `layout-change` 携带消息 key
  - 虚拟列表优先只重测受影响消息
  - 流式内容更新优先只重测最后一条消息
- [x] 为会话事件快照增加单飞复用与短 TTL 缓存。
  - `preloadSessionDetail` 与 `loadSessionDetail` 优先复用同一份 `getSessionEvents` 结果
  - watchdog 避免在前台详情加载期间重复补拉会话事件快照
  - 发送/恢复/停止流以及 watch 收到新事件时主动失效事件缓存
  - 前台打开线程时若后台预热仍在进行，则直接承接该次预热结果，避免马上重发 `getSession`
  - 预热刚结束时，短时复用会话详情快照与事件快照，减少 hover 后点击线程的二次详情请求
- [x] 为会话详情消息水合增加版本复用快路径。
  - 预热后前台打开同一线程时，若服务端消息指纹与事件游标未变化，则直接复用已水合消息
  - 后台预热在无增量变化时不再重复执行 `attachWorkflowEvents + hydrateMessage`
  - 活跃流式会话仍保持保守策略，避免误复用旧运行态消息
- [x] 收窄时间线 preview 预热的后台请求成本。
  - 右侧时间线 preview 优先复用会话列表已有 `last_user_message_preview/summary`
  - 缺少 preview 时改为轻量 `getSession` 抽取最近用户消息，不再为 hover 预热触发 `/events`
  - 避免后台 preview 预热与主线程输入竞争，减少 hover 后点击线程前的额外长任务

## 下一批高收益项

- [ ] 继续压缩右侧边缘 hover 与消息区高频滚动路径。
  - 进一步减少主线程上与指针移动相关的布局读取
  - 评估消息高度观测是否值得从轮询测量切到 `ResizeObserver`
- [ ] 继续收缩消息区布局测量成本。
  - 复核剩余 `offsetHeight/getBoundingClientRect/scrollHeight` 的读取频率
  - 继续减少消息流式刷新时的连锁视口更新
- [ ] 降低会话事件整包回放的主线程回调成本。
  - 继续削减 `rounds` 挂载与 `attachWorkflowEvents` 的同步回放开销
  - 评估会话详情接口是否补充轻量摘要/版本号，减少前台打开时的整包拼装
- [ ] 进一步拆分 MessengerView 巨型组件。
  - 收敛顶层 `computed/watch`
  - 把智能体设置、消息列表、右侧停靠栏的数据加载职责继续下沉

## 验证关注点

- 首次进入 `/chat?section=agents` 时，设置页基础字段是否先于工具目录出现。
- 智能体设置页切换不同 agent 时，名称/描述/提示词是否先出现，再补齐工具组和模型列表。
- 从消息页切到智能体设置页时，`user_tools/catalog` 是否减少重复请求。
- 右侧技能区首次展开后再次展开时，是否命中缓存并减少等待。
- 鼠标预热会话后立即打开线程时，`getSessionEvents` 是否不再出现成对重复请求。
- 鼠标预热会话后立即打开线程时，主线程中 `attachWorkflowEvents/createWorkflowProcessor` 的同步耗时是否明显下降。
- 右侧时间线展开或 hover 后，Network 中是否不再为 preview 预热触发 `/chat/sessions/:id/events`。
