# XMPP 文件传输优化方案

## 1. 背景

当前 Wunder 的微信渠道已经具备比较完整的文件闭环：

- 入站可解析平台文件消息
- 文件可下载、解密并落到工作区
- 文件可进入 `pending_files` 与后续模型处理链路
- 出站可把本地/远程/data 附件上传到平台 CDN 后回发

XMPP 渠道目前没有做到同等能力。现状更接近“URL 型附件透传”，而不是“真正托底的文件传输”。

## 2. 当前现状

### 2.1 已有能力

- 支持解析标准 XMPP `jabber:x:oob` 与 `urn:xmpp:reference:0` 中的 `http(s)` 链接。
- 已接入通用远程下载落工作区逻辑，OOB 链接进入后可以被拉取到工作区。
- 长连接收发、MUC、心跳、联系人列表等基础运行态已经具备。

### 2.2 主要缺口

- 自定义 XMPP `file_transfer` 消息当前只写入 `meta`，没有转成 `attachments`。
- 出站不会自动把正文里的工作区文件链接提取成 XMPP OOB/reference 附件。
- 非 `http(s)` 的本地文件、`data:` 附件无法直接出站。
- 文件公网下载地址依赖 `public_base_url` 推导，长连接场景容易退回到本机地址。
- 远程下载链路缺少更严格的 timeout、地址过滤与来源约束。

## 3. 目标

首期目标不是一次做完整 XMPP 文件协议栈，而是先把最核心的业务闭环补齐：

1. 自定义 XMPP 文件消息进入 Wunder 后，必须作为真正附件进入统一链路。
2. Wunder 产出的工作区文件，至少能稳定转换成可被 XMPP 客户端点击下载的外链。
3. XMPP 文件链路的行为要尽量对齐微信、飞书等既有渠道，减少特殊分支。

## 4. 分阶段落地

### Phase 1：补齐入站闭环

范围：

- 将自定义 `file_transfer` 结构映射为 `ChannelAttachment`
- 附件进入 `download_remote_attachments_to_workspace`
- 文件进入 `pending_files`
- 为 XMPP 自定义文件消息补回归测试

预期结果：

- 用户从支持该格式的 XMPP 客户端发文件后，Wunder 可以像处理微信文件一样先收下文件
- 文件消息在没有额外文本时，进入“缓冲待处理”而不是错误触发普通文本对话

### Phase 2：补齐出站文件链接

范围：

- 为 XMPP 增加正文链接提取逻辑
- 将 `/workspaces/...` 与 `temp_dir` 链接转换为 OOB/reference 节点
- 优先保证 Wunder 自己产出的文件可稳定回发

预期结果：

- 模型生成的工作区文件或工具产出的下载链接，可以被 XMPP 客户端直接点击

### Phase 3：增强安全与稳定性

范围：

- 为远程下载增加超时、地址过滤、私网拦截
- 支持更明确的 `public_base_url` 配置
- 为下载失败、链接不可达、超限等情况补 runtime log

### Phase 4：评估协议级文件上传

范围：

- 评估 XEP-0363 HTTP Upload
- 如目标客户端生态允许，再考虑真正的 XMPP 原生文件上传/发送

说明：

- 这一阶段不是首批必须项
- 在当前 Wunder 场景里，稳定的公网下载链接通常比协议级上传更务实

## 5. 当前落地进展

当前已完成：

- 新增本方案文档
- 将自定义 `file_transfer` 消息转成真正附件
- 让该附件直接复用现有远程下载落工作区逻辑
- 补充解析层测试，确保文件消息不再只停留在 `meta`

当前继续推进：

- 为 XMPP 出站补充正文中的工作区文件与 `temp_dir` 下载链接提取
- 在生成 OOB/reference 时清理正文里的附件型 Markdown/裸链接，减少重复展示

## 6. 验收标准

- 自定义 XMPP 文件消息解析后，`attachments.len() > 0`
- 文件型消息在无正文时，`message_type` 为 `file`
- 附件带有可下载 URL、文件名、大小与合理的 `kind`
- 进入 ChannelHub 后能被现有 `pending_files` 机制识别

## 7. 风险说明

- 首期仍以“链接型文件”作为核心传输手段，不等同于完整 XMPP 文件协议支持
- 若下载链接只能在内网或本机访问，外部客户端仍会出现“看到附件但打不开”
- 若对端链接策略不稳定，仍可能出现下载失败，需要在后续阶段补 runtime 可观测性
