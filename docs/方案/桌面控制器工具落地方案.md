# 桌面控制器工具落地方案

## 1. 背景与目标
- 参照 `C:\Users\32138\Desktop\eva` 的实现，引入“桌面控制器（controller）”与“桌面监视器（monitor）”两类内置工具。
- 目标：让模型在桌面模式下安全、可控地完成“截图→定位→单步操作→再截图”的交互闭环；监视器用于等待 UI 状态变化并提供新截图。

## 2. 工具清单与职责划分
### 2.1 桌面控制器（controller）
- 职责：执行一次原子级桌面操作（点击/拖拽/滚动/键盘/输入/延时/移动），并返回最新截图与坐标空间信息。
- 关键约束：
  - 一次调用只做一个动作；多步需多轮调用。
  - `bbox` 使用归一化坐标系（配置 `norm_width` × `norm_height`），原点为左上角。
  - `type_text` 会先单击聚焦再输入；禁止点击/输入到聊天输入框。

### 2.2 桌面监视器（monitor）
- 职责：等待 `wait_ms` 毫秒后刷新截图，帮助模型判断 UI 是否进入目标状态。
- 使用方式：若未就绪，继续调用 monitor（可退避）；就绪后转用 controller。
- 与 controller 共享同一归一化坐标系，避免切换时坐标混淆（参照 EVA 设计）。

## 3. 关键流程节点（端到端）
1. **工具注册**
   - `src/services/tools.rs` 注册内置工具规格、别名、权限判断与路由。
   - 中文名、英文别名、短别名统一（controller/monitor）。
2. **请求解析**
   - `src/services/tools/desktop_control.rs` 解析 `bbox/action/...` 参数，统一坐标格式。
3. **动作执行**
   - 调用 OS 输入/鼠标 API 执行原子动作。
4. **截图获取**
   - 捕获屏幕 → 归一化缩放 → PNG 编码 → 限制大小（MAX_SCREENSHOT_BYTES）。
5. **结果返回**
   - 返回 `screenshot_path/screenshot_download_url/normalized_width/normalized_height/...`。
6. **跟随消息（可选）**
   - `build_followup_user_message` 把截图转为 data URL 回灌模型（保持与 EVA 的“观察回路”一致）。
7. **持久化落盘**
   - 额外将截图写入用户私有容器（container=0），方便后续追踪与审计（但不渲染到聊天 UI）。

## 4. 参数与返回
### 4.1 controller 入参（核心字段）
- `bbox`：`[x1,y1,x2,y2]` 或 `[cx,cy]`（归一化坐标）。
- `action`：`left_click/left_double_click/right_click/middle_click/left_hold/right_hold/middle_hold/left_release/right_release/middle_release/scroll_down/scroll_up/press_key/type_text/delay/move_mouse/drag_drop`。
- `description`：动作意图说明（用于日志/可追溯）。
- `key/text/delay_ms/duration_ms/scroll_steps/to_bbox`：动作补充参数。

### 4.2 monitor 入参
- `wait_ms`：等待时间（0~30000）。
- `note`：可选说明。

### 4.3 返回（共性）
- `normalized_width/normalized_height`、`screen_width/screen_height`。
- `screenshot_path`（临时路径）
- `screenshot_download_url`（临时下载）
- `screenshot_bytes`（大小）
- `followup_prompt`（建议下一步提示）

## 5. 配置与约束
- `server.mode=desktop` 且 `tools.desktop_controller.enabled=true` 才允许调用。
- `capture_timeout_ms`、`max_frames` 控制截图超时与保留帧数量。
- `MAX_MONITOR_WAIT_MS`、`MAX_SCREENSHOT_BYTES` 保护性能与稳定性。

## 6. 持久化方案（新增）
- **目标**：截图写入用户私有容器（container=0），不进入聊天渲染。
- **落盘路径建议**：`desktop_controller/{session_id}/desktop_shot_<uuid>.png`。
- **实现节点**：
  - 在 `tool_desktop_controller` 与 `tool_desktop_monitor` 捕获截图后执行复制。
  - 使用 `WorkspaceManager.scoped_user_id_by_container(user_id, 0)` 解析用户私有容器根。
  - 遇到失败仅记录日志，不影响工具正常返回。

## 7. 安全与交互约束
- 仅在用户明确授权或明确请求时调用（沿用 EVA 的“高风险工具”提示原则）。
- 禁止自动点击/输入到聊天输入框、系统敏感区域。
- 单步操作，确保可回滚与可追踪。

## 8. 测试与验收
- **单测**：bbox 解析/坐标映射/边界裁剪/动作枚举。
- **集成**：
  - controller 执行 + 截图返回。
  - monitor 等待 + 截图返回。
  - 截图落盘到用户私有容器。
- **验收**：
  - 触发 controller/monitor 均可返回截图信息。
  - 私有容器中可看到 `desktop_controller/` 目录与图片文件。
  - 聊天 UI 不渲染上述截图（仅供系统内部引用）。

## 9. 与 EVA 的对齐点
- monitor 负责“等待 + 返回截图”，配合 controller 完成“观察→操作”闭环。
- 归一化坐标系保持一致（避免 monitor→controller 切换时坐标错位）。
- 工具提示中强调“未就绪则继续 monitor，就绪后 controller”。
