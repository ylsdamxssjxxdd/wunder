# wunder 测试大纲总览

## 1. 文档定位

- 本文档是测试总入口，不再承载全部测试细节。
- 测试大纲已拆分为两份：
  - 手工测试大纲：[`docs/测试大纲-手工.md`](./测试大纲-手工.md)
  - 自动化测试大纲：[`docs/测试大纲-自动化.md`](./测试大纲-自动化.md)

## 2. 适用范围

- 运行形态：`server`、`cli`、`desktop`。
- 前端表面：用户侧前端（`frontend`）、管理员侧前端（`web` + 管理员 Vue 页面）。
- 后端能力：会话与编排、工具生态、渠道、知识库、监控与评测、工作区、鉴权。

## 3. 覆盖原则

- 功能完整：覆盖主路径、关键分支、失败与恢复路径。
- 隐性链路：覆盖“默认看不到但影响一致性”的机制，如提示词冻结、记忆注入时机、token 口径、队列重试。
- 治理能力：管理员侧能力与权限边界单独成块覆盖。
- 稳定与性能：手工和自动化均需覆盖 Soak、吞吐、恢复时间、资源曲线。

## 4. 使用建议

- 发版前：先执行手工大纲中的 `Smoke` 与 `P0` 清单，再执行自动化 `PR Gate + Release Gate`。
- 缺陷回归：先补自动化，再按手工大纲执行受影响域的最小回归集。
- 版本升级：按两份大纲同步更新，避免“手工和自动化覆盖偏移”。
