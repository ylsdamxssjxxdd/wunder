# 桌面控制器工具落地方案

## 1. 目标与边界
- 目标：在 **desktop 本地模式**提供“桌面控制器/桌面监视器”内置工具，参考 EVA 的 bbox+action 协议，实现可控、可审计的桌面操作闭环。
- 边界：只在 `server.mode=desktop` 且 `tools.desktop_controller.enabled=true` 时可用；非桌面模式默认不可用。
- 平台优先级：先支持 Windows（GDI 截图 + SendInput 输入），其他平台返回明确错误并留好扩展位。

## 2. 工具定义
### 2.1 桌面控制器（桌面控制器）
**用途**：执行单个原子动作（点击/拖拽/输入/滚动等），并在动作完成后返回新截图。  
**核心约束**：
- `bbox` 必填，使用“归一化截图坐标”：
  - 格式 `[x1,y1,x2,y2]` 或 `[cx,cy]`
  - 坐标系默认 `1000x1000`（可配置）
- `action`、`description` 必填
- 一次调用=一个原子动作；多步操作通过多轮调用完成

**Schema（精简）**：
```
{
  bbox: [x1,y1,x2,y2] | [cx,cy],
  action: "left_click|left_double_click|right_click|middle_click|left_hold|right_hold|middle_hold|left_release|right_release|middle_release|scroll_down|scroll_up|press_key|type_text|delay|move_mouse|drag_drop",
  description: string,
  key?: string,
  text?: string,
  delay_ms?: number,
  duration_ms?: number,
  scroll_steps?: number,
  to_bbox?: [x1,y1,x2,y2] | [cx,cy]
}
```

### 2.2 桌面监视器（桌面监视器）
**用途**：等待 UI 变化并获取新截图，用于“等加载/等弹窗/等按钮可点”等场景。  
**Schema（精简）**：
```
{
  wait_ms: 0..30000,
  note?: string
}
```
**约束**：每次调用仅等待一次并返回截图，必要时由模型多轮调用。

## 3. 数据流与执行节点
### 3.1 调度链路
1. 用户请求进入 Orchestrator。
2. 模型调用工具：
   - controller：执行动作 → 截图 → 工具结果
   - monitor：等待 → 截图 → 工具结果
3. Orchestrator 在工具结果后 **自动注入 image_url**：
   - 读取工具返回的 `screenshot_path`
   - 转换为 `data:image/png;base64,...`
   - 作为新一条 user 消息进入下一轮模型上下文

### 3.2 截图与坐标归一化
- 实际截图先使用 **屏幕物理像素**抓取（Win32 GDI/BitBlt）。
- 输出图像缩放到 `norm_width x norm_height`（默认 1000x1000）。
- bbox 解析统一在该归一化空间中完成，工具内部用 `mapCoord` 映射回真实屏幕坐标。

## 4. 模块与代码落点
### 4.1 新增模块
- `src/services/tools/desktop_control.rs`
  - 参数解析：bbox/动作/输入等
  - 屏幕截图：GDI/BitBlt → RGBA → PNG
  - 鼠标键盘输入：SendInput
  - 返回截图路径与归一化尺寸

### 4.2 调度改造
- `src/orchestrator/execute.rs`
  - 在 tool_result 后自动补充截图附件（image_url）
- `src/orchestrator/mod.rs`
  - 引入 `build_desktop_followup_user_message`

### 4.3 工具注册与别名
- `src/services/tools.rs`
  - `builtin_tool_specs_with_language` 增加两套 schema
  - `builtin_aliases` 增加 `desktop_controller/controller`、`desktop_monitor/monitor`
  - `execute_builtin_tool` 新增两工具调用分支
  - `filter_tool_names_by_model_capability` 按 `support_vision` 过滤

### 4.4 配置
- `src/core/config.rs`
  - `tools.desktop_controller` 新配置：
    - `enabled`：是否启用
    - `norm_width` / `norm_height`：归一化尺寸
    - `max_frames`：截图保留数量
    - `capture_timeout_ms`：截图超时
- `wunder-desktop/runtime.rs`
  - desktop 模式默认启用工具与内置配置开关

## 5. 安全与审批
- 新增审批类型：`ApprovalRequestKind::Control`
  - 控制器/监视器与执行命令一样属于高风险操作
  - `approval_mode=suggest/auto_edit` 时强制审批
- 工具结果不直接写入本地文件系统，仅存放 `temp_dir/desktop_controller/*` 并自动清理旧帧。

## 6. 运行时行为
- 动作执行后才截图，确保模型看到最新状态。
- `type_text` 会先执行一次左键点击聚焦输入框，再注入文本。
- `drag_drop` 使用 `to_bbox` 指定目标，内部按平滑轨迹移动并释放。

## 7. 可观测与调试
- 工具结果包含：
  - `center_norm` / `center_screen`
  - `normalized_width` / `normalized_height`
  - `screenshot_path` / `screenshot_download_url`
- 允许运维通过 `/wunder/temp_dir/download` 拉取截图复盘。
