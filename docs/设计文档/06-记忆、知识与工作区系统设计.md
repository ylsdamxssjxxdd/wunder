# 记忆、知识与工作区系统设计

## 1. 设计目标

记忆、知识与工作区共同构成 wunder 的“资料现实层”。它们解决的不是模型会不会说，而是智能体是否能持续记住、可靠检索、真实操作和长期积累。

## 2. 三个子系统的定位

| 子系统 | 作用 | 主要目录 |
| --- | --- | --- |
| 长期记忆 | 记录用户或智能体跨线程可复用的长期信息 | `src/services/memory.rs` `src/services/memory_fragments.rs` `src/services/memory_auto_extract.rs` |
| 知识库 | 解析、缓存、检索文档内容并服务模型使用 | `src/services/knowledge.rs` `src/services/vector_knowledge.rs` |
| 工作区 | 承载真实文件、目录、附件和工具读写现实 | `src/services/workspace.rs` |

三者的区别必须清楚：

- 记忆是“应该记住什么”。
- 知识是“需要查什么资料”。
- 工作区是“真正要操作什么文件和资产”。

## 3. 核心约束

资料现实层长期必须遵守以下约束：

- 长期记忆只允许在线程初始化时以 snapshot 形式注入一次。
- 知识检索是按需召回，不允许把整库内容直接灌进上下文。
- 工作区不是临时附件目录，而是智能体实际执行环境的一部分。
- `data` 与 `temp_dir` 都不应被当作长期业务存储。
- 服务端以 PostgreSQL 为主存储，桌面端以 SQLite 为主存储，但文件现实和目录规则需要保持一致。

## 4. 长期记忆设计

长期记忆面向“跨线程但有归属”的信息沉淀，主要包括：

- 用户级记忆启停与配置。
- agent 级记忆作用域。
- 记忆摘要、片段与自动提取。
- 用于线程初始化的 `MemorySnapshot`。

设计重点有两条：

- 记忆系统属于 Wunder 产品能力。
- “一次性记忆注入”属于智能体内核语义。

也就是说，记忆记录、召回、排序、管理工具放在 Wunder；线程初始化时只拿到一个稳定 snapshot，不在中途不断追加。

## 5. 知识库设计

知识库用于把结构化或半结构化资料变成可检索的知识单元，当前重点包括：

- Markdown 等文档解析与章节切分。
- 缓存知识 section、document、preview。
- 基于查询构建 candidate sections，并结合 LLM 或规则做进一步筛选。
- 支持向量检索能力扩展，但不把向量检索当作唯一检索手段。

知识库设计应优先保证：

- 文档可增量刷新。
- 检索结果可解释。
- 与工作区和工具系统协同，而不是孤立存在。

## 6. 工作区设计

工作区是智能体真实执行环境的一部分，不应被降格为“聊天附件容器”。

工作区主要承担：

- 文件读写与目录路由。
- 附件落地、转换和引用。
- 与工具系统、桌面控制、浏览器控制联动。
- 为 server、desktop、cli 统一提供现实世界资产入口。

工作区设计中要特别注意：

- 不把 `temp_dir` 当业务源数据。
- 文件容器与沙盒容器要有清晰的边界。
- 路径解析、路由和 fence 规则必须一致，避免越权访问。

## 7. 模块组成

| 模块 | 作用 |
| --- | --- |
| `src/services/memory.rs` | 记忆存储、摘要规范化、提示块构建 |
| `src/services/memory_auto_extract.rs` | 记忆自动提取 |
| `src/services/memory_fragments.rs` | 记忆片段管理（三层内容、语义字段、生命周期） |
| `src/services/knowledge.rs` | 知识解析、缓存、查询 |
| `src/services/vector_knowledge.rs` | 向量知识支持 |
| `src/services/workspace.rs` | 工作区服务 |

## 8. 与其他系统的关系

- 与智能体核心：资料现实层提供 snapshot、查询结果和文件现实，但不直接重写线程认知态。
- 与工具系统：文件、记忆管理、知识检索通常通过工具暴露给模型。
- 与存储系统：记忆和知识元信息落数据库，文件实体落工作区目录。
- 与前端：前端只展示和管理资料，不应绕开后端直接定义工作区状态。

## 9. 验收标准

- 长期记忆只在初始化时注入，后续轮次不会再次污染线程 prompt。
- 知识检索能够稳定输出候选结果和可解释摘要。
- 工作区、附件、目录和工具访问遵守统一边界。
- server、desktop、cli 的资料现实层语义一致，只允许存储后端不同。
