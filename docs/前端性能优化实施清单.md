# 前端性能优化实施清单

## 目标

- 让用户侧前端更快出现可交互骨架，减少首次进入时的白屏/长时间 loading。
- 让智能体设置页优先显示基础信息，并尽快补齐工具信息、模型信息等次级数据。
- 降低消息器中的非必要请求、重复请求和大范围重渲染，提升主观流畅度。

## 本轮已开始实施

- [x] 收窄 Messenger bootstrap 关键路径。
  - 非消息页不再强制等待会话列表。
  - 非消息/智能体/蜂群/文件页不再强制等待智能体列表。
- [x] 为用户工具相关数据增加前端单飞缓存。
  - 缓存 `user_tools/catalog`
  - 缓存 `user_tools/tools`
  - 缓存 `user_tools/skills`
  - 工具或技能发生更新时主动失效缓存
- [x] 按页面需求预热工具数据。
  - 进入智能体设置页时预热工具目录
  - 进入右侧技能区时预热工具摘要和技能目录
  - 不再在 Messenger 全局挂载时无条件强拉这些数据
- [x] 拆开智能体设置页的串行阻塞链路。
  - Agent 详情优先加载
  - 工具目录异步补齐
  - 模型列表异步补齐
  - 蜂群分组异步补齐

## 下一批高收益项

- [ ] 优化窄屏中栏 overlay 的悬停展开逻辑。
  - 避免 `mouseenter` 直接触发整块面板挂载
  - 改为延迟预览或常驻挂载后显隐
- [ ] 继续收缩消息区布局测量成本。
  - 降低 `offsetHeight/getBoundingClientRect/scrollHeight` 的读取频率
  - 减少 `workflow layout-change` 对整块消息视口刷新的连锁影响
- [ ] 降低会话事件整包回放的主线程回调成本。
  - 减少非必要 `getSessionEvents`
  - 活动会话优先走增量事件
- [ ] 进一步拆分 MessengerView 巨型组件。
  - 收敛顶层 `computed/watch`
  - 把智能体设置、消息列表、右侧停靠栏的数据加载职责继续下沉

## 验证关注点

- 首次进入 `/chat?section=agents` 时，设置页基础字段是否先于工具目录出现。
- 智能体设置页切换不同 agent 时，名称/描述/提示词是否先出现，再补齐工具组和模型列表。
- 从消息页切到智能体设置页时，`user_tools/catalog` 是否减少重复请求。
- 右侧技能区首次展开后再次展开时，是否命中缓存并减少等待。
