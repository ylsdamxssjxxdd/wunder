# 部署、启动与运行模式

## 1. 三种运行形态

| 形态 | 入口 | 说明 |
| --- | --- | --- |
| Server | `wunder-server` | 组织级核心，多租户、多用户、多渠道 |
| Desktop | `wunder-desktop` + bridge | 个人用户主推，本地即用 |
| CLI | `wunder-cli` | 脚本化、批处理、自动化 |

另有 `sandbox` 模式：`WUNDER_SERVER_MODE=sandbox`。

## 2. 入口文件

| 形态 | 入口 |
| --- | --- |
| Server | `src/main.rs` |
| CLI | `wunder-cli/main.rs` |
| Desktop | `desktop/tauri/main.rs` |
| Desktop Bridge | `desktop/tauri/bridge_main.rs` |

所有形态共用 `Cargo.toml` 中的 `wunder-server` crate，通过 feature flag 和 `AppRuntimeProfile` 区分。

## 3. 配置路径

| 配置 | 路径 |
| --- | --- |
| 默认配置 | `config/wunder.yaml` |
| 示例配置 | `config/wunder-example.yaml` |
| MCP 配置 | `config/mcp_config.json` |
| 自定义路径 | `WUNDER_CONFIG_PATH` 环境变量覆盖 |

## 4. Server 启动

```bash
# 编译并运行
cargo run --release --bin wunder-server

# 覆盖地址和模式
WUNDER_HOST=0.0.0.0 WUNDER_PORT=18000 WUNDER_SERVER_MODE=api \
  cargo run --release --bin wunder-server
```

默认 HTTP 端口以 `config/wunder.yaml` 为准，默认样例 `18000`。

## 5. 前端开发

```bash
# 用户侧前端开发
npm run dev --workspace wunder-frontend
# 默认端口 18001，代理 http://127.0.0.1:18000

# 生产构建
npm run build --workspace wunder-frontend
```

## 6. CLI 启动

```bash
cargo run --release --bin wunder-cli
```

CLI 启动时自动：
- 解析 repo root
- 生成本地运行配置副本
- 设置 `WUNDER_HOME`、`WUNDER_USER_TOOLS_ROOT` 等变量
- 将 `config.server.mode` 调整为 `cli`

## 7. Desktop 启动

```bash
# 开发运行
cargo run --release --features desktop --bin wunder-desktop

# Bridge 单独运行
cargo run --release --bin wunder-desktop-bridge
```

Desktop 启动时自动生成运行配置，模式设置为 `desktop`。

## 8. 构建与分发

```bash
# 前端构建
npm run build --workspace wunder-frontend

# 桌面资源准备
npm run prepare:desktop:resources

# Electron 打包
npm run build --workspace wunder-desktop-electron

# Linux 桌面打包
npm run build:desktop:linux:x64
```

## 9. 启动前检查清单

- 配置文件路径正确
- 数据库可用（server: PostgreSQL；desktop/CLI: SQLite）
- `workspace.root`、浏览器数据目录、导出目录可写
- LLM 配置可达（endpoint + API key）
- MCP 配置文件中 endpoint 与鉴权配置正确
- 前端 dev 代理目标与服务端地址一致
