# 目标态系统

## 1. 概述

目标态系统（Goal Mode System）是 wunder 的任务驱动执行模式。当用户设定目标后，智能体将持续工作直到目标完成或预算耗尽，期间自动处理等待、重试、预算检查等边界情况。

## 2. 核心概念

### 2.1 目标（Goal）

目标是绑定到会话的长期任务描述，包含：

| 字段 | 类型 | 说明 |
| --- | --- |
| `goal_id` | string | 目标唯一标识 |
| `session_id` | string | 所属会话 |
| `user_id` | string | 所属用户 |
| `objective` | string | 目标描述（最大 4000 字符） |
| `status` | string | 状态：active / paused / budget_limited / complete |
| `token_budget` | int? | Token 预算上限（可选） |
| `tokens_used` | int | 已消耗 Token |
| `time_used_seconds` | int | 已消耗时间（秒） |
| `source` | string | 来源：api / model / system |

### 2.2 目标状态

| 状态 | 含义 | 可转换到 |
| --- | --- | --- |
| `active` | 活跃执行中 | paused, budget_limited, complete |
| `paused` | 用户暂停 | active |
| `budget_limited` | 预算耗尽 | —（终态） |
| `complete` | 已完成 | —（终态） |

### 2.3 目标工具

智能体通过以下工具操作目标：

| 工具名 | 作用 | 调用方 |
| --- | --- | --- |
| `get_goal` | 读取当前目标 | 智能体 |
| `create_goal` | 创建新目标（仅当无活跃目标时） | 智能体 |
| `update_goal` | 标记目标完成 | 智能体 |

## 3. 执行流程

### 3.1 目标启动

```
用户通过 /goal 命令或目标编辑器设定目标
    ↓
API 调用 PUT /chat/sessions/{id}/goal
    ↓
创建 SessionGoalRecord，status=active
    ↓
调度目标延续任务
    ↓
线程运行时接收延续请求
    ↓
注入 [GOAL_CONTINUATION] 提示词
    ↓
智能体开始执行
```

### 3.2 目标延续

当智能体完成一轮执行后：

```
检查目标状态
    ↓
若 status=active 且未达预算上限
    ↓
计算延续延迟（冷却期 1 秒）
    ↓
调度下一轮延续
    ↓
注入延续提示词继续执行
```

### 3.3 预算检查

每轮执行后：

```
累加 tokens_used 和 time_used_seconds
    ↓
若 token_budget > 0 且 tokens_used >= token_budget
    ↓
自动设置 status=budget_limited
    ↓
发送 goal_budget_limited 事件
    ↓
停止目标延续
```

### 3.4 目标完成

智能体调用 `update_goal(status="complete")`：

```
验证当前目标存在且非终态
    ↓
设置 status=complete, completed_at=now
    ↓
发送 goal_updated 事件
    ↓
停止目标延续
```

## 4. API 接口

### 4.1 获取目标

```
GET /wunder/chat/sessions/{session_id}/goal
```

**响应：**

```json
{
  "data": {
    "goal": {
      "goal_id": "abc123",
      "session_id": "session-1",
      "user_id": "user-1",
      "objective": "完成用户认证模块的开发",
      "status": "active",
      "token_budget": 100000,
      "tokens_used": 45230,
      "time_used_seconds": 180,
      "created_at": 1715049600.0,
      "updated_at": 1715049780.0,
      "completed_at": null,
      "last_continued_at": 1715049770.0,
      "source": "api"
    }
  }
}
```

### 4.2 设置目标

```
PUT /wunder/chat/sessions/{session_id}/goal
```

**请求体：**

```json
{
  "objective": "完成用户认证模块的开发",
  "token_budget": 100000
}
```

**响应：**

```json
{
  "data": {
    "goal": { ... },
    "continuation": {
      "should_start": true,
      "reason": "started",
      "session_id": "session-1",
      "goal_id": "abc123"
    }
  }
}
```

### 4.3 清除目标

```
DELETE /wunder/chat/sessions/{session_id}/goal
```

**注意**：通过 API 清除目标会返回错误，必须通过聊天停止按钮退出目标态。

## 5. 前端集成

### 5.1 目标编辑器

组件：`MessengerGoalDialog.vue`

- 显示当前目标（如有）
- 编辑目标描述
- 可选设置 Token 预算
- 提交后立即启动目标态

### 5.2 目标分隔线

组件：`MessageGoalDivider.vue`

在消息列表中显示目标开始标记：

```
────────── 目标已开始：完成用户认证模块的开发 ──────────
```

### 5.3 目标状态指示

在聊天面板显示当前目标状态：

- 活跃：绿色徽章
- 暂停：黄色徽章
- 预算受限：红色徽章
- 已完成：灰色徽章

### 5.4 命令支持

| 命令 | 作用 |
| --- | --- |
| `/goal` | 打开目标编辑器 |
| `/goal 目标描述` | 直接设置目标 |
| `/stop` 或 `/cancel` | 退出目标态 |

## 7. 事件流

目标状态变化通过事件流推送：

| 事件 | 触发时机 |
| --- | --- |
| `goal_updated` | 目标创建、状态变更 |
| `goal_cleared` | 目标清除 |
| `goal_continuation_started` | 延续任务启动 |
| `goal_budget_limited` | 预算耗尽 |

## 8. 安全约束

1. **目标描述长度**：最大 4000 字符
2. **状态变更限制**：
   - API 不能设置 paused 或 complete 状态
   - 智能体只能设置 complete 状态
   - budget_limited 由系统自动设置
3. **延续冷却期**：两次延续间隔至少 1 秒
4. **线程锁定**：目标态期间禁止切换会话

## 9. 与其他系统的关系

| 系统 | 关系 |
| --- | --- |
| 线程运行时 | 目标注入提示词，驱动延续执行 |
| 工具系统 | 提供 get_goal / create_goal / update_goal |
| 上下文压缩 | 目标描述作为重要上下文保留 |
| 实时投影 | 目标状态变化实时推送到前端 |
| 形象系统 | 目标执行时形象显示 running 状态 |
