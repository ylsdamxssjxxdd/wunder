# 形象系统

## 1. 概述

形象系统（Companion System）是 wunder 的桌面端与网页端的视觉陪伴层，通过精灵动画形象为用户提供沉浸式的交互体验。形象系统独立于智能体核心运行时，作为前端渲染层存在，不影响后端执行链路。

## 2. 核心概念

### 2.1 形象包（Companion Package）

形象包是一个 ZIP 压缩文件，包含：

| 文件 | 说明 |
| --- | --- |
| `pet.json` | 清单文件，定义 id、displayName、description、spritesheetPath |
| `{spritesheetPath}` | 精灵表图片，支持 WebP/PNG/GIF/JPEG |

**清单示例：**

```json
{
  "id": "cat-assistant",
  "displayName": "小助手",
  "description": "一只可爱的小猫形象",
  "spritesheetPath": "spritesheet.webp"
}
```

### 2.2 形象作用域

| 作用域 | 存储位置 | 管理方式 |
| --- | --- | --- |
| `global` | 服务端 `config/data/companions/global/` | 管理员上传，所有用户可见 |
| `private` | 浏览器 IndexedDB | 用户自行导入，仅本地可用 |

### 2.3 精灵状态

形象支持以下动画状态：

| 状态 ID | 触发场景 |
| --- | --- |
| `idle` | 默认静止状态 |
| `running` | 智能体执行中 |
| `running-left` | 拖拽向左移动 |
| `running-right` | 拖拽向右移动 |
| `waving` | 点击形象 |
| `jumping` | 智能体执行完成 |
| `failed` | 智能体执行失败 |
| `waiting` | 等待用户确认 |
| `review` | 展示消息气泡 |

## 3. 架构设计

### 3.1 前端组件

```
frontend/src/
├── stores/companions.ts              # 形象状态管理
├── components/companions/
│   ├── CompanionFloatingLayer.vue    # 浮层容器
│   └── CompanionSprite.vue           # 精灵渲染
└── utils/companionPackage.ts         # 包解析工具
```

### 3.2 后端服务

```
src/
├── services/companions.rs            # 形象包管理服务
└── api/companions.rs                 # HTTP 接口
```

### 3.3 数据流

```
用户导入形象包
    ↓
前端解析 ZIP → 提取精灵表 → 存储到 IndexedDB
    ↓
用户选择形象 → 保存到 localStorage
    ↓
CompanionFloatingLayer 渲染浮层
    ↓
监听智能体运行状态 → 切换精灵动画
    ↓
（桌面端）同步状态到 Tauri/Electron 桥接层
```

## 4. API 接口

### 4.1 获取全局形象列表

```
GET /wunder/companions/global
```

**响应：**

```json
{
  "data": {
    "items": [
      {
        "id": "cat-assistant",
        "display_name": "小助手",
        "description": "一只可爱的小猫形象",
        "spritesheet_path": "spritesheet.webp",
        "spritesheet_data_url": "data:image/webp;base64,...",
        "spritesheet_mime": "image/webp",
        "imported_at": 1715049600.0,
        "updated_at": 1715049600.0
      }
    ]
  }
}
```

### 4.2 下载形象包

```
GET /wunder/companions/global/{id}/package
```

返回 ZIP 文件流。

### 4.3 管理员接口

```
POST   /admin/companions/global       # 上传形象包
PUT    /admin/companions/global/{id}  # 更新元信息
DELETE /admin/companions/global/{id}  # 删除形象
```

## 5. 前端状态管理

### 5.1 Store 结构

```typescript
interface CompanionSettings {
  selectedId: string;        // 当前选中的形象 ID
  enabled: boolean;          // 是否启用形象
  position: { x: number; y: number };  // 屏幕位置
  scale: number;             // 缩放比例 (0.5 ~ 1.6)
  messageHintsEnabled: boolean;  // 是否显示消息气泡
}

interface AgentCompanionOverride {
  show?: boolean;            // 该智能体是否显示形象
  scale?: number;            // 该智能体的形象缩放
  updatedAt: number;
}
```

### 5.2 智能体关联

智能体可通过 `icon` 字段配置形象：

```json
{
  "icon": {
    "kind": "companion",
    "scope": "global",
    "id": "cat-assistant",
    "show": true,
    "scale": 1.0,
    "messageHints": true
  }
}
```

## 6. 桌面端集成

### 6.1 Tauri 桥接

桌面端通过 `wunderDesktop` 全局对象与前端通信：

```typescript
interface DesktopCompanionBridge {
  showCompanion(payload: CompanionState): Promise<boolean>;
  updateCompanion(payload: CompanionState): Promise<boolean>;
  hideCompanion(payload?: { persistEnabled?: boolean }): Promise<boolean>;
  getCompanionState(): Promise<CompanionSettings>;
  onCompanionStateChanged(callback: (state: CompanionSettings) => void): () => void;
}
```

### 6.2 状态同步

- 前端 → 桌面：通过 `showCompanion` / `updateCompanion` / `hideCompanion` 推送
- 桌面 → 前端：通过 `onCompanionStateChanged` 监听并应用

## 7. 安全约束

1. **包大小限制**：ZIP 包最大 24MB，精灵表最大 18MB
2. **路径安全**：禁止 `..` 路径穿越，禁止绝对路径
3. **ID 规范化**：仅保留字母、数字、下划线、连字符
4. **文本长度**：displayName 最大 80 字符，description 最大 240 字符

## 8. 文件落点

| 内容 | 路径 |
| --- | --- |
| 全局形象存储 | `config/data/companions/global/{id}/` |
| 清单文件 | `config/data/companions/global/{id}/pet.json` |
| 精灵表 | `config/data/companions/global/{id}/{spritesheetPath}` |
