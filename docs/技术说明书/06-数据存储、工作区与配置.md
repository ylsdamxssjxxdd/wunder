# 数据存储、工作区与配置

## 1. 四类数据

| 类型 | 特征 | 举例 |
| --- | --- | --- |
| 数据库存储 | Durable，跨进程持久 | 用户、agent、hive、session、stream events |
| 文件现实 | 智能体真实读写的文件和目录 | 工作区、下载、导出、浏览器数据 |
| 配置文件 | 启动时加载，运行时只读为主 | `config/wunder.yaml` |
| 进程内状态 | 可丢失、可重建 | watch、presence、热缓存、临时目录 |

## 2. 数据库

| 运行形态 | 后端 |
| --- | --- |
| `wunder-server` | PostgreSQL |
| `wunder-desktop` | SQLite |
| `wunder-cli` | 本地 SQLite |

统一抽象位于 `src/storage/`。具体实现在 `src/storage/postgres.rs` 和 `src/storage/sqlite.rs`。

### 关键 durable 对象

- 用户、组织、权限、令牌、配额
- agent 预设、agent 配置、工具授权
- team run、team task
- 会话、消息历史、stream events
- 记忆记录、知识元信息
- channel / gateway 绑定
- 工具日志、附件日志

## 3. 工作区

工作区是智能体读写真实文件的现实层，不是附件桶。

能力包括：文件读写、附件存储与转换、导出路径、目录路由与 fence、与工具/浏览器/桌面系统联动。

**禁止**：
- 把长期业务文件写进 `data/` 或 `temp_dir/`
- 把 `temp_dir/` 当长期业务目录

## 4. 记忆与知识

| 子系统 | 说明 |
| --- | --- |
| 长期记忆 | 跨线程的用户/agent 记忆，通过 `memory.rs` + `memory_fragments.rs` 管理 |
| 知识库 | 文档解析、切分、缓存和检索（`knowledge.rs`） |
| 向量知识 | 向量化检索的补充能力（`vector_knowledge.rs`），不是唯一知识入口 |

## 5. 配置文件

默认路径：`config/wunder.yaml`。示例：`config/wunder-example.yaml`。可通过 `WUNDER_CONFIG_PATH` 覆盖。

配置结构定义：`src/core/config.rs`。

### 关键配置段

| 配置段 | 作用 |
| --- | --- |
| `security` | API key、执行策略、允许路径与命令 |
| `server` | host、port、mode、流式通道 |
| `llm` | 默认模型、提供方和超时参数 |
| `tools` | 内置工具、web fetch、browser、desktop controller |
| `browser` | 浏览器运行时与 docker/headless 参数 |
| `cron` | 定时任务运行参数 |
| `workspace` | 工作区根目录和保留策略 |
| `mcp` | MCP 服务器列表和超时 |
| `storage` | 存储后端设置 |
| `channels` | 渠道配置 |

### 环境变量

| 变量 | 作用 |
| --- | --- |
| `WUNDER_CONFIG_PATH` | 指定配置文件路径 |
| `WUNDER_HOST` | 覆盖绑定主机 |
| `WUNDER_PORT` | 覆盖绑定端口 |
| `WUNDER_SERVER_MODE` | 覆盖运行模式 |
| `WUNDER_HOME` | CLI / 本地运行时 home |
| `WUNDER_USER_TOOLS_ROOT` | 用户工具目录 |

CLI 和 Desktop 会在启动时生成运行时配置副本，设置相应路径变量。

## 6. 配置管理原则

- 以 `config/wunder.yaml` 为主，环境变量覆盖为辅。
- 运行时裁剪在 CLI / Desktop 初始化阶段完成，不在业务逻辑里散落模式判断。
- 变更配置结构后，同步更新示例配置、技术说明书和 API 文档。

## 7. 排查矩阵

| 问题 | 检查 |
| --- | --- |
| 服务端口不对 | `WUNDER_PORT` 与 `config.server.port` |
| 模式异常 | `WUNDER_SERVER_MODE` 与 `config.server.mode` |
| 工作区路径不对 | `workspace.root` 与目录路由规则 |
| MCP 不生效 | `config/mcp_config.json` endpoint、headers、enabled |
| 本地 CLI 配置漂移 | 运行时生成的配置副本 |
