# wunder 技术说明书

## 文档定位

本目录是 wunder 的**工程执行手册**，回答五个实际问题：

- 怎么部署、启动、配置？
- 各模块在哪里，改功能该动哪个目录？
- 一条请求在代码里怎么流转？
- 出了问题看什么？
- 开发和验收遵循什么口径？

与 `docs/设计文档` 的分工：

| 文档层 | 回答的问题 |
| --- | --- |
| `docs/设计文档` | 为什么这样设计？边界怎么划分？长期目标是什么？ |
| `docs/技术说明书` | 具体怎么接、怎么跑、怎么改、怎么查？ |

## 读者

- **后端开发**：运行时、存储、接口、扩展路径
- **前端/桌面开发**：实时事件、路由、状态恢复、本地桥接
- **运维/集成**：部署方式、配置项、网关、渠道、观测
- **评审/验收**：系统边界、运行链路、风险点、验收标准

## 发布边界

当前默认发布场景：**一人一会话、各自分散聊天**。

在该边界下，验收以消息主链、连接恢复、会话列表/正文同步和设置页补热为主。

## 文档约定

- 设计约束以 `docs/设计文档` 为准，API 事实以 `docs/API文档.md` 为准。
- 本目录优先写**真实代码落点、启动方式、排障手段、改动步骤**。
- 主链、配置项、部署口径变化时，先更新本目录。
