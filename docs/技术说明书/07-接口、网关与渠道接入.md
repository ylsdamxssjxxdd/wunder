# 接口、网关与渠道接入

## 1. 接入层定位

接入层把 wunder 的运行时能力稳定暴露给前端、桌面、CLI、外部节点和第三方渠道。职责：鉴权、参数校验、协议适配、路由分发。**不承担业务主逻辑。**

三个接入子系统：

| 子系统 | 目录 | 接入对象 |
| --- | --- | --- |
| API | `src/api/` | 前端、桌面、CLI、管理端 |
| Gateway | `src/gateway/` | 外部节点、operator |
| Channels | `src/channels/` | 第三方通讯渠道 |

## 2. API 层

主路由构建：`src/api/mod.rs`。Server 调用 `build_router`，Desktop 本地模式调用 `build_desktop_router`。

### 路由分组

| 接口域 | 说明 |
| --- | --- |
| `auth` | 登录鉴权 |
| `chat` / `chat_ws` | 聊天与会话实时链路 |
| `core` / `core_ws` | 核心实时链路 |
| `user_world` / `user_world_ws` | 用户世界与实时链路 |
| `beeroom` / `beeroom_ws` | beeroom 实时链路 |
| `user_tools` / `user_prompt_templates` | 用户侧能力管理 |
| `admin*` | 管理员控制面 |
| `channel` / `user_channels` | 渠道配置与收发 |
| `gateway_ws` | Gateway 接入 |
| `workspace` / `temp_dir` | 文件与临时目录能力 |
| `a2a` | 智能体互联能力 |
| `team_runs` | Team run 管理 |
| `benchmark` | 基准测试 |
| `cron` | 定时任务 |
| `desktop` / `desktop_lan` | 本地模式裁剪接口 |
| `attachment_convert` / `doc2md` | 附件与文档转换 |
| `browser_control` | 浏览器控制 |

完整接口族以 `docs/API文档.md` 为准。

## 3. Gateway

Gateway 是多角色接入中枢，位于 `src/gateway/`，WS 入口在 `src/api/gateway_ws.rs`。

职责：
- operator / node / channel 等角色接入
- 路由所有权和路由租约
- presence 与 health 管理
- 节点调用计划与回包协议

## 4. Channels

Channels 负责 wunder 与外部通讯世界对接，位于 `src/channels/`。

### 已支持渠道

飞书、微信/微信公众号/企业微信、QQ、XMPP、WhatsApp Cloud

### 内部拆分

```
src/channels/
├── adapter.rs        渠道适配器注册
├── binding.rs        渠道绑定
├── catalog.rs        渠道目录
├── feishu*.rs        飞书适配
├── wechat*.rs        微信适配
├── weixin*.rs        企业微信适配
├── qqbot.rs          QQ 适配
├── xmpp*.rs          XMPP 适配
├── whatsapp_cloud.rs WhatsApp 适配
├── inbound_queue.rs  入站消息队列
├── outbox.rs         出站消息队列
├── outbound_attachments.rs 出站附件
├── media.rs          媒体处理
├── pending_files.rs  待处理文件
├── rate_limit.rs     限流
├── runtime_log.rs    渠道运行日志
├── workspace_routing.rs 工作区路由
├── service.rs        渠道服务
├── registry.rs       注册表
└── types.rs          公共类型
```

新增渠道时必须复用这套拆分，不写全耦合大文件适配器。

## 5. 身份规则

接入层严格区分四类身份：

| 身份 | 来源 |
| --- | --- |
| 注册用户 | 用户表，有完整 user_id |
| 线程归属 | `user_id`，不要求是已注册用户 |
| Gateway / Node / Channel | 网关和渠道的接入凭证 |
| 管理员 | 管理端鉴权 |

## 6. 新增 API 步骤

1. 在 `src/api/` 增加路由文件
2. 业务逻辑下沉到 `src/services/*`
3. 在 `src/api/mod.rs` 挂载路由
4. 更新 `docs/API文档.md` 对应主题

## 7. 新增 Gateway 能力步骤

1. 判断是协议字段、路由所有权还是节点调用逻辑
2. 在 `src/gateway/` 修改协议结构或执行逻辑
3. 通过 `src/api/gateway_ws.rs` 暴露接入面
4. 补齐 role / auth / route lease / presence 联动验证

## 8. 新增渠道步骤

1. 在 `src/channels/` 新建独立适配文件
2. 接入注册表和消息服务
3. 处理 inbound / outbound / 媒体 / 重试 / 限流 / 工作区路由
4. 增加配置说明与接入文档
