# 仓库结构与模块职责

## 1. 顶层目录

```
wunder/
├── Cargo.toml        纯 workspace manifest，默认成员为 crates/wunder-server
├── crates/           Rust workspace 子 crate（core、runtime、server、cli、desktop）
├── frontend/         用户侧前端（Vue 3）
├── web/              管理员侧前端（原生 HTML + JS 模块）
├── desktop/electron/ Electron 打包壳
├── config/           配置样例、i18n、字体、MCP 配置
├── docs/             设计文档、技术说明书、API 文档、方案
├── scripts/          仓库脚本
├── tests/            非 Rust 测试夹具与脚本；Rust 集成测试在 crates/wunder-runtime/tests/
├── extra_mcp/        额外 MCP 运行时与工具脚本
└── packaging/        构建/分发附加资源（docker、python、windows）
```

## 2. Rust 后端分层

### 2.1 接入层

把外部请求接入内部运行时。只做鉴权、参数校验、协议适配和接线。

| 目录 | 职责 |
| --- | --- |
| `crates/wunder-server/src/` | Server 入口、middleware、静态资源挂载与启动装配 |
| `crates/wunder-runtime/src/api/` | HTTP / WS 路由、请求编排、payload shaping |
| `crates/wunder-runtime/src/gateway/` | Gateway 协议、节点调用、管理面消息 |
| `crates/wunder-runtime/src/channels/` | 外部渠道适配（飞书、微信、QQ 等）、消息收发 |

### 2.2 运行时层

智能体执行的核心引擎。

| 目录 | 职责 |
| --- | --- |
| `crates/wunder-runtime/src/orchestrator/` | 智能体执行调度主链（turn / prompt / context / tool / recovery） |
| `crates/wunder-runtime/src/services/runtime/thread/` | 线程运行时（线程生命周期、lease、排队、dispatch） |
| `crates/wunder-runtime/src/services/runtime/mission/` | Mission 运行时（多智能体协作任务调度） |
| `crates/wunder-runtime/src/core/` | 配置、鉴权、审批、exec policy、i18n、schemas、state |
| `crates/wunder-runtime/src/sandbox/` | 沙盒服务 |
| `crates/wunder-core/src/` | 低依赖稳定基础能力，目前承载仓库资源路径等公共逻辑 |

### 2.3 业务服务层

领域逻辑和编排。

| 目录 | 职责 |
| --- | --- |
| `crates/wunder-runtime/src/services/swarm/` | 协作任务、team run、聚合汇总 |
| `crates/wunder-runtime/src/services/tools/` | 工具实现（文件、命令、浏览器、渠道消息等） |
| `crates/wunder-runtime/src/services/memory*` | 长期记忆与自动提取、片段管理 |
| `crates/wunder-runtime/src/services/knowledge.rs` | 知识解析与检索 |
| `crates/wunder-runtime/src/services/vector_knowledge.rs` | 向量知识检索 |
| `crates/wunder-runtime/src/services/workspace.rs` | 工作区与文件现实 |
| `crates/wunder-runtime/src/services/user_world.rs` | 用户世界与消息视图 |
| `crates/wunder-runtime/src/services/beeroom_realtime.rs` | Beeroom 实时服务 |
| `crates/wunder-runtime/src/services/hive_pack.rs` | 蜂群资产封装 |
| `crates/wunder-runtime/src/services/abilities/` | 智能体能力模型 |
| `crates/wunder-runtime/src/services/inner_visible/` | 内部可见性服务 |
| `crates/wunder-runtime/src/services/cron*` | 定时任务调度 |
| `crates/wunder-runtime/src/services/skills.rs` | 技能管理与调用 |
| `crates/wunder-runtime/src/services/mcp.rs` | MCP 管理与接入 |
| `crates/wunder-runtime/src/services/bridge/` | 桥接服务与身份适配 |
| `crates/wunder-runtime/src/services/external/` | 外部系统接入 |
| `crates/wunder-runtime/src/services/sim_lab/` | 仿真与评估实验室 |
| `crates/wunder-runtime/src/services/subagents.rs` | 子智能体派生与控制 |

### 2.4 基础设施层

| 目录 | 职责 |
| --- | --- |
| `crates/wunder-runtime/src/storage/` | PostgreSQL / SQLite 统一抽象与实现 |
| `crates/wunder-runtime/src/ops/` | 监控、性能、吞吐、基准测试 |
| `crates/wunder-runtime/src/lsp/` | LSP 相关入口 |

## 3. Orchestrator 内部结构

`crates/wunder-runtime/src/orchestrator/` 是 wunder 的智能体执行调度引擎，管理从请求接入到最终回复的完整链路：

```
crates/wunder-runtime/src/orchestrator/
├── mod.rs              模块入口与公共类型
├── config.rs           运行时配置
├── constants.rs        常量定义
├── error.rs            错误类型
├── types.rs            公共类型定义
├── request.rs          请求模型
├── turn_state.rs       Turn 状态机
├── execute.rs          Turn 执行主链
├── preflight.rs        执行前预处理
├── prompt.rs           Prompt 组装与冻结
├── context.rs          上下文管理
├── context_compactor.rs 上下文压缩
├── compaction_policy.rs 压缩策略
├── memory.rs           记忆注入
├── llm.rs              模型调用
├── tool_calls.rs       工具调用解析
├── tool_exec.rs        工具执行调度
├── tool_parallel.rs    工具并行执行
├── retry_governor.rs   重试治理
├── result_normalizer.rs 结果归一化
├── event_stream.rs     事件流
├── stream_persist.rs   流式持久化
├── limiter.rs          限流器
├── runtime_snapshot.rs 运行时快照
└── thread_runtime.rs   线程运行时
```

## 5. 常见改动落点

| 要做的事 | 放在哪里 | 不要放 |
| --- | --- | --- |
| 新增 API | `crates/wunder-runtime/src/api/*` + `crates/wunder-runtime/src/services/*` | 在路由里堆业务逻辑 |
| 新增工具实现 | `crates/wunder-runtime/src/services/tools/*.rs` | 往旧 `crates/wunder-runtime/src/services/tools.rs` 堆 |
| 新增运行时语义 | `crates/wunder-runtime/src/orchestrator/*` | 在 API 层新造执行链路 |
| 新增线程调度 | `crates/wunder-runtime/src/services/runtime/thread/*` | 跳过 ThreadRuntime 直接操作 |
| 新增前端页面能力 | `frontend/src/views/messenger/` + `frontend/src/components/messenger/` | 往 `MessengerView.vue` 堆 |
| 新增管理页能力 | `web/modules/*` | 在 `web/app.js` 写耦合逻辑 |
| 新增 CLI/TUI 能力 | `crates/wunder-cli/*` | 放回仓库根旧目录 |
| 新增 Tauri 桌面能力 | `crates/wunder-desktop/*` | 放回 `desktop/tauri` |

## 6. 改代码前的判断顺序

1. 这是**接入层**问题、**运行时**问题、**投影**问题还是**业务服务**问题？
2. 这段逻辑是**核心执行语义**还是**产品适配**？
3. 操作的是 **durable state**、**实时状态** 还是**进程内临时状态**？
4. 改动是否需要同步更新设计文档、API 文档或本说明书？
