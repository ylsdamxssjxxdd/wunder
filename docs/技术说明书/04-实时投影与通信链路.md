# 实时投影与通信链路

## 1. 核心原则

wunder 的实时系统遵循一条铁律：**内部运行态先变成投影，再由前端消费**。前端、桌面、CLI 或外部观察者都只能通过投影获取运行态，不能直接读后端内部状态。

## 2. 代码落点

| 目录 | 职责 |
| --- | --- |
| `src/services/runtime/thread/` | 线程运行态与公开事件 |
| `src/services/runtime/mission/` | mission 运行态与公开事件 |
| `src/services/projection/` | session / beeroom / user_world 投影构建 |
| `src/services/presence/` | watch 和 presence 统计 |
| `src/api/realtime_public_view.rs` | HTTP/WS 首包回显投影目标与 snapshot 指针 |
| `src/api/chat_ws.rs` | 会话实时链路 |
| `src/api/user_world_ws.rs` | 用户世界实时链路 |
| `src/api/beeroom_ws.rs` | beeroom 实时链路 |
| `frontend/src/realtime/agentWorld/` | 前端实时消费者、watcher、恢复主链 |

## 3. 协议模型

所有实时目标必须适配这组协议，不能私造事件格式：

```
Snapshot → 首次接入时获取完整公共视图
Delta    → 运行时变化时发送增量事件
Replay   → 断线后补齐未消费事件
sync_required → 客户端无法安全追上时，要求整目标重同步
```

首包（Projection Public View）直接带 `projection_target + projection_snapshot`，作为 watcher 起点。

## 4. Watch 与 Presence

**Watch** — 某个连接正在观察某个投影目标。用于控制目标级订阅、计算活跃观察者数量、辅助节流和回放。

**Presence** — 记录哪些用户在线、哪些连接存在、某个目标有多少订阅者。Presence 是进程内观测状态，可丢失可重建，不是 durable business state。

## 5. 前端消费流程

```
1. 路由确定观察目标
2. 本地缓存恢复已有视图
3. 消费首包 projection public view → 写回 cursor / snapshot seed
4. WS 发起 watch
5. 接收 snapshot 或承接后续 delta
6. 断线 / lag / targeted gap → replay → targeted hydrate → 整目标重同步
```

前端允许短暂 optimistic 更新，但最终必须服从投影回压。

## 6. 会话类实时链路

负责：当前会话消息流、渠道消息即时渲染、线程运行态、waiting/approval/user_input 控制态、resume 后状态收敛。

常见问题：

| 问题 | 原因 |
| --- | --- |
| 控制面事件被当成消息事件 | 事件类型过滤缺失 |
| active session 与 list summary 收敛顺序错位 | 两路投影消费时序未协调 |
| 冷 session 恢复时详情和列表互相回压 | 缺少 targeted hydrate |
| watcher 被误清导致首条实时后必须手动刷新 | 补水逻辑错误重置了 watcher |

## 7. 协作类实时链路

负责：mission public board、hive / beeroom 公开摘要、team run 状态和 agent activity。

beeroom 是实时房间附着点，不应承担领域聚合逻辑。领域逻辑在 hive 层完成，beeroom 只做实时分发。

## 8. 通信原则

- **WebSocket 为主**，SSE 为兜底。
- 相同投影目标必须复用共享 replay / sync_required / watch helper。
- 页面卸载只取消当前 watch，**不关闭全局共享连接**。
- 工具页、设置页、工作目录默认走补热链路，不抢占消息热路径。

## 9. 排查矩阵

| 现象 | 检查点 |
| --- | --- |
| 页面实时卡住 | 连接在线？watch 仍有效？ |
| 重连后列表不一致 | replay 命中？是否被标记 sync_required？ |
| 活跃会话正文不刷新 | 当前 target 是否指向最新 session？ |
| 设置页/工具页切换变慢 | 是否重复请求同一份数据？single-flight 是否生效？ |
| beeroom 信息混乱 | 是否把 hive 逻辑堆进了 beeroom 层？ |
