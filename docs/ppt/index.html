是什么

系统内置的文件与命令操作工具，默认可用

MCP 工具：对接外部能力

当内置不够用，就把外部系统接进来

是什么

通过 MCP 协议接入外部服务，以 server@tool 调用

典型用途
  • 调用企业系统、搜索、BI 等服务
  • 让模型具备跨系统操作能力
  • 统一纳入工具清单管理
治理方式
  • allow_tools 白名单控制可调用工具
  • tool_specs 缓存，避免频繁远端发现
  • 统一 timeout，避免外部服务阻塞
server@tool allow_tools timeout

自建工具：个人专属能力

人人都能拥有自己的专属工具箱

是什么

每个用户维护自己的工具配置包

关键特性
  • 配置路径：data/user_tools/<user_id>
  • 别名统一为 user_id@tool
  • 可独立启用 MCP、技能、知识库
有什么用
  • 满足个人差异化需求
  • 不影响他人，隔离又可控
  • 可选择共享给团队
效果：个人工具箱随账号长期保留

共享工具：团队复用能力

个人能力还能变成团队共享能力池

是什么

共享的是配置与可用清单，而非共享工作区

共享方式
  • 名称统一为 owner_id@tool
  • 共享标记即白名单
  • 使用者需显式勾选
有什么用
  • 让成熟工具在团队内快速复制
  • 不共享对方工作区与额外提示词
  • 降低协作门槛与沟通成本
效果：像“工具市场”一样复用能力

工作区是什么？

工具执行需要长期落脚点，这就是工作区

核心定位

每个用户一块持久化空间，用于存放文件与产物

路径示例

主要资产

有什么用
  • 让模型“写出来”的内容有处可存
  • 跨会话继续使用同一套资料
  • 避免对话结束后成果消失
效果：对话变成可持续的工作资产

第4节 上下文压缩
为什么 流程

为什么要做上下文压缩？

对话越长越重，需要机制保持可持续

问题
  • 长对话会占满模型上下文
  • 越到后面越慢、越贵、越容易超限
  • 历史信息被截断会导致答非所问
解决目标
  • 保留关键知识与近期上下文
  • 避免请求因超限失败
  • 让会话可持续进行
一句话：让对话“能一直聊下去”
第4节 上下文压缩
为什么 流程

上下文压缩是怎么做的？

用结构化摘要保住重点，再继续对话

触发条件
  • 历史 token 占用达到阈值（默认 80%）
  • 当前消息接近安全上限
核心流程
  • 保留系统提示词 + 最近消息
  • 生成结构化摘要并插入系统消息
  • 重置历史占用，避免重复触发
结果效果
  • 关键信息被保留，细节被压缩
  • 长会话仍可稳定执行工具
  • 对用户透明，不影响使用体验
标记：摘要消息以 [上下文摘要] 形式注入
第5节 长期记忆
生成 价值

长期记忆是怎么生成的？

压缩解决单次会话，记忆贯穿长期协作

触发时机
  • 最终回复完成后自动触发总结
  • 使用专用提示词 memory_summary.txt
写入方式
  • 保存到 memory_records 表
  • 带年月日时分时间前缀
  • 单用户最多 30 条
对话注入
  • 后续会话自动追加 [长期记忆]
  • 让模型记住用户偏好与关键信息
  • 跨会话依然能承接任务
效果：从“临时聊天”升级为“长期助理”
第5节 长期记忆
生成 价值

长期记忆有什么用？

让协作信息持续一致，同时保持可控

业务价值
  • 保留常用结论、固定约束、偏好
  • 让协作中的信息持续一致
  • 减少重复解释与回顾成本
可控性
  • 支持开关控制是否启用
  • 可查看、清空或删除记录
  • 避免误记或过时信息干扰
效果：长期记忆可用且可控
