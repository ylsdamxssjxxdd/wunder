# 智能体内见系统落地方案

## 1. 目标

内见系统采用 **File-First**：通过用户私有目录中的文件直接控制智能体配置，用户与智能体都可读写。

- 不新增“专用配置服务面板协议”
- 不依赖 `.wunder` 内部目录
- 不在聊天右侧增加“家目录切换按钮”

## 2. 根目录与最小结构

内见根目录即用户私有目录（`container_id=0`）：

```text
<HOME>/
  agents/
  skills/
  knowledge/
  global/
```

仅保留以上 4 个目录作为内见系统标准结构。

## 3. agents 目录规范（扁平）

`agents/` 目录只允许工蜂卡文件：

```text
agents/<agent_id>.worker-card.json
```

约束：

- 不再使用 `agents/<agent_id>/` 子目录
- 不再使用 `agents/<agent_id>/system_prompt.md`
- 不再使用 `agents/<agent_id>/skills/`
- 不再写入 `agents/**/README.md`

## 4. 配置载体与同步

### 4.1 主载体

- 单智能体配置：`agents/<agent_id>.worker-card.json`
- 用户级默认卡：`global/defaults.worker-card.json`
- 用户级工具策略：`global/tooling.json`

### 4.2 同步原则

- 运行时读取文件后同步到 `user_agents` / default agent meta（声明依赖与运行工具按现有逻辑对齐）
- 用户或智能体改动 `.worker-card.json` 后，下一次同步应更新设置页可见字段（名称、描述、提示词、工具、技能等）
- 线程 system prompt 仍保持冻结语义：已冻结线程不热替换

## 5. 异常处理

不再依赖 `.wunder/diagnostics` 与 `.wunder/last_good` 文件回退，改为：

- 解析失败时保留当前数据库已生效配置
- 记录服务端日志告警
- 下次同步继续尝试修复

## 6. 与会话线程控制工具关系

- 内见系统：管理“配置与资产文件”
- 会话线程控制工具：管理“线程切换/主线程/归档恢复”

两者互补：改配置后如需立即体验，建议切到新线程验证。

## 7. 兼容与迁移策略

- 读取时兼容旧结构 `agents/<agent_id>/worker-card.json`
- 同步写回时统一落到新结构 `agents/<agent_id>.worker-card.json`
- 同步后清理旧 `agents/<agent_id>/` 遗留目录
- 清理用户私有目录下的 `.wunder/` 遗留目录

## 8. 协议提示词要求

系统提示词中的内见协议应满足：

- 使用 `HOME + 相对目录`，避免重复长路径
- 显式提供“当前智能体 ID”与“当前智能体工蜂卡路径”
- 当当前智能体不是默认智能体时，不再强调默认智能体路径
- 不再暴露 `.wunder` 相关路径
