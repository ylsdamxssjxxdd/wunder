# 人力资源管理制度汇编（V2026-07）

文档编号：`HR-POLICY-COMPILATION-2026-07`  
生效日期：`2026-01-01`  
版本用途：2026 年度人力资源制度统一口径（含模拟任务场景）。

## 第1章 总则

### 1.1 目标

本汇编用于统一组织内人力资源管理口径，覆盖招聘、任职资格、绩效、培训、纪律、晋升、例外审批与审计追溯。

### 1.2 适用范围

- 在册员工（正式、劳务派遣、外包）；
- 人员管理相关部门（HRBP、用人部门、纪检、流程治理）。

### 1.3 数据优先级

1. 数据库事实优先。  
2. 制度条款用于解释口径与边界。  
3. 存在冲突时，以“生效日期更近 + 章节优先级更高”条款为准。

## 第2章 岗位与职级体系

### 2.1 序列定义

- 专业序列；
- 管理序列；
- 项目序列。

### 2.2 职级定义

- 初级；
- 中级；
- 高级。

### 2.3 任职起算

同一职级任职时长，以 `current_level_start_date` 起算。

## 第3章 招聘与在离职管理

### 3.1 入职字段

关键字段包括 `hire_date`、`employment_type`、`status`。

### 3.2 状态变更

状态以数据库 `employees.status` 为准，离职状态不参与本轮晋升合格名单。

## 第4章 考勤与假勤

本章定义考勤行为规范，用于绩效与纪律数据的业务解释，不直接作为本轮晋升硬性判定字段。

## 第5章 绩效管理

### 5.1 周期

采用季度绩效口径，场景使用 `2025Q3~2026Q2` 四个季度。

### 5.2 等级

绩效等级：`S/A/B/C/D`。

### 5.3 绩效数据来源

`performance_quarterly` 表为唯一统计口径。

## 第6章 培训与认证

### 6.1 培训学时

有效学时统计口径：`training_year=2025 AND passed=1` 的 `hours` 累加。

### 6.2 必需认证

必需认证口径：`is_required_for_promotion=1` 且 `valid_until >= 截止日期`。

## 第7章 纪律与合规

### 7.1 有效处分

`is_revoked=0` 视为有效处分。

### 7.2 高风险处分

高风险处分级别：`记过`、`严重违纪`。

## 第8章 例外审批与复核

### 8.1 例外规则

- `PROM-MID-EXC-001`：任职时长豁免；
- `PROM-MID-EXC-002`：培训学时豁免；
- `PROM-MID-EXC-003`：必需认证豁免。

### 8.2 生效条件

`approved=1` 且 `approved_at <= 截止日期`。

## 第9章 职级晋升管理（中级场景）

> 说明：本章为本次模拟任务主口径；其余章节用于补充解释与边界控制。

### 9.1 任务参数

- `cutoff_date = 2026-07-31`
- `target_level = 中级`
- 输出目标：生成可晋升中级人员 Excel 名单。

### 9.2 硬性规则（必须全部通过）

1. `PROM-MID-HARD-001`：`status=在职 AND employment_type=正式`
2. `PROM-MID-HARD-002`：`current_level=初级` 且任职时长 `>=24` 个月（可被 `PROM-MID-EXC-001` 覆盖）
3. `PROM-MID-HARD-003`：四季度绩效均不低于 B，且 A/S 季度不少于 2
4. `PROM-MID-HARD-004`：2025 年通过培训学时 `>=40`（可被 `PROM-MID-EXC-002` 覆盖）
5. `PROM-MID-HARD-005`：近 12 个月（`2025-08-01~2026-07-31`）无高风险有效处分
6. `PROM-MID-HARD-006`：具备有效必需认证（可被 `PROM-MID-EXC-003` 覆盖）

### 9.3 综合评分规则

规则编号：`PROM-MID-SCORE-001`

`total_score = avg_performance_score*0.45 + competency_score*0.25 + potential_score*0.15 + integrity_score*0.15`

达标阈值：`total_score >= 78`。

### 9.4 合格判定

`eligible = (hard_rules_all_passed) AND (PROM-MID-SCORE-001 passed)`

### 9.5 机器可读块（供检索）

```yaml
scenario_id: PROMOTION_MID_202607
cutoff_date: "2026-07-31"
target_level: "中级"
rules:
  - rule_id: PROM-MID-HARD-001
    type: hard
    expr: "employees.status='在职' AND employees.employment_type='正式'"
  - rule_id: PROM-MID-HARD-002
    type: hard
    expr: "employees.current_level='初级' AND TIMESTAMPDIFF(MONTH, employees.current_level_start_date, '2026-07-31') >= 24"
    exception_rule: PROM-MID-EXC-001
  - rule_id: PROM-MID-HARD-003
    type: hard
    expr: "all(grades in [S,A,B]) AND count(grades in [S,A])>=2 for quarters(2025Q3,2025Q4,2026Q1,2026Q2)"
  - rule_id: PROM-MID-HARD-004
    type: hard
    expr: "SUM(training.hours where training_year=2025 and passed=1) >= 40"
    exception_rule: PROM-MID-EXC-002
  - rule_id: PROM-MID-HARD-005
    type: hard
    expr: "no effective discipline in ['记过','严重违纪'] between 2025-08-01 and 2026-07-31"
  - rule_id: PROM-MID-HARD-006
    type: hard
    expr: "exists required certification valid_until >= 2026-07-31"
    exception_rule: PROM-MID-EXC-003
  - rule_id: PROM-MID-SCORE-001
    type: score
    expr: "avg_performance_score*0.45 + competency_score*0.25 + potential_score*0.15 + integrity_score*0.15 >= 78"
```

### 9.6 输出字段

`employee_id,name,dept_id,current_level,target_level,eligible,total_score,hit_rule_ids,evidence_summary,cutoff_date`
