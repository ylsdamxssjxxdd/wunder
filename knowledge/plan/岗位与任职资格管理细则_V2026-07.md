# 岗位与任职资格管理细则（V2026-07）

文档编号：`HR-JOB-QUAL-DETAIL-2026-07`  
生效日期：`2026-01-01`  

## 第1章 总则

### 1.1 目的
本细则旨在规范公司岗位管理和任职资格管理，建立科学、规范的岗位体系和任职资格体系，为人力资源管理提供基础支撑，优化人力资源配置，提升员工能力素质，促进员工与公司共同发展。

### 1.2 适用范围
本细则适用于公司全体正式员工。劳务派遣员工、外包人员参照执行。

### 1.3 基本原则
1. **战略导向原则**：岗位体系和任职资格体系必须服务于公司战略目标，支撑公司业务发展。
2. **科学合理原则**：岗位设置和任职资格标准必须科学合理，符合公司实际情况。
3. **动态优化原则**：岗位体系和任职资格体系必须根据内外部环境变化持续优化。
4. **公平公正原则**：岗位评估和任职资格评定必须公平公正，确保评估结果客观准确。
5. **能力导向原则**：任职资格体系必须以能力为导向，注重员工能力提升和职业发展。

## 第2章 岗位体系

### 2.1 岗位设置原则
岗位设置遵循以下原则：
1. **因事设岗原则**：根据工作需要设置岗位，避免因人设岗。
2. **精简高效原则**：岗位设置必须精简高效，避免岗位重叠和职责不清。
3. **权责对等原则**：岗位责任和权力必须对等，确保岗位能够有效履行职责。
4. **统一规范原则**：岗位设置必须统一规范，便于管理和比较。

### 2.2 岗位分类
公司岗位分为以下类别：
1. **管理类**：承担管理职责，负责团队管理和业务决策。
2. **专业技术类**：从事专业技术工作，依靠专业知识和技能创造价值。
3. **市场营销类**：从事市场营销、销售、客户关系管理等工作。
4. **职能支持类**：从事人力资源、财务、法务、行政等职能支持工作。
5. **生产操作类**：从事生产制造、物流仓储等生产操作工作。
6. **项目管理类**：从事项目管理工作，负责项目规划、执行和监控。

### 2.3 岗位编码
岗位编码采用统一的编码规则，格式为：`部门代码-岗位类别代码-岗位序号`
- 部门代码：2位字母，代表所属部门
- 岗位类别代码：2位数字，代表岗位类别
- 岗位序号：3位数字，代表岗位在类别中的序号

岗位类别代码如下：
- 01：管理类
- 02：专业技术类
- 03：市场营销类
- 04：职能支持类
- 05：生产操作类
- 06：项目管理类

示例：`RD-02-001` 代表研发部门专业技术类第001号岗位。

### 2.4 岗位说明书
每个岗位必须制定岗位说明书，岗位说明书包括以下内容：
1. **基本信息**：
   - 岗位名称
   - 岗位编码
   - 所属部门
   - 岗位序列
   - 岗位职级
   - 直接上级
   - 直接下级
   - 岗位编制
2. **岗位概述**：简要描述岗位的目的、职责和定位。
3. **主要职责**：详细描述岗位的主要工作职责，一般8-12条。
4. **工作权限**：描述岗位拥有的决策权限、人事权限、财务权限等。
5. **工作关系**：描述岗位的内部工作关系和外部工作关系。
6. **工作环境**：描述岗位的工作环境特点，如办公室、车间、户外等。
7. **任职资格**：描述岗位的任职资格要求，包括学历、专业、工作经验、能力要求等。
8. **职业发展通道**：描述岗位的职业发展通道和晋升方向。
9. **其他信息**：其他需要说明的信息。

### 2.5 岗位评估
岗位评估是确定岗位相对价值的过程，岗位评估采用因素评分法，评估因素包括：
1. **影响范围**：岗位影响的范围和程度，权重25%。
2. **职责复杂度**：岗位职责的复杂程度，权重20%。
3. **知识技能要求**：岗位所需的知识和技能要求，权重20%。
4. **工作经验要求**：岗位所需的工作经验要求，权重15%。
5. **决策权限**：岗位的决策权限，权重10%。
6. **工作压力**：岗位的工作压力和强度，权重10%。

岗位评估流程如下：
1. 成立岗位评估委员会。
2. 制定岗位评估方案和标准。
3. 岗位评估委员会对岗位进行评估。
4. 统计评估结果，确定岗位等级。
5. 审核评估结果并报批。
6. 发布评估结果。

## 第3章 任职资格体系

### 3.1 任职资格定义
任职资格是指员工承担某一岗位工作所必须具备的知识、技能、能力和经验等方面的要求。

### 3.2 任职资格维度
任职资格包括以下维度：
1. **基本条件**：
   - 学历要求
   - 专业要求
   - 工作经验要求
   - 职称/资格证书要求
   - 年龄要求（如适用）
2. **知识要求**：
   - 专业知识
   - 行业知识
   - 公司知识
   - 管理知识（管理序列）
   - 项目管理知识（项目序列）
3. **技能要求**：
   - 专业技能
   - 沟通技能
   - 协作技能
   - 管理技能（管理序列）
   - 项目管理技能（项目序列）
4. **能力要求**：
   - 思维能力
   - 学习能力
   - 创新能力
   - 问题解决能力
   - 执行力
   - 影响力
5. **素质要求**：
   - 价值观
   - 职业操守
   - 责任心
   - 团队合作
   - 客户导向
6. **绩效要求**：
   - 过往绩效表现
   - 关键成果
   - 贡献记录

### 3.3 任职资格标准
任职资格标准按岗位序列和职级分别制定，标准包括：
1. **专业序列任职资格标准**：
   - P1-P10各级任职资格标准
   - 各专业领域的专业知识和技能要求
2. **管理序列任职资格标准**：
   - 基层管理任职资格标准
   - 中层管理任职资格标准
   - 高层管理任职资格标准
3. **项目序列任职资格标准**：
   - 初级项目管理任职资格标准
   - 中级项目管理任职资格标准
   - 高级项目管理任职资格标准
   - 资深项目管理任职资格标准

### 3.4 任职资格等级
任职资格等级与岗位职级对应：
| 岗位职级 | 任职资格等级 | 典型称谓 |
|---------|-----------|---------|
| P1 | 一级 | 助理 |
| P2 | 二级 | 专员/工程师 |
| P3 | 三级 | 主管/高级工程师 |
| P4 | 四级 | 专家/资深工程师 |
| P5 | 五级 | 高级专家 |
| P6 | 六级 | 权威专家 |
| P7 | 七级 | 资深专家 |
| P8 | 八级 | 院士级专家 |
| P9 | 九级 | 商业领袖 |
| P10 | 十级 | 行业领袖 |

## 第4章 任职资格评定

### 4.1 任职资格评定类型
任职资格评定分为以下类型：
1. **入职评定**：新员工入职时进行的任职资格评定，确定初始任职资格等级。
2. **晋升评定**：员工拟晋升时进行的任职资格评定，确认是否达到目标职级任职资格要求。
3. **定期评定**：每1-2年进行一次的任职资格评定，确认员工当前任职资格等级是否匹配。
4. **破格评定**：对做出重大贡献的员工进行的破格任职资格评定。

### 4.2 任职资格评定流程
#### 4.2.1 入职评定流程
1. 新员工入职时，HR部门根据员工学历、工作经验、能力等初步确定任职资格等级。
2. 试用期结束前，部门负责人对新员工进行试用期考核和任职资格评定。
3. HR部门审核评定结果。
4. 确定员工正式任职资格等级。

#### 4.2.2 晋升评定流程
1. 员工本人提出晋升申请或部门推荐。
2. 员工填写《任职资格评定表》，准备相关证明材料。
3. 部门负责人初审。
4. HR部门审核。
5. 组织任职资格评定委员会进行评定。
6. 评定方式包括：
   - 材料审核
   - 面试答辩
   - 能力测试
   - 360度评估
   - 业绩评估
7. 形成评定意见。
8. 人力资源管理委员会审批。
9. 发布评定结果。
10. 如评定通过，办理晋升手续。

#### 4.2.3 定期评定流程
1. HR部门发布定期评定通知。
2. 员工填写《任职资格评定表》，准备相关证明材料。
3. 部门负责人审核。
4. HR部门审核。
5. 组织任职资格评定。
6. 形成评定意见。
7. 审批发布。
8. 根据评定结果调整员工任职资格等级。

#### 4.2.4 破格评定流程
1. 员工本人申请或部门推荐。
2. 说明破格理由和贡献证明。
3. 部门负责人审核。
4. HR部门审核。
5. 人力资源管理委员会特别审议。
6. CEO审批。
7. 发布评定结果。
8. 办理任职资格等级调整手续。

### 4.3 任职资格评定方法
任职资格评定采用以下方法：
1. **材料审核法**：审核员工提交的学历证书、资格证书、工作证明、业绩证明等材料。
2. **面试答辩法**：通过面试和答辩，评估员工的知识、技能和能力。
3. **能力测试法**：通过专业测试、心理测试等，评估员工的能力水平。
4. **360度评估法**：通过上级、同级、下级、客户等多方面评价，评估员工的综合表现。
5. **业绩评估法**：通过分析员工的过往业绩表现，评估员工的能力和贡献。
6. **情景模拟法**：通过模拟工作场景，评估员工的实际工作能力。

### 4.4 任职资格评定委员会
成立任职资格评定委员会，负责任职资格评定工作：
1. **委员会组成**：
   - 主任：人力资源总监
   - 副主任：各业务部门负责人
   - 委员：HR部门负责人、外部专家（必要时）
2. **委员会职责**：
   - 制定任职资格评定标准和流程
   - 组织实施任职资格评定
   - 审议任职资格评定结果
   - 处理任职资格评定争议
3. **工作机制**：
   - 定期召开会议
   - 重大事项集体决策
   - 保密原则

## 第5章 任职资格应用

### 5.1 招聘配置
1. **招聘标准**：任职资格标准作为招聘的重要依据，明确招聘要求。
2. **人岗匹配**：根据任职资格标准，评估候选人与岗位的匹配度。
3. **录用决策**：任职资格评定结果作为录用决策的重要参考。

### 5.2 培训发展
1. **培训需求分析**：根据任职资格标准和员工当前水平，分析培训需求。
2. **培训内容设计**：根据任职资格标准，设计培训内容和课程。
3. **培训效果评估**：根据任职资格标准，评估培训效果。

### 5.3 绩效管理
1. **绩效目标设定**：根据任职资格标准，设定绩效目标。
2. **绩效评估参考**：任职资格作为绩效评估的重要参考。
3. **绩效改进**：根据任职资格标准，制定绩效改进计划。

### 5.4 薪酬管理
1. **薪酬定级**：任职资格等级作为薪酬定级的重要依据。
2. **薪酬调整**：任职资格等级变化作为薪酬调整的重要依据。
3. **薪酬激励**：对达到更高任职资格等级的员工给予薪酬激励。

### 5.5 职业发展
1. **职业规划**：根据任职资格体系，帮助员工制定职业规划。
2. **晋升通道**：任职资格体系明确员工的晋升通道和标准。
3. **发展指导**：根据任职资格标准，为员工提供发展指导和支持。

## 第6章 任职资格动态管理

### 6.1 任职资格等级调整
出现以下情况时，可调整员工任职资格等级：
1. **晋升**：员工达到更高职级任职资格标准，可晋升。
2. **降级**：员工连续两个季度绩效为D，或连续三个季度绩效为C，经评估达不到当前任职资格标准，可降级。
3. **破格晋升**：员工做出重大贡献，可破格晋升。
4. **岗位调整**：员工岗位发生重大变化，需重新评定任职资格等级。

### 6.2 任职资格标准更新
根据以下情况，定期或不定期更新任职资格标准：
1. 公司战略调整。
2. 业务发展变化。
3. 技术进步。
4. 市场环境变化。
5. 法律法规变化。

任职资格标准更新流程：
1. 成立任职资格标准修订小组。
2. 调研分析。
3. 起草新标准。
4. 征求意见。
5. 审核审批。
6. 发布实施。

### 6.3 任职资格档案管理
建立员工任职资格档案，档案包括：
1. 员工基本信息。
2. 任职资格评定表。
3. 相关证明材料。
4. 培训记录。
5. 绩效记录。
6. 其他相关材料。

任职资格档案由HR部门统一管理，永久保存。
