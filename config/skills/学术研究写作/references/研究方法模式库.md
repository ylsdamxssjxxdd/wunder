# 研究方法模式库

## 用途
这份文件把参考项目里最值得保留的研究设计模板压缩成离线可用版本。它不是教科书，而是选题落地时的快速判断工具。

## 使用方式
- 当你还不知道“这题应该用什么方法”时，先看“适用场景”。
- 当你已经知道大致方法时，再看“最小执行结构”和“质量要求”。
- 如果当前资料条件根本不支持该方法，就不要硬套。

## 1. 文献综述
### 适用场景
- 想系统整理一个主题下已有研究。
- 想找共识、分歧和研究缺口。
- 手头已有一批文献或摘录，但还没形成结构。

### 最小执行结构
1. 明确综述主题、时间范围和材料范围。
2. 说明当前材料来自哪里，而不是假装已做完整检索。
3. 按主题、理论、方法或结论路径归类。
4. 比较不同文献之间的共识与冲突。
5. 提炼研究缺口和后续问题。

### 质量要求
- 不能只做摘要拼接。
- 必须有比较、归纳和评价。
- 必须明确“综述基于哪些材料，而不是全领域穷尽检索”。

## 2. 比较案例研究
### 适用场景
- 比较多个组织、地区、项目、政策或制度安排。
- 想看不同情境下某一现象如何变化。

### 最小执行结构
1. 说明为什么选这些案例。
2. 统一比较维度。
3. 先做单案例分析，再做跨案例比较。
4. 找共性、差异和可能解释。

### 推荐比较维度
| 维度 | 案例A | 案例B | 案例C | 说明 |

| 背景 |  |  |  |  |
| 资源条件 |  |  |  |  |
| 执行过程 |  |  |  |  |
| 结果表现 |  |  |  |  |
| 关键差异 |  |  |  |  |

### 质量要求
- 案例选择理由必须写清楚。
- 比较维度必须一致。
- 不能只挑对自己观点有利的案例。

## 3. 政策分析
### 适用场景
- 评估一项政策或制度安排是否有效。
- 比较不同方案、不同地区或不同执行路径。

### 最小执行结构
1. 明确政策问题是什么。
2. 写清政策目标，而不是只写政策名称。
3. 设定评估标准，例如效果、成本、公平性、可执行性。
4. 比较不同选项的收益与代价。
5. 给出建议时写明前提和权衡。

### 质量要求
- 不只看效果，还要看副作用和执行条件。
- 不把价值判断伪装成客观结论。
- 建议必须有材料基础。

## 4. 混合方法研究
### 适用场景
- 既有数据，又有访谈、观察或文本材料。
- 单一资料类型不足以解释问题。

### 最小执行结构
1. 先分别说明量化材料和质性材料来自哪里。
2. 两条分析线各自独立完成基本处理。
3. 在讨论部分明确两类材料如何互相印证或互相修正。
4. 最终结论必须同时回应两类材料，而不是只偏向其中一类。

### 质量要求
- 不能只是“既有数字又有访谈”这么简单。
- 必须写清两类证据如何整合。

## 5. 文本或文档分析
### 适用场景
- 研究政策文本、制度文件、会议纪要、新闻稿、访谈记录、历史材料。
- 想从文档中提炼主题、立场、叙事结构或制度变化。

### 最小执行结构
1. 确定材料集合。
2. 说明纳入标准。
3. 制定分类框架或编码思路。
4. 从片段上升到主题，再从主题回到研究问题。

### 质量要求
- 文档范围必须透明。
- 不能只摘引，不分析。
- 结论必须能回到文本证据。

## 6. 探索性研究
### 适用场景
- 主题还比较新，现成框架不足。
- 目标是澄清问题、形成假设或搭建解释框架。

### 最小执行结构
1. 先承认问题仍在探索阶段。
2. 通过访谈、观察、案例或材料摘录形成初步主题。
3. 对反例和例外保持敏感。
4. 输出的是“暂时框架”而不是“终局定论”。

### 质量要求
- 必须写清楚研究的开放性和限制。
- 不能把初步观察写成普遍规律。

## 7. 基准比较或对标研究
### 适用场景
- 想比较某对象与标杆对象的差距。
- 想形成改进建议或能力对标结论。

### 最小执行结构
1. 说明为什么选这些标杆。
2. 统一指标口径。
3. 比较差距。
4. 分析差距背后的原因和可借鉴做法。

### 质量要求
- 指标必须可比。
- 要承认背景差异，不做机械复制建议。

## 选型提醒
- 材料以文献为主：优先文献综述。
- 材料以多案例为主：优先比较案例研究。
- 材料以制度与方案为主：优先政策分析。
- 材料类型混合明显：优先混合方法。
- 材料以文本和记录为主：优先文档分析。
- 主题尚不成熟：优先探索性研究。
- 目标是找差距和最佳实践：优先对标研究。

## 常见错误
- 先有结论，再倒找方法。
- 方法名字很大，实际材料根本撑不住。
- 研究设计写得很正式，但正文只是资料堆积。
- 把“没有资料做完整检索”写成“已经系统回顾全部研究”。
