# 终稿交付清单

## 用途
在 Markdown 转 DOCX 或正式提交前，做最后一轮交付级检查。

## 结构检查
- 标题是否明确。
- 各章节是否完整。
- 图表是否编号并在正文中被引用。
- 参考文献区是否存在且与正文对应。

## 内容检查
- 核心结论是否都能回到材料。
- 是否有待核对信息仍留在正文关键位置。
- 是否存在过度推断。
- 是否存在明显重复段落。

## 图表检查
- 图表是否真的增加信息量。
- 图注是否清楚。
- 表头、单位、图例是否完整。
- 图片在 DOCX 中是否被遮挡、压缩或错误分页。

## 文档检查
- DOCX 中是否残留 Markdown 标记。
- 标题、表格、图片是否居中或对齐异常。
- 是否出现固定行距压图、压表或断字异常。
- 页边距、字体、段前段后是否统一。

## 文件交付检查
- 是否同时保留 Markdown 源稿。
- 是否保留图表原文件。
- 是否给出清晰文件名，例如“主稿-终版.docx”。
- 如需 PDF，是否从最终版 DOCX 再导出而非中间稿。
