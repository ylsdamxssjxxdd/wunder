# 文献综述与证据整合

## 用途
把分散的文献、报告或本地材料，整理成可写入正文的综述与证据链。

## 一步到位的流程
1. 先定义综述问题。
2. 再确定材料范围。
3. 然后做文献矩阵或来源矩阵。
4. 之后按主题归类，而不是按来源顺序堆叠。
5. 最后写出共识、分歧、缺口与下一步问题。

## 材料范围控制
- 先写清楚纳入什么。
- 再写清楚排除什么。
- 如果来源是用户给的、本地的、内部的，要把边界写明。
- 如果来源是外部检索来的，要记录检索口径，但不要把没核实的来源写成既定事实。

## 文献矩阵建议

| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 待补充 | 待补充 | 待补充 |  |  |  |  |  |  |

## 主题整合方式
### 1. 按共识写
先写多个来源一致支持的结论。

### 2. 按争议写
再写存在分歧、需要解释的地方。

### 3. 按缺口写
最后写当前材料仍然无法回答的问题。

## 证据分层建议
- 强证据：方法清楚、结果与结论匹配、可直接支撑正文主张。
- 中证据：可支持辅助判断，但不宜单独承载核心结论。
- 弱证据：只能做背景或线索。
- 待核对：只能在草稿里保留，不宜直接定稿。

## 常见错误
- 把摘要拼接成综述。
- 只按时间排，不按主题排。
- 不区分一手证据和二手转述。
- 把局部材料写成全局结论。
- 缺少“为什么这样归纳”的解释。
