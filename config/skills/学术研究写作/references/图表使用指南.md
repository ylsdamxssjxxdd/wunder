# 图表使用指南

## 目标
让研究文稿里的图表真正帮助理解，而不是装饰。

## 先问三个问题
1. 这张图要回答什么问题。
2. 图表比正文或表格是否更清楚。
3. 是否有足够的数据或结构信息支撑它。

## 图表选择
### 用表格
- 适合比较
- 适合矩阵
- 适合清单
- 适合证据等级

### 用流程图
- 适合步骤
- 适合阶段
- 适合方法路径

### 用概念图
- 适合关系
- 适合框架
- 适合层级
- 适合模块

### 用示意图
- 适合系统结构
- 适合工作流
- 适合架构

## 图表基本要求
- 图表标题要直接说明内容。
- 图注要解释读者离开正文后还能看懂什么。
- 图表里的术语要和正文一致。
- 不要用图表重复正文已经很清楚的内容。

## 什么时候必须问用户
- 需要真实数据画图，但当前没有数据。
- 需要用户确认想要的图形类型。
- 需要用户确认图里是否能出现具体名称或业务信息。

## 什么时候可以自行生成
- 文献矩阵
- 研究流程图
- 研究问题树
- 证据评估表
- 章节检查清单
- 概念结构图

## 常见错误
- 为了专业而画图。
- 图中字段太多，看不懂。
- 图表和正文说的是两回事。
- 把不确定的数据画成确定图。
