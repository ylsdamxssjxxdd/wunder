# 气象服务表达规范

## 本专题离线条目

| 条目 | 文件 | 用途 |
| --- | --- | --- |
| 气象风险提示话术 | `气象风险提示话术.md` | 生成公众服务风险提示和防御建议 |
| 决策服务产品写法 | `决策服务产品写法.md` | 组织气象专报、影响研判和决策服务文本 |

## 基本表达结构

面向用户回答气象问题时，推荐使用：

```markdown
结论：……
依据：……
影响：……
建议：……
不确定性：……
来源：……
```

短回答可以压缩为“结论 + 依据 + 边界”，但不要省略关键不确定性。

## 不确定性表达

| 场景 | 推荐表达 |
| --- | --- |
| 缺少实时资料 | 当前技能不含实时观测和官方预报，以下仅为通用知识解释。 |
| 资料支持较弱 | 现有材料只能支持初步判断，仍需结合实时资料核验。 |
| 法规版本不明 | 该结论需以官方原文和最新有效版本为准。 |
| 地方规则可能不同 | 具体阈值、流程和防御指南需核验所在地现行规定。 |
| 高影响天气 | 请持续关注当地气象主管机构发布的最新预报预警信息。 |

## 公众服务表达

- 用清楚的天气对象和影响描述替代专业术语堆叠。
- 防御建议要可执行，例如减少涉水出行、远离临时搭建物、检查排水设施。
- 不要制造恐慌，不要使用绝对化措辞。
- 不要将概率性判断写成确定结果。

## 决策服务表达

- 先说风险等级或关注重点，再解释气象依据。
- 明确时间窗口、影响区域、关键触发条件和更新节点。
- 对跨部门协同事项，表达为“建议关注/建议核验/建议准备”，不要代替调度命令。
- 对资料不足之处给出补充资料清单。

## 法规政策表达

法规政策类回答避免“拍板式结论”。推荐：

```markdown
从本地索引可确认，该问题涉及……。可优先核验……。在未读取官方最新原文前，只能概括为……。
```

不得把法规摘要写成法律意见；不得编造条号、施行日期或处罚依据。

## 引用格式

本地技能资料：

```markdown
来源：通用气象知识/references/02-预报预警业务/index.md
```

官方网页：

```markdown
来源：[文件名或页面标题](https://example.com)
```

法规政策：

```markdown
依据：《文件名》，官方入口：……
```
