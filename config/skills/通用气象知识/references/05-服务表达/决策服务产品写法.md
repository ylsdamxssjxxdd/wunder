# 决策服务产品写法

## 元信息

- 分类：服务表达
- 关键词：决策服务、气象专报、风险研判、影响分析、服务产品、摘要
- 来源等级：B
- 适用范围：离线组织面向管理或行动决策的气象服务文本
- 使用边界：不替代官方专报、应急预案或调度命令

## 推荐结构

1. `摘要`：一句话说明主要天气、时间窗、影响重点。
2. `天气形势`：简述触发系统和主要气象要素。
3. `影响研判`：把天气翻译成可能影响。
4. `关注区域/对象`：写清楚高暴露场景。
5. `建议`：给出准备、核验和更新建议。
6. `不确定性`：说明资料缺口和需要持续跟踪的因素。

## 写作原则

- 先给结论，再给依据。
- 使用时间窗和影响对象，而不是堆专业术语。
- 区分“已发生、可能发生、需要核验”。
- 不替代应急部门发布命令。
- 不编造雨量、风速、温度、站点和预警级别。

## 简版模板

```markdown
摘要：未来一段时间需关注……天气对……的影响。

天气与依据：当前可确认的背景是……，主要依据包括……。

影响研判：可能对……造成……影响，其中……场景风险较高。

建议：建议提前……，持续关注……，并根据官方预报预警和实时观测更新判断。

不确定性：当前仍需核验……。
```
