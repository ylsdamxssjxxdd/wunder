# 预警信号解释通用框架

## 元信息

- 分类：预报预警业务
- 关键词：预警信号、预警级别、防御指南、发布主体、解除、更新、传播
- 来源等级：B
- 适用范围：离线解释预警信号含义和用户理解方式
- 使用边界：具体预警级别、阈值、发布权限和防御指南以官方原文和地方现行规定为准

## 核心结论

预警信号是官方发布体系中的风险提示，不是普通天气描述。解释预警时要同时说明天气对象、级别含义、可能影响、防御建议和信息来源，不能替代官方发布或解除。

## 解释顺序

1. 说明预警对象：暴雨、雷电、大风、高温、寒潮、大雾等。
2. 说明级别含义：颜色或级别越高，一般表示风险越高，但具体阈值需查当地规定。
3. 说明影响：对交通、农业、设施、户外活动和公众安全的可能影响。
4. 说明行动：关注官方更新，采取通用防御措施。
5. 说明边界：本地知识库不判断当前是否应发布、升级或解除。

## 防误读要点

- 预警不是灾害已经发生的证明，而是风险提示。
- 没有预警不等于完全没有风险，尤其是局地强对流和突发短时强降水。
- 不同地区同名预警的阈值和防御指南可能有差异。
- 传播预警时要保留来源、发布时间和更新状态。

## 离线回答模板

```markdown
这个问题涉及预警信号解释。离线条件下只能说明通用理解：该类预警通常提示……风险，用户应关注……影响，并核验当地气象主管机构发布的最新预警、解除和防御指南。
```
