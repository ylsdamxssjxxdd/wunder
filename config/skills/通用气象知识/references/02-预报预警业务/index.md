## 基本概念

| 概念 | 含义 | 注意点 |
| --- | --- | --- |
| 天气预报 | 对未来天气状态、天气过程和气象要素的预估 | 需要说明时间范围、空间范围和不确定性 |
| 灾害性天气警报 | 对可能造成灾害影响的天气过程进行警示 | 应与官方发布体系保持一致 |
| 气象灾害预警信号 | 面向社会公众发布的预警信息 | 涉及发布主体、级别、防御指南和传播规范 |
| 临近预报/短临预报 | 面向未来数小时的高频更新预报 | 依赖雷达、卫星、自动站、闪电等实时资料 |
| 影响预报 | 从“天气会怎样”转向“可能影响什么” | 需要结合暴露度、脆弱性、行业场景和防御措施 |

## 业务回答的基本结构

回答预报预警类问题时，优先按以下结构组织：

1. `对象`：天气类型、区域、时间段、用户关心的影响。
2. `依据`：气象形势、观测信号、数值模式、历史经验或法规政策。
3. `判断`：可能发生什么，可信度如何，关键不确定性在哪里。
4. `影响`：对交通、生产、户外活动、设施运行、公众安全等可能影响。
5. `建议`：只给通用防御建议；不替代官方预警、调度命令或专业决策。

## 预警表达边界

- 不要说“我发布某某预警”。只能说“建议关注当地气象部门是否发布或更新相关预警”。
- 不要用本地知识库条目判定具体预警级别。预警阈值和发布权属以官方规则及实时业务资料为准。
- 不要把模型推断、历史案例或雷达形态直接等同于官方预警结论。
- 需要行动建议时，使用“关注、减少、避开、提前检查、留意更新”等保守措辞。

## 临近预报常用资料

| 资料 | 主要用途 | 局限 |
| --- | --- | --- |
| 天气雷达 | 识别降水回波、强对流结构、移动方向、径向速度特征 | 受地物遮挡、距离、波束高度、速度模糊、非气象回波影响 |
| 卫星云图 | 识别云系发展、对流云顶、台风云系、冷暖云区 | 对地面降水强度和近地层风场不能直接替代观测 |
| 自动站 | 识别降水、风、温湿、气压等地面实况 | 受站点代表性、质控、局地环境影响 |
| 闪电定位 | 识别雷暴活动和对流增强 | 无闪电不代表无强降水或无强风风险 |
| 数值模式 | 提供环境场、天气系统演变和情景参考 | 对小尺度强对流时空位置仍有不确定性 |

## 常见问法处理

### “这个天气会不会危险？”

先确认用户是否提供时间、地点和实时资料。没有实时资料时，应回答通用风险识别框架，而不是给确定结论。

### “某个预警是什么意思？”

先检索法规政策索引，再解释预警类别、级别含义、防御指南和发布主体。地方预警信号存在差异时，要求核验当地规定。

### “能不能帮我写一段天气风险提示？”

使用服务表达规范。内容应包含天气对象、影响时段、可能影响、防御建议和信息来源，不要虚构实时实况。
