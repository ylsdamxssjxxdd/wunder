# 临近预报工作框架

## 元信息

- 分类：预报预警业务
- 关键词：临近预报、短临预报、0-2小时、0-6小时、雷达外推、卫星监测、自动站、闪电、集合判断
- 来源等级：B
- 适用范围：离线解释临近预报思路、资料使用顺序和常见局限
- 使用边界：不含实时资料，不能替代业务值班和官方预报

## 核心结论

临近预报的核心不是单纯“看一张雷达图”，而是把连续雷达、卫星、自动站、闪电和环境场放在同一时间轴上，判断天气系统是否正在发生、是否增强、会向哪里移动，以及影响会持续多久。离线场景下只能提供判读框架，不能给出真实落区和强度。

## 标准流程

1. 明确对象：天气类型、关注区域、未来时间窗和用户关心的影响。
2. 看连续演变：至少比较多个连续时次，判断新生、增强、合并、减弱或移出。
3. 看组织形态：单体、线状、片状、螺旋雨带、层状降水、非气象回波分别处理。
4. 看多源一致性：雷达回波是否与卫星云图、闪电、自动站实况一致。
5. 看环境支撑：水汽、能量、垂直风切变、触发边界是否支持继续发展。
6. 输出风险：用“可能影响、重点关注、仍需核验”表达，不写成确定命令。

## 资料使用顺序

| 资料 | 用途 | 关键问题 |
| --- | --- | --- |
| 雷达连续体扫 | 回波强度、结构、移速、移向、速度特征 | 是真降水还是非气象回波，是否增强 |
| 卫星云图 | 云系发展、云顶温度、对流爆发、台风结构 | 云顶是否快速冷却，是否有新生云团 |
| 自动站 | 降水、风、温湿压实况 | 实况是否验证雷达信号 |
| 闪电 | 雷暴活跃程度和对流发展 | 闪电频次是否快速增加 |
| 环境场 | 水汽、能量、风切变、触发条件 | 后续是否仍有维持条件 |

## 常见错误

- 只看最后一张雷达图，不看演变。
- 只看反射率颜色，不看速度、距离、波束高度和质控。
- 把外推当成预报结论，忽略新生和减弱。
- 没有把地面实况作为核验。
- 忽略地形、海陆风、边界层辐合等局地触发。

## 离线回答模板

```markdown
这是一个临近预报问题。离线条件下可以按以下框架判断：
第一，确认连续雷达回波是否持续增强或向目标区域移动；
第二，核验卫星、自动站、闪电是否支持对流发展；
第三，结合环境场判断是否仍有水汽、能量和触发条件；
第四，只能给出关注重点和资料清单，不能替代官方短临预报。
```
