# OSINT 方法论

## 核心原则

### 1. 情报循环

```
规划 -> 采集 -> 处理 -> 分析 -> 传递
    ^                                                    |
    +----------------------------------------------------+
```

**规划：** 定义目标、范围与需求
**采集：** 从来源收集原始数据
**处理：** 整理与格式化采集数据
**分析：** 提炼含义、识别模式
**传递：** 向相关方汇报发现

### 2. 来源层级

**一级 - 一手来源：**
- 官方登记机构（SEC、SoS、USPTO）
- 法院记录（PACER、州法院）
- 政府数据库
- 公司官方披露

**二级 - 可核实二手来源：**
- 成熟新闻媒体
- 学术出版物
- 行业报告
- 专业数据库（Crunchbase、LinkedIn）

**三级 - 社区/社交来源：**
- 社交媒体账号
- 论坛讨论
- 评价网站
- 众包数据

**四级 - 技术来源：**
- DNS 记录
- WHOIS 数据
- 证书透明度
- Shodan/Censys

### 3. 多来源验证

**最低验证门槛：**
- 关键主张：3 个以上独立来源
- 重要主张：2 个以上独立来源
- 支撑性主张：1 个可验证来源

**独立性标准：**
- 不同组织
- 不同采集方式
- 必要时为不同时间段

## 采集方法

### 并行研究模式

**同时部署多个研究代理：**

```typescript
// 示例：公司研究编队
Task({ subagent_type: "PerplexityResearcher", prompt: "Entity verification" })
Task({ subagent_type: "ClaudeResearcher", prompt: "Leadership backgrounds" })
Task({ subagent_type: "GeminiResearcher", prompt: "Competitive analysis" })
Task({ subagent_type: "GrokResearcher", prompt: "Risk assessment" })
```

**优势：**
- 更快采集
- 视角多样
- 覆盖冗余
- 内置交叉验证

### 技术 vs. 研究分工

**技术工具用于：**
- DNS 枚举
- IP 地理定位
- 证书分析
- 端口扫描（已授权）
- WHOIS 查询

**研究代理用于：**
- 商业情报
- 口碑/声誉调查
- 威胁情报
- 历史分析
- 事实验证

## 分析框架

### 置信度等级

**高（80-100%）：**
- 多个独立来源确认
- 官方来源验证
- 直接观察/访问
- 无相反证据

**中（50-79%）：**
- 有支持性证据
- 独立确认有限
- 单一可信来源
- 轻微矛盾已解释

**低（20-49%）：**
- 单一未验证来源
- 间接证据
- 信息缺口大
- 存在矛盾

**推断性（<20%）：**
- 仅基于推理
- 无直接证据
- 信息冲突明显
- 未经确认的模式匹配

### 风险信号分级

**严重（阻断调查）：**
- 欺诈迹象
- 监管违规
- 关键事实虚假陈述
- 刑事活动

**高（重大关注）：**
- 关键注册缺失
- 主张无法验证
- 透明度不足
- 既往监管问题

**中（值得记录）：**
- 轻微不一致
- 线上存在感弱
- 行业性风险
- 竞争劣势

**低（持续关注）：**
- 轻微缺口
- 常规商业风险
- 行业常见问题

## 域名优先协议

**公司 OSINT 中，域名发现是阻断性步骤：**

1. 必须执行全部 7 种枚举技术：
   - 证书透明度
   - DNS 枚举
   - 搜索引擎发现
   - 社媒链接提取
   - 工商/注册资料网站字段
   - WHOIS 反向查询
   - 相关 TLD 检查

2. 质量门槛：95%+ 置信度后才可进入下一阶段

3. 对发现域名分类：
   - 主站
   - 投资者门户
   - 营销/活动站
   - 产品门户
   - 邮件域名
   - 开发/预发布环境

4. 记录缺口与置信度

**意义：**
避免遗漏投资者门户或替代 TLD（如 .partners、.capital、.fund）导致的情报缺口。

## 质量门槛

### 阶段切换要求

**进入下一阶段前：**
- [ ] 所有必需技术已执行
- [ ] 置信度达到阈值
- [ ] 缺口已记录
- [ ] 风险信号已标注
- [ ] 验证已完成

**若未通过质量门槛：**
1. 记录缺口
2. 追加采集
3. 重新评估置信度
4. 达到阈值后再继续
5. 或记录限制并带免责声明继续

## 报告标准

### 必要组成

1. **执行摘要**
   - 关键发现
   - 风险评估
   - 结论建议

2. **方法论**
   - 参考来源
   - 使用工具
   - 采集时间线
   - 约束与限制

3. **分类发现**
   - 商业/实体信息
   - 技术基础设施
   - 口碑/媒体
   - 风险因素

4. **置信度评估**
   - 单条发现置信度
   - 总体置信度
   - 信息缺口

5. **建议**
   - 下一步行动
   - 追加调查方向
   - 风险缓解措施

### 报告质量清单

- [ ] 所有主张有来源
- [ ] 置信度已标注
- [ ] 矛盾已处理
- [ ] 缺口已承认
- [ ] 方法论透明
- [ ] 建议可执行

## 集成点

### 技能调用

**OSINT 会自动调用：**
- **Research Skill** - 用于大规模多代理研究
- **Recon Skill** - 技术侦察（如可用）

### 代理编队规模

- **快速查询：**4-6 名代理
- **标准调查：**8-16 名代理
- **全面尽调：**24-32 名代理

### 超时管理

- **标准研究：**5 分钟
- **深度研究：**10 分钟
- **以已返回结果继续**
