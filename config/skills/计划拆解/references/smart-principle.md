# SMART 原则详解

SMART 原则是目标设定的经典框架，确保目标清晰、可执行、可衡量。

## S - Specific（具体的）

目标必须明确具体，不能模糊笼统。

**错误示例**：
- "提高工作效率"
- "加强团队建设"
- "优化业务流程"

**正确示例**：
- "将项目交付周期从30天缩短至20天"
- "完成3次团队技能培训，覆盖全员"
- "将客户投诉处理时长从48小时降至24小时"

**检查要点**：
- [ ] 目标描述是否清晰无歧义？
- [ ] 是否明确了"做什么"？
- [ ] 是否界定了范围边界？

## M - Measurable（可衡量的）

目标必须有量化指标，能够客观判断是否达成。

**关键问题**：
- 如何衡量目标完成度？
- 有哪些关键绩效指标（KPI）？
- 达成标准是什么？

**常用指标类型**：
| 类型 | 示例 |
|------|------|
| 数量型 | 完成10个项目、培训100人次 |
| 比例型 | 客户满意度达到95%、合格率提升至98% |
| 时间型 | 响应时间<2小时、交付周期≤15天 |
| 成本型 | 预算控制在X万元以内 |

## A - Achievable（可达成的）

目标应当具有挑战性，但必须在现有资源和条件下可实现。

**评估维度**：
1. **资源可行性**：人力、资金、设备是否充足？
2. **技术可行性**：现有技术能力是否支撑？
3. **时间可行性**：时间窗口是否合理？
4. **外部条件**：是否依赖不可控因素？

**常见问题**：
- 目标过高导致团队挫败
- 目标过低失去激励作用
- 忽视资源约束条件

## R - Relevant（相关性）

目标应与上级战略、单位使命、岗位职责相关联。

**对齐检查**：
- 该目标是否支撑上级战略目标？
- 是否与单位年度重点工作相关？
- 是否在责任人职责范围内？
- 是否与其他目标协同一致？

**示例**：
```
单位战略目标：提升市场占有率
    ↓ 支撑
部门目标：开拓3个新客户渠道
    ↓ 支撑
个人目标：完成50次客户拜访
```

## T - Time-bound（有时限的）

目标必须有明确的起止时间和关键节点。

**时间要素**：
- **启动时间**：何时开始执行？
- **截止时间**：何时必须完成？
- **检查节点**：何时评估进度？
- **里程碑**：关键成果何时交付？

**时间分解示例**：
```
年度目标：完成系统升级改造
├── Q1：完成需求调研与方案设计（3月底）
├── Q2：完成系统开发与测试（6月底）
├── Q3：完成试点部署与优化（9月底）
└── Q4：全面推广上线（12月底）
```

## SMART 检查清单

设定目标时，逐项检查：

| 要素 | 检查问题 | 是否通过 |
|------|----------|----------|
| S | 目标描述是否具体明确？ | □ |
| M | 是否有量化衡量指标？ | □ |
| A | 现有条件下是否可达成？ | □ |
| R | 是否与战略目标相关？ | □ |
| T | 是否有明确时间节点？ | □ |

只有五项全部通过，目标才算符合 SMART 原则。
