# RACI 责任矩阵

RACI 矩阵是一种明确任务责任分工的工具，确保每项任务都有清晰的责任归属。

## RACI 含义

| 代码 | 英文 | 含义 | 职责说明 |
|------|------|------|----------|
| **R** | Responsible | 执行者 | 实际完成任务的人/团队 |
| **A** | Accountable | 审批者 | 对结果负责并有权决策的人 |
| **C** | Consulted | 咨询方 | 提供意见或信息的人/部门 |
| **I** | Informed | 知会方 | 需要知晓结果的人/部门 |

## 核心规则

### 规则1：每个任务必须有且仅有一个 A（审批者）

- A 是最终责任人，" buck stops here "
- 不能多人同为 A，否则责任分散
- 不能没有 A，否则无人负责

### 规则2：R 可以有多个，但建议单一

- 多个 R 需明确分工边界
- 单一 R 更利于责任清晰

### 规则3：C 和 I 可有多个或没有

- C：需要双向沟通的对象
- I：单向通知即可的对象

## 矩阵示例

```
任务                    R      A      C      I
─────────────────────────────────────────────
需求调研              项目组   经理   客户    高管
方案设计              架构师   经理   项目组  客户
开发实施              开发组   经理   架构师  测试组
测试验收              测试组   经理   开发组  客户
上线部署              运维组   经理   开发组  客户
```

## 常见问题

### 问题1：R 和 A 混淆

**错误**：同一人既执行又审批，失去监督
**正确**：R 和 A 应由不同人担任（小任务可例外）

### 问题2：A 缺位

**错误**：任务只有执行者，无人对结果负责
**正确**：明确谁有权批准、谁对结果兜底

### 问题3：全员都是 I

**错误**：把所有人都列为 I，信息泛滥
**正确**：只通知真正需要知晓的人

### 问题4：C 过多

**错误**：咨询方太多，沟通成本过高
**正确**：只咨询关键干系人

## RACI 编制步骤

1. **列出所有任务**：按工作分解结构（WBS）逐项列出
2. **识别所有角色**：列出涉及的岗位/部门/人员
3. **分配 R**：确定每项任务的执行主体
4. **分配 A**：确定每项任务的审批主体
5. **分配 C**：确定需要咨询的对象
6. **分配 I**：确定需要通知的对象
7. **检查验证**：确保每项任务有且仅有一个 A

## 扩展变体

### RACI-VS

| 代码 | 含义 |
|------|------|
| V | Verifier（验证者） |
| S | Signatory（签发者） |

### RASCI

| 代码 | 含义 |
|------|------|
| S | Support（支持者） |

选择适合组织管理模式的变体即可，不必过度复杂。
