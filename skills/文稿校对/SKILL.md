# 文稿校对（公文格式 + 错别字）

## 技能目标
- 检查 `.docx` 文稿是否符合常见公文版式规范（纸张、页边距、标题、正文、标题层级）。
- 查找疑似错别字与常见文本问题（常见误写词、重复字、异常标点）。
- 输出**标准化结果**，便于直接反馈给用户或二次修订。

## 适用范围
- 适用于 Word `.docx` 文稿。
- 若用户提供的是 `.doc`，先提示其另存为 `.docx` 后再校对。
- 校对标准参考 `skills/公文写作` 的默认格式约定。

## 快速流程（必须按顺序）
1. 获取待校对文档路径。
2. 运行校对脚本生成 JSON + Markdown 报告。
3. 按“标准结果模板”向用户汇报：先结论，再问题清单，再修订建议。

## 脚本
脚本路径：`scripts/proofread_docx.py`

示例命令：
```bash
python skills/文稿校对/scripts/proofread_docx.py \
  "C:/path/to/公文.docx" \
  --output-json temp_dir/proofread_result.json \
  --output-md temp_dir/proofread_result.md
```

常用参数：
- `--output-json`：输出结构化校对结果（推荐保留）。
- `--output-md`：输出可直接阅读/转发的 Markdown 报告。
- `--max-findings`：限制最大问题条目数量（默认 `200`）。

## 标准结果模板（回复用户时使用）
1. **总体结论**
   - 合规等级：A/B/C
   - 评分：`xx/100`
   - 是否建议直接发文：是/否
2. **格式问题（按严重度排序）**
   - 列出位置、现状、标准要求、修改建议。
3. **错别字与文本问题**
   - 列出“原文片段 -> 建议修改”。
4. **优先修复清单（Top 3）**
   - 给出最影响合规性的 3 条。
5. **复核建议**
   - 说明是否需要二次校对。

## 质量要求
- 不只报问题，要给可执行修改建议。
- 结论必须与问题数量和严重度一致。
- 若未发现明显问题，明确写“本次未检出明显格式/错字问题”，并提示人工终审。
