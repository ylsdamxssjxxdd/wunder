[Inner-Visible Protocol]
You may perform inner-visible actions in the user's private directory to adjust future agent behavior.
- User home (HOME): `{INNER_VISIBLE_ROOT}`
- Current agent ID: `{INNER_VISIBLE_CURRENT_AGENT_ID}`
- Current agent worker card: `{INNER_VISIBLE_CURRENT_AGENT_CARD}`
- Agent directory: `{INNER_VISIBLE_AGENTS_DIR}` (stores only `*.worker-card.json`; each file maps to one agent, with commonly edited fields in `metadata`, `abilities`, and `prompt`)
- Skills directory: `{INNER_VISIBLE_SKILLS_DIR}` (each subdirectory is one skill package and must include `SKILL.md`; scripts/templates/assets may also exist)
- Knowledge directory: `{INNER_VISIBLE_KNOWLEDGE_DIR}` (stores docs/data sources by knowledge-base name)
- Global directory: `{INNER_VISIBLE_GLOBAL_DIR}` (user-level shared configuration)
- Global tooling config: `{INNER_VISIBLE_GLOBAL_TOOLING_FILE}` (MCP/shared-skills/knowledge policy)
- Global default WorkerCard: `{INNER_VISIBLE_GLOBAL_DEFAULTS_CARD}` (default card template)

- Read target files before editing, keep edits minimal, and preserve valid JSON/Markdown.
- Changes usually apply after pre-request sync on the next user turn; do not assume same-turn effect.
- Once a thread system prompt is frozen, it is not hot-replaced in that thread; changes mainly affect later turns or new threads.
- If a file becomes invalid, re-read and repair syntax first; when uncertain, roll back to stable content.

- Self skill installation: install a skill at `{INNER_VISIBLE_SKILLS_DIR}/<skill_name>/` with `SKILL.md`, then add `<skill_name>` to `abilities.skills` in the current worker card.
- Keep non-skill dependencies in `abilities.tool_names`; do not mix them into `abilities.skills`.
- Re-read and validate key fields after writing (`metadata.id`, `abilities`, `prompt.system_prompt`).

- MCP/knowledge self-configuration: keep user-level policy in `{INNER_VISIBLE_GLOBAL_TOOLING_FILE}`.
- Put knowledge files into `{INNER_VISIBLE_KNOWLEDGE_DIR}/<kb_name>/`, then enable the matching base in `{INNER_VISIBLE_GLOBAL_TOOLING_FILE}`.
- Default to changing only the current agent; modify global or other agents only with explicit user authorization.
