[Inner-Visible Action Rules]
- Read target files before editing, keep edits minimal, and preserve valid JSON/Markdown.
- File changes usually apply after pre-request sync on the next user turn; do not assume same-turn effect.
- Once a session thread system prompt is frozen, it is not hot-replaced in that thread; changes mainly affect later turns or new threads.
- If config becomes invalid, inspect diagnostics first, then repair or rollback.

[Self Skill Installation Flow]
1. Install the skill directory into `{INNER_VISIBLE_SKILLS_DIR}/<skill_name>/` and ensure `SKILL.md` exists.
2. Add `<skill_name>` to `abilities.skills` in the target agent `worker-card.json`.
3. Keep non-skill dependencies in `abilities.tool_names`; do not mix them into `abilities.skills`.
4. Re-read and validate key fields after writing; check diagnostics when needed.

[MCP/Knowledge Self-Configuration]
- Supported: update user-level config in `{INNER_VISIBLE_GLOBAL_TOOLING}` (`mcp.servers`, `skills.shared`, `knowledge.bases`, `shared_tools`, etc.).
- Supported: place knowledge files in `{INNER_VISIBLE_KNOWLEDGE_DIR}/<kb_name>/` and then enable matching knowledge-base entries in config.
- Default to changing only the current target agent; modify global or other agents only with explicit user authorization.
