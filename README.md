# wunder 心舰

> 一句话：心舰是一个“会执行任务”的智能体系统。你给目标，它会拆解任务、调用工具并交付结果。

它当前主推 **wunder-desktop**（下载即用），同时底层内核也支撑 `wunder-server` 与 `wunder-cli`。

## 这份 README 解决什么

- 你第一次接触 wunder，不想先看大段架构文档
- 你希望 3 步跑起来，先拿到可用结果
- 你想快速搞清“心舰 -> 用户 -> 蜂群 -> 智能体 -> 线程”这套结构

## 系统结构示意图

```text
心舰（wunder 平台）
├─ 用户 1
│  ├─ 蜂群 1
│  │  ├─ 智能体 1
│  │  │  ├─ 线程 1
│  │  │  └─ 线程 2
│  │  └─ 智能体 2
│  └─ 蜂群 2
│     └─ ...
├─ 用户 2
│  └─ ...
└─ ...
```

- `用户`：系统隔离与资源归属的顶层主体。每个用户有自己的会话与工作区。
- `蜂群`：围绕一个业务目标组织的一组智能体协作单元。
- `智能体`：具体执行角色，负责规划、调用模型与工具、产出结果。
- `线程`：智能体内部的连续执行上下文，承载对话历史、记忆注入与状态演进。

你可以把它理解为：**目标先落到用户，再在蜂群中分工，由智能体在各自线程内连续执行。**

## 先用起来（3 步）

1. 打开项目 Releases 页面，下载与你系统匹配的 `wunder-desktop` 安装包（以发布页为准）。
2. 安装/解压并启动 `wunder-desktop`。
3. 在输入框直接描述任务并回车。

小提示：只想体验的话，到这一步就够了。

## 按你的目标选入口

| 你的目标 | 推荐入口 | 说明 |
| --- | --- | --- |
| 我只想马上用起来 | `wunder-desktop` | 本地主推形态，门槛最低 |
| 我需要团队协作与统一治理 | `wunder-server` | 多用户、多租户、权限与渠道统一接入 |
| 我需要脚本化自动执行 | `wunder-cli` | 终端任务、批处理、自动化流水线 |

## 你可以让心舰做什么

- 办公场景：写文档、做方案、整理材料、会议纪要、表格与演示生成
- 研发场景：写代码、重构、生成脚本、排查问题、自动化执行
- 集成场景：连接 MCP/外部系统，组合 Skills 固化流程
- 持续任务：定时任务、周期巡检、跨渠道消息处理
- 自举场景：让 wunder 参与构建 wunder（代码、文档、任务执行）

## 核心能力一览（基于当前实现）

| 能力 | 具体说明 | 常见用途 |
| --- | --- | --- |
| 桌面控制与自动化 | 本地桌面形态可在授权后操作应用、文件与网页 | 整理资料、批量处理、生成文档 |
| MCP 工具体系 | 支持 `/wunder/mcp`，自托管与跨系统工具接入 | 连接外部服务、跨系统调用 |
| Skills 流程化能力 | 以技能形式固化流程与步骤 | 报告生成、固定流程一键执行 |
| 多智能体并行协作 | 多个智能体并行、互相交接 | 一个查资料、一个写稿、一个复核 |
| 定时任务与调度 | 网关/渠道/定时任务支持长期自动化 | 定时整理、自动提醒、周期任务 |
| 长会话与记忆 | 上下文压缩 + 长期记忆 | 长期项目、持续迭代 |
| 多渠道 | 每个智能体可接入 QQ、飞书等通讯平台 | 多端一致体验、稳定接入 |

## 三种运行形态

| 形态 | 适合谁 | 一句话 |
| --- | --- | --- |
| **wunder-desktop（推荐）** | 普通用户 / 个人 | 下载即用，本地桌面智能体 |
| `wunder-server` | 团队 / 组织 | 统一接入、权限管理、多租户协作 |
| `wunder-cli` | 开发者 / 自动化 | 命令行驱动、脚本化任务执行 |

> 只想快速上手：选 desktop。要组织级协作：看 server。要自动化脚本：用 cli。

## 最容易搞错的点

- `/wunder` 接口的 `user_id` 不要求是已注册用户。
- token 统计记录的是**上下文占用量**，不是账单总消耗量。
- 会话分“用户轮次 / 模型轮次”：一个用户轮次里可能有多次模型动作。
- 通讯链路优先 WebSocket，SSE 是兜底。

## 常见问题

**Q: 我需要先部署 server 吗？**  
A: 不需要。多数个人用户直接用 `wunder-desktop` 即可，只有团队治理与深度集成才需要 server。

**Q: 我不懂技术也能用吗？**  
A: 可以。你只要描述目标，系统会自动拆解并执行。

**Q: 多智能体怎么触发？**  
A: 直接给目标即可，系统会自动分工；你也可以明确要求“并行完成”。
