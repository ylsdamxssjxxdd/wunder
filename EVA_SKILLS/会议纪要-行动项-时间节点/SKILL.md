# 会议纪要 + 待办/行动项 + 时间节点清单

## 概要
用于把会议速记或转写内容整理为结构化纪要，自动提取行动项与里程碑，并输出可复用的 Markdown 与 JSON。

## 快速流程
1. 准备会议记录文本（建议含“行动项/时间节点/决议”标记）。
2. 运行脚本生成 `minutes.md` 与 `minutes.json`。
3. 人工复核负责人、截止日期、结论表述。

## 输入格式建议
### 元信息（可选）
使用“键：值”单行格式：
- `日期：2026-01-24`
- `主持：张三`
- `参会：李四、王五`
- `地点：会议室 1`
- `记录人：赵六`
- `主题：季度 OKR 对齐`

### 行动项（建议）
- `行动项：` 或 `待办：` 开头并列出条目
- 条目推荐写法：
  - `- 任务：整理需求清单；负责人：李四；截止：2026-01-28；优先级：P1；状态：进行中`

### 时间节点（建议）
- `时间节点：` 或 `里程碑：` 开头并列出条目
- 条目推荐写法：
  - `- 2026-01-30：接口评审`

## 输出结构
- Markdown：包含元信息、议题与讨论、决议、行动项表、时间节点表。
- JSON：结构化字段，便于系统对接或二次处理。

## 脚本用法
```bash
python scripts/minutes_from_notes.py \
  --input examples/meeting_notes.txt \
  --output examples/meeting_minutes.md \
  --json examples/meeting_minutes.json
```

## 注意事项
- 支持日期格式：`YYYY-MM-DD` / `YYYY/MM/DD` / `MM-DD`。
- 未标注负责人或截止日期的行动项会保留为空值，需人工补全。
- 建议将结论与行动项分开书写，降低误判。
